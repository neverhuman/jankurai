# Support

Jankurai support happens through GitHub issues and repository discussions when enabled.

## Before Opening An Issue

Check:

- [README.md](README.md)
- [docs/install.md](docs/install.md)
- [docs/adoption.md](docs/adoption.md)
- [docs/testing.md](docs/testing.md)
- existing issues for similar reports

## Bugs

Include:

- Jankurai version from `jankurai --version`,
- install source and operating system,
- exact command,
- redacted output,
- whether the target repo is public or private,
- generated report paths under `target/jankurai/`, if safe to share.

## Questions

For usage questions, describe the repository shape and the adoption level you are trying to use: observe, agents, full, or ratchet.

## Security

Do not use public issues for vulnerabilities, exposed secrets, prompt-injection bypasses, unsafe file writes, or CI permission escalation. Follow [SECURITY.md](SECURITY.md).
