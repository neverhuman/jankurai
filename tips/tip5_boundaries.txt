Boundaries are the architecture.

Keep ownership narrow:
- `apps/web`: presentation, forms, generated clients
- `apps/api`: authz, request shaping, rate limits, response shaping
- `crates/domain`: pure invariants, no I/O
- `crates/application`: use cases and transactions, not transport logic
- `crates/adapters`: side effects only
- `db/migrations`: schema truth only
- `python/ai-service`: model/data work only

Every exception needs a written exit criteria. An exception without an exit criteria is debt.
