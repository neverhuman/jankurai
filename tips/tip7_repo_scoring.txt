Repo scoring is triage.

Score repos on:
- ownership and navigation
- contracts and boundaries
- proof lanes and test routing
- security and supply chain
- code shape and semantic surface
- data truth and workflow safety
- observability and repair evidence
- context economy and agent instructions
- Python containment
- build speed signals

Hard caps matter more than averages. Missing root instructions, one-command validation, a fast lane, or security scanning should cap the score.

Every finding should include severity, category, path, problem, fix, and evidence. The fix queue should point to the smallest next change.
