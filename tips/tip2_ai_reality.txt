AI changes the ranking rule.

Assume models can learn syntax quickly. The real question is whether the stack makes wrong output easy to catch.

Prefer stacks that:
- fail fast in compiler and tests
- keep boundaries typed and generated
- make security checks part of the normal loop
- keep production behavior observable and auditable

Use "second place is the first loser" as shorthand: if AI can write it faster than humans can verify it, the stack is wrong.
