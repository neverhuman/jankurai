# Jankurai ProofCell: the one crate I would add

**Proposed crate:** `jankurai-proofcell`  
**Import name:** `jankurai_proofcell`  
**Cargo subcommand:** `cargo jankurai-proofcell`  
**Jankurai command:** `jankurai proofcell verify`  
**Primary output:** `target/jankurai/proof-cells/index.json` plus a Jankurai-compatible proof receipt  
**Core value:** turn `cargo test` from an unstructured green/red signal into behavior-linked, owner-routed, security-aware, tamper-evident proof that Jankurai can use in the merge witness.

---

## 1. Executive decision

The best single new crate is **`jankurai-proofcell`**, a small Rust dev-tooling crate that makes tests carry machine-readable proof metadata.

Jankurai already has the right control-plane: owner maps, test maps, proof lanes, proof receipts, security and UX evidence, merge witnesses, score history, and repair queues. The biggest remaining gap is that a passing proof lane can still be a **false green**: tests might be shallow, over-mocked, snapshot-only, missing assertions, missing negative authorization/input cases, unrelated to the changed behavior, or impossible for an agent to map back to the product invariant being claimed.

`jankurai-proofcell` upgrades the most common Rust proof primitive, `cargo test`, into a Jankurai-native evidence source. It gives each important test a **ProofCell**:

```rust
use jankurai_proofcell::{proof_cell, jk_assert, jk_negative};

proof_cell! {
    id: "orders.cancel.rejects_cross_tenant_user",
    owner: "crates/application",
    lane: "fast",
    rules: ["HLT-008-FALSE-GREEN-RISK", "HLT-022-AUTHZ-ISOLATION-GAP"],
    changed: ["crates/application/src/orders/**", "crates/domain/src/orders/**"],
    test: cancel_rejects_cross_tenant_user {
        let result = cancel_order_as_other_tenant();

        jk_negative!("cross_tenant_user_is_denied", result.is_err());
        jk_assert!("error_has_stable_reason", result.unwrap_err().reason_code() == "ORDER_FORBIDDEN");
    }
}
```

When the lane runs:

```bash
cargo jankurai-proofcell run \
  --changed-from origin/main \
  --nextest \
  --out target/jankurai/proof-cells/index.json \
  --receipt target/jankurai/proof-receipts/proofcell.json

cargo run -p jankurai -- witness . \
  --changed-from origin/main \
  --proof-receipts target/jankurai/proof-receipts \
  --out target/jankurai/merge-witness.json \
  --md target/jankurai/merge-witness.md
```

Jankurai can now answer the question agents usually blur:

> “Which behavior, invariant, security rule, owner, and changed path did this passing test actually prove?”

---

## 2. Why this is the best idea for Jankurai

### 2.1 It closes the most dangerous gap in the standard

Jankurai’s thesis is not “run more checks.” It is “changed paths must bind to owners, proof lanes, evidence receipts, and a merge witness.” The current repo already has `jankurai prove`, `proof-verify`, `witness`, `security run`, `ux-qa`, `rust witness`, route maps, schema validation, and repair queues.

That leaves one extremely high-leverage missing primitive: **semantic proof quality**.

A proof receipt can say a command ran. It cannot always prove that the tests were relevant, had real assertions, covered the changed behavior, included negative cases for authz/input boundaries, avoided fake mocks, or were tied to stable rule IDs.

`jankurai-proofcell` gives Jankurai a first-class way to detect and repair:

- `HLT-008-FALSE-GREEN-RISK`
- `HLT-022-AUTHZ-ISOLATION-GAP`
- `HLT-023-INPUT-BOUNDARY-GAP`
- `HLT-018-PERF-CONCURRENCY-DRIFT`
- `HLT-027-HUMAN-REVIEW-EVIDENCE-GAP`
- parts of `HLT-016-SUPPLY-CHAIN-DRIFT` when dependency changes require extra proof

### 2.2 It improves a standard piece of code everyone depends on

Almost every Rust repository already depends on `cargo test` as the default local proof gate. `cargo test` is useful, but its output is not agent-native. It is mostly a boolean: pass/fail.

`jankurai-proofcell` makes `cargo test`, `cargo nextest`, `miri`, and `kani` emit durable proof receipts without requiring product runtime dependencies. It is a dev dependency and CI tool.

### 2.3 It reduces vibe coding at the point agents fake confidence

Agents often produce plausible tests that validate implementation details instead of product invariants. ProofCell makes the proof shape explicit:

- What behavior does this prove?
- Which owner owns it?
- Which Jankurai rules does it cover?
- Which changed paths does it route to?
- How many real assertions ran?
- Did a required negative case run?
- Did the test use mocks, snapshots, or generated contracts?
- Which command, commit, and artifact digests produced the receipt?

### 2.4 It is low-friction enough to auto-include

The crate can be added automatically by `jankurai init` as a dev-dependency and proof lane. A repository can adopt it in three steps:

1. Add the crate.
2. Wrap high-value tests with `proof_cell!` and `jk_assert!`/`jk_negative!`.
3. Add one proof lane command.

No product runtime architecture needs to change.

## 3. Research and ecosystem grounding

This proposal is intentionally not another generic linter.

It is a bridge between Jankurai’s proof-carrying merge model and the Rust testing/security ecosystem.

Relevant ecosystem signals:

- AI-generated code and AI-assisted programming can produce vulnerable code and high-confidence wrong answers, so proof receipts should not trust model confidence or shallow tests.
- Recent research on coding-agent commits has specifically examined over-mocked tests in AI-generated changes, which matches Jankurai’s `false-green` concern.
- `cargo-nextest` is already a widely used richer test runner with JUnit, per-test execution, retries, and better CI ergonomics.
- `kani` can prove Rust properties with model checking when a behavior deserves stronger proof than examples.
- `miri` can catch undefined behavior and unsafe-code issues by interpreting Rust tests.
- `cargo-vet`, OpenSSF Scorecard, and SLSA address supply-chain trust; ProofCell should not replace them, but it should route dependency/security-sensitive changes to the right receipt.

## 5. What the crate proves

A ProofCell is a declared test/proof unit.

It can represent:

- unit test
- integration test
- doctest wrapper
- property test
- regression test
- negative authorization test
- input-boundary test
- unsafe-code Miri test
- Kani proof harness
- public API/semver proof
- concurrency/performance-sensitive proof

Each ProofCell emits this contract:

```json
{
  "schema_version": "1.0.0",
  "cell_id": "orders.cancel.rejects_cross_tenant_user",
  "owner": "crates/application",
  "lane": "fast",
  "rules_covered": [
    "HLT-008-FALSE-GREEN-RISK",
    "HLT-022-AUTHZ-ISOLATION-GAP"
  ],
  "changed_path_globs": [
    "crates/application/src/orders/**",
    "crates/domain/src/orders/**"
  ],
  "assertions": 2,
  "negative_cases": 1,
  "mocks": [],
  "snapshots": [],
  "status": "pass",
  "decision": "pass",
  "risk_notes": [],
  "artifacts": [],
  "started_at": "2026-05-05T00:00:00Z",
  "finished_at": "2026-05-05T00:00:01Z"
}
```

The aggregate index ties cells to git, command, manifests, path routing, and artifact digests.

## 6. Merge decisions introduced by ProofCell

ProofCell uses Jankurai’s existing decision vocabulary.

| Condition | ProofCell decision | Typical Jankurai effect |
|---|---|---|
| Required cell ran, passed, has assertion evidence, route matches owner/test map | `pass` | receipt can satisfy mapped lane |
| Cell passed but uses snapshots only, too many mocks, missing optional stronger proof | `review` | human review or repair queue |
| Cell missing, failed, has zero assertions, or required negative cases absent | `block` | merge witness blocks |
| Score/proof regression from baseline | delegated to Jankurai | `ratchet_fail` handled by witness |
| Release contract lacks required proof category | delegated to Jankurai | `release_fail` handled by release gate |

## 8. Adoption policy

ProofCell should be required only where it creates real value.

### Required in HL4/HL5 for

- authorization and tenant/data isolation logic
- input validation boundaries
- durable product invariants
- idempotency and transaction behavior
- unsafe Rust
- public API compatibility
- security-sensitive generated code
- concurrency-sensitive business flows

### Advisory in HL2/HL3 for

- normal leaf functions
- internal refactors
- low-risk pure transformations

### Not required for

- generated files that are already covered by generation receipts
- docs-only changes unless docs claim proof
- purely visual UX proof already covered by UX QA

### `crates/jankurai-proofcell/src/lib.rs`

```rust
//! Jankurai ProofCell.
//!
//! This crate lets Rust tests emit behavior-linked proof evidence that can be
//! consumed by Jankurai proof lanes and merge witnesses.
//!
//! MVP design goals:
//! - dev-dependency only
//! - no network access
//! - one public crate
//! - works with `cargo test` and `cargo nextest`
//! - emits JSON/NDJSON under `target/jankurai/proof-cells/`

mod assertions;
mod digest;
mod event;
mod git;
pub mod model;
pub mod runner;
pub mod verdict;

pub use assertions::{record_assertion, record_mock, record_negative_case};
pub use model::{AssertionKind, MockKind, ProofCellSpec};

use std::panic::{catch_unwind, resume_unwind, AssertUnwindSafe};

/// Guard around one test/proof body.
///
/// The guard emits start/finish events and lets assertion macros attach
/// semantic evidence to the active cell.
pub struct ProofGuard {
    spec: ProofCellSpec,
    finished: bool,
}

impl ProofGuard {
    pub fn start(spec: ProofCellSpec) -> Self {
        assertions::enter_cell(spec.cell_id.clone());
        let _ = event::emit_event(model::ProofEvent::CellStarted {
            at: chrono::Utc::now().to_rfc3339(),
            spec: spec.clone(),
        });
        Self {
            spec,
            finished: false,
        }
    }

    pub fn pass(mut self) {
        self.finished = true;
        assertions::leave_cell();
        let _ = event::emit_event(model::ProofEvent::CellFinished {
            at: chrono::Utc::now().to_rfc3339(),
            cell_id: self.spec.cell_id.clone(),
            status: "pass".to_string(),
            panic_message: None,
        });
    }

    pub fn fail(mut self, message: String) {
        self.finished = true;
        assertions::leave_cell();
        let _ = event::emit_event(model::ProofEvent::CellFinished {
            at: chrono::Utc::now().to_rfc3339(),
            cell_id: self.spec.cell_id.clone(),
            status: "fail".to_string(),
            panic_message: Some(message),
        });
    }
}

impl Drop for ProofGuard {
    fn drop(&mut self) {
        if !self.finished && !std::thread::panicking() {
            assertions::leave_cell();
            let _ = event::emit_event(model::ProofEvent::CellFinished {
                at: chrono::Utc::now().to_rfc3339(),
                cell_id: self.spec.cell_id.clone(),
                status: "unknown".to_string(),
                panic_message: Some("proof guard dropped without explicit finish".to_string()),
            });
        }
    }
}

/// Execute one proof body and preserve panic behavior for the Rust test runner.
pub fn run_proof_body<F>(spec: ProofCellSpec, body: F)
where
    F: FnOnce() + std::panic::UnwindSafe,
{
    let guard = ProofGuard::start(spec);
    match catch_unwind(AssertUnwindSafe(body)) {
        Ok(()) => guard.pass(),
        Err(payload) => {
            guard.fail("test panicked".to_string());
            resume_unwind(payload);
        }
    }
}

/// Define a `#[test]` function with Jankurai proof metadata.
///
/// This macro intentionally stays declarative. That keeps the MVP as a single
/// crate instead of requiring a proc-macro crate.
#[macro_export]
macro_rules! proof_cell {
    (
        id: $id:literal,
        owner: $owner:literal,
        lane: $lane:literal,
        rules: [$($rule:literal),* $(,)?],
        changed: [$($changed:literal),* $(,)?],
        test: $name:ident $body:block
    ) => {
        #[test]
        fn $name() {
            let __spec = $crate::ProofCellSpec {
                cell_id: $id.to_string(),
                owner: $owner.to_string(),
                lane: $lane.to_string(),
                rules_covered: vec![$($rule.to_string()),*],
                changed_path_globs: vec![$($changed.to_string()),*],
                proof_kind: "test".to_string(),
                source_file: file!().to_string(),
                source_line: line!(),
                requires_negative_case: false,
                max_mock_count: None,
                tags: Vec::new(),
            };
            $crate::run_proof_body(__spec, || $body);
        }
    };
    (
        id: $id:literal,
        owner: $owner:literal,
        lane: $lane:literal,
        rules: [$($rule:literal),* $(,)?],
        changed: [$($changed:literal),* $(,)?],
        requires_negative_case: $requires_negative_case:expr,
        test: $name:ident $body:block
    ) => {
        #[test]
        fn $name() {
            let __spec = $crate::ProofCellSpec {
                cell_id: $id.to_string(),
                owner: $owner.to_string(),
                lane: $lane.to_string(),
                rules_covered: vec![$($rule.to_string()),*],
                changed_path_globs: vec![$($changed.to_string()),*],
                proof_kind: "test".to_string(),
                source_file: file!().to_string(),
                source_line: line!(),
                requires_negative_case: $requires_negative_case,
                max_mock_count: None,
                tags: Vec::new(),
            };
            $crate::run_proof_body(__spec, || $body);
        }
    };
}

/// Assert a positive invariant and emit assertion evidence.
#[macro_export]
macro_rules! jk_assert {
    ($assertion_id:literal, $condition:expr $(,)?) => {{
        let __passed = $condition;
        $crate::record_assertion(
            $assertion_id,
            stringify!($condition),
            $crate::AssertionKind::Positive,
            __passed,
            file!(),
            line!(),
        );
        assert!(__passed, "jankurai assertion failed: {}", $assertion_id);
    }};
}

/// Assert a required negative case and emit negative-case evidence.
#[macro_export]
macro_rules! jk_negative {
    ($assertion_id:literal, $condition:expr $(,)?) => {{
        let __passed = $condition;
        $crate::record_assertion(
            $assertion_id,
            stringify!($condition),
            $crate::AssertionKind::Negative,
            __passed,
            file!(),
            line!(),
        );
        $crate::record_negative_case($assertion_id, __passed, file!(), line!());
        assert!(__passed, "jankurai negative-case assertion failed: {}", $assertion_id);
    }};
}

/// Record a mock/stub usage in a proof cell.
///
/// Agents are allowed to use mocks, but the receipt must make mocking visible.
#[macro_export]
macro_rules! jk_mock {
    ($mock_id:literal, $kind:expr $(,)?) => {{
        $crate::record_mock($mock_id, $kind, file!(), line!());
    }};
}
```

```rust
use crate::model::{MockKind, ProofCellReport};

pub fn decide_cell(report: &mut ProofCellReport) {
    let mut block = false;
    let mut review = false;

    if report.status != "pass" {
        block = true;
        report
            .risk_notes
            .push(format!("proof cell status is `{}`", report.status));
        report
            .repair_hints
            .push("fix the failing proof cell before relying on this lane".to_string());
    }

    if report.assertions == 0 {
        block = true;
        report
            .risk_notes
            .push("proof cell executed with zero Jankurai assertions".to_string());
        report.repair_hints.push(
            "replace smoke-only execution with jk_assert! or jk_negative! tied to a behavior".to_string(),
        );
    }

    if report
        .rules_covered
        .iter()
        .any(|r| r.contains("AUTHZ") || r.contains("INPUT-BOUNDARY"))
        && report.negative_cases == 0
    {
        block = true;
        report
            .risk_notes
            .push("authz/input-boundary proof has no negative case".to_string());
        report
            .repair_hints
            .push("add jk_negative! for denied role, tenant, malformed input, or unsafe sink".to_string());
    }

    let non_contract_mocks = report
        .mocks
        .iter()
        .filter(|mock| !matches!(mock.kind, MockKind::ContractStub | MockKind::Clock))
        .count();
    if non_contract_mocks > 2 {
        review = true;
        report
            .risk_notes
            .push(format!("proof cell used {non_contract_mocks} non-contract mocks"));
        report.repair_hints.push(
            "prefer generated contract stubs, fake adapters, or integration containers over broad mocks"
                .to_string(),
        );
    }

    if !report.snapshots.is_empty() && report.assertions == report.snapshots.len() {
        review = true;
        report
            .risk_notes
            .push("snapshot-heavy proof needs semantic assertions".to_string());
        report
            .repair_hints
            .push("add semantic jk_assert! checks next to snapshot updates".to_string());
    }

    report.decision = if block {
        "block".to_string()
    } else if review {
        "review".to_string()
    } else {
        "pass".to_string()
    };
}
```

### `crates/jankurai-proofcell/src/runner.rs`

```rust
use crate::digest::{sha256_file, sha256_string};
use crate::event::EVENTS_ENV;
use crate::git;
use crate::model::{
    ArtifactDigest, AssertionKind, MockObservation, ProofCellDecisionSummary, ProofCellIndex,
    ProofCellReport, ProofEvent, PROOF_CELL_SCHEMA_VERSION,
};
use crate::verdict::decide_cell;
use anyhow::{bail, Context, Result};
use serde_json::Value;
use std::collections::{BTreeMap, BTreeSet};
use std::fs;
use std::path::{Path, PathBuf};
use std::process::Command;

#[derive(Debug, Clone)]
pub struct RunOptions {
    pub repo: PathBuf,
    pub changed_from: Option<String>,
    pub out: PathBuf,
    pub receipt: Option<PathBuf>,
    pub use_nextest: bool,
    pub package: Option<String>,
    pub extra_args: Vec<String>,
    pub continue_on_error: bool,
}

pub fn run(options: RunOptions) -> Result<ProofCellIndex> {
    let event_log = options.repo.join("target/jankurai/proof-cells/events.ndjson");
    if let Some(parent) = event_log.parent() {
        fs::create_dir_all(parent)?;
    }
    fs::write(&event_log, "")?;

    let command_string = command_string(&options);
    let status = run_test_command(&options, &event_log)?;
    let command_exit_code = status.code().unwrap_or(1);

    if command_exit_code != 0 && !options.continue_on_error {
        // Still write the index below, then return an error after persistence.
    }

    let events = read_events(&event_log)?;
    let mut cells = aggregate_cells(events);
    for cell in &mut cells {
        decide_cell(cell);
    }

    let mut decisions = ProofCellDecisionSummary::default();
    let mut rules = BTreeSet::new();
    for cell in &cells {
        match cell.decision.as_str() {
            "pass" => decisions.pass += 1,
            "review" => decisions.review += 1,
            "block" => decisions.block += 1,
            _ => decisions.block += 1,
        }
        for rule in &cell.rules_covered {
            rules.insert(rule.clone());
        }
    }

    let changed_paths = git::changed_paths(&options.repo, options.changed_from.as_deref());
    let required_cells_missing = required_cells_missing_for_changed_paths(&changed_paths, &cells);
    let overall_decision = if command_exit_code != 0
        || decisions.block > 0
        || !required_cells_missing.is_empty()
    {
        "block"
    } else if decisions.review > 0 {
        "review"
    } else {
        "pass"
    }
    .to_string();

    let event_rel = rel(&options.repo, &event_log);
    let event_hash = sha256_file(&event_log)?;
    let mut artifact_digests = vec![ArtifactDigest {
        path: event_rel.clone(),
        sha256: event_hash.clone(),
    }];

    let index = ProofCellIndex {
        schema_version: PROOF_CELL_SCHEMA_VERSION.to_string(),
        generated_at: chrono::Utc::now().to_rfc3339(),
        repo_root: options.repo.display().to_string(),
        git_head: git::git_head(&options.repo),
        dirty_worktree: git::dirty_worktree(&options.repo),
        command: command_string,
        command_exit_code,
        changed_from: options.changed_from.clone(),
        changed_paths,
        event_log: event_rel,
        event_log_sha256: event_hash,
        cells,
        required_cells_missing,
        decisions,
        artifact_digests: artifact_digests.clone(),
        jankurai_rules_covered: rules.into_iter().collect(),
        overall_decision,
    };

    write_json(&options.out, &index)?;
    artifact_digests.push(ArtifactDigest {
        path: rel(&options.repo, &options.out),
        sha256: sha256_file(&options.out)?,
    });

    if let Some(receipt_path) = &options.receipt {
        write_jankurai_receipt(&options.repo, receipt_path, &index, &artifact_digests)?;
    }

    if index.overall_decision == "block" && !options.continue_on_error {
        bail!("proofcell decision is block; see {}", options.out.display());
    }

    Ok(index)
}

fn run_test_command(options: &RunOptions, event_log: &Path) -> Result<std::process::ExitStatus> {
    let mut cmd = if options.use_nextest {
        let mut cmd = Command::new("cargo");
        cmd.args(["nextest", "run"]);
        cmd
    } else {
        let mut cmd = Command::new("cargo");
        cmd.arg("test");
        cmd
    };

    if let Some(package) = &options.package {
        cmd.args(["-p", package]);
    }
    cmd.args(&options.extra_args);
    cmd.env(EVENTS_ENV, event_log);
    cmd.current_dir(&options.repo);
    cmd.status().context("run proofcell test command")
}

fn command_string(options: &RunOptions) -> String {
    let mut parts = if options.use_nextest {
        vec!["cargo".to_string(), "nextest".to_string(), "run".to_string()]
    } else {
        vec!["cargo".to_string(), "test".to_string()]
    };
    if let Some(package) = &options.package {
        parts.extend(["-p".to_string(), package.clone()]);
    }
    parts.extend(options.extra_args.clone());
    parts.join(" ")
}

fn read_events(path: &Path) -> Result<Vec<ProofEvent>> {
    let text = fs::read_to_string(path).with_context(|| format!("read {}", path.display()))?;
    let mut events = Vec::new();
    for (idx, line) in text.lines().enumerate() {
        if line.trim().is_empty() {
            continue;
        }
        let event: ProofEvent = serde_json::from_str(line)
            .with_context(|| format!("parse proofcell event {} line {}", path.display(), idx + 1))?;
        events.push(event);
    }
    Ok(events)
}

fn aggregate_cells(events: Vec<ProofEvent>) -> Vec<ProofCellReport> {
    let mut cells: BTreeMap<String, ProofCellReport> = BTreeMap::new();

    for event in events {
        match event {
            ProofEvent::CellStarted { at, spec } => {
                cells.entry(spec.cell_id.clone()).or_insert(ProofCellReport {
                    schema_version: PROOF_CELL_SCHEMA_VERSION.to_string(),
                    cell_id: spec.cell_id,
                    owner: spec.owner,
                    lane: spec.lane,
                    rules_covered: spec.rules_covered,
                    changed_path_globs: spec.changed_path_globs,
                    proof_kind: spec.proof_kind,
                    source_file: spec.source_file,
                    source_line: spec.source_line,
                    started_at: Some(at),
                    finished_at: None,
                    status: "started".to_string(),
                    decision: "block".to_string(),
                    assertions: 0,
                    positive_assertions: 0,
                    negative_assertions: 0,
                    negative_cases: 0,
                    mocks: Vec::new(),
                    snapshots: Vec::new(),
                    risk_notes: Vec::new(),
                    repair_hints: Vec::new(),
                });
            }
            ProofEvent::CellFinished {
                at,
                cell_id,
                status,
                panic_message,
            } => {
                if let Some(cell) = cells.get_mut(&cell_id) {
                    cell.finished_at = Some(at);
                    cell.status = status;
                    if let Some(message) = panic_message {
                        cell.risk_notes.push(message);
                    }
                }
            }
            ProofEvent::Assertion {
                cell_id,
                kind,
                passed,
                assertion_id,
                ..
            } => {
                if let Some(cell) = cells.get_mut(&cell_id) {
                    cell.assertions += 1;
                    match kind {
                        AssertionKind::Negative => cell.negative_assertions += 1,
                        _ => cell.positive_assertions += 1,
                    }
                    if !passed {
                        cell.risk_notes
                            .push(format!("assertion `{assertion_id}` failed"));
                    }
                }
            }
            ProofEvent::NegativeCase { cell_id, passed, case_id, .. } => {
                if let Some(cell) = cells.get_mut(&cell_id) {
                    cell.negative_cases += 1;
                    if !passed {
                        cell.risk_notes
                            .push(format!("negative case `{case_id}` failed"));
                    }
                }
            }
            ProofEvent::MockUsed {
                cell_id,
                mock_id,
                kind,
                file,
                line,
                ..
            } => {
                if let Some(cell) = cells.get_mut(&cell_id) {
                    if matches!(kind, crate::model::MockKind::Snapshot) {
                        cell.snapshots.push(mock_id.clone());
                    }
                    cell.mocks.push(MockObservation {
                        mock_id,
                        kind,
                        file,
                        line,
                    });
                }
            }
        }
    }

    cells.into_values().collect()
}

fn required_cells_missing_for_changed_paths(
    changed_paths: &[String],
    cells: &[ProofCellReport],
) -> Vec<String> {
    // MVP heuristic: if a changed Rust path is under crates/ and no proof cell
    // declares a glob prefix that covers it, report the path as missing.
    // Jankurai audit integration should replace this with owner/test-map routing.
    let mut missing = Vec::new();
    for path in changed_paths {
        if !path.ends_with(".rs") || !path.starts_with("crates/") {
            continue;
        }
        let covered = cells.iter().any(|cell| {
            cell.changed_path_globs.iter().any(|glob| prefix_glob_matches(glob, path))
        });
        if !covered {
            missing.push(path.clone());
        }
    }
    missing
}

fn prefix_glob_matches(glob: &str, path: &str) -> bool {
    if let Some(prefix) = glob.strip_suffix("/**") {
        path.starts_with(prefix)
    } else if let Some(prefix) = glob.strip_suffix("*") {
        path.starts_with(prefix)
    } else {
        glob == path
    }
}

fn write_json<T: serde::Serialize>(path: &Path, value: &T) -> Result<()> {
    if let Some(parent) = path.parent() {
        fs::create_dir_all(parent)?;
    }
    let text = serde_json::to_string_pretty(value)?;
    fs::write(path, text).with_context(|| format!("write {}", path.display()))
}

fn write_jankurai_receipt(
    repo: &Path,
    path: &Path,
    index: &ProofCellIndex,
    artifact_digests: &[ArtifactDigest],
) -> Result<()> {
    let receipt = serde_json::json!({
        "schema_version": "1.2.0",
        "standard_version": "0.7.0",
        "auditor_version": "0.7.0",
        "receipt_id": format!("proofcell:{}", sha256_string(&index.command)),
        "lane": "proofcell",
        "command": index.command,
        "exit_code": index.command_exit_code,
        "elapsed_ms": 0,
        "artifacts": [rel(repo, path), index.event_log.clone(), "target/jankurai/proof-cells/index.json"],
        "changed_paths": index.changed_paths,
        "git_head": index.git_head,
        "dirty_worktree": index.dirty_worktree,
        "rules_covered": index.jankurai_rules_covered.iter().map(|rule| {
            serde_json::json!({ "rule_id": rule, "status": index.overall_decision })
        }).collect::<Vec<Value>>(),
        "artifact_digests": artifact_digests,
        "extensions": {
            "jankurai.proofcell.index": "target/jankurai/proof-cells/index.json",
            "jankurai.proofcell.decision": index.overall_decision,
            "jankurai.proofcell.pass": index.decisions.pass,
            "jankurai.proofcell.review": index.decisions.review,
            "jankurai.proofcell.block": index.decisions.block,
            "jankurai.proofcell.required_cells_missing": index.required_cells_missing
        }
    });
    write_json(path, &receipt)
}

fn rel(repo: &Path, path: &Path) -> String {
    path.strip_prefix(repo)
        .unwrap_or(path)
        .to_string_lossy()
        .replace('\\', "/")
}
```

```rust
    #[arg(last = true)]

                    .into_iter()

                    .collect(),

                "proofcell: {} pass, {} review, {} block; decision={}",
                index.decisions.pass,
                index.decisions.review,
                index.decisions.block,
                index.overall_decision
            );

    }
```

```rust
use anyhow::{Context, Result};
use serde::{Deserialize, Serialize};
use std::fs;
use std::path::{Path, PathBuf};

use crate::render::{write_json, write_markdown};
use crate::validation::{self, ArtifactSchema};

#[derive(Debug, Clone)]
pub struct ProofcellVerifyArgs {
    pub repo: PathBuf,
    pub index: String,
    pub out: String,
    pub md: String,
    pub strict: bool,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProofCellIndexLite {
    pub schema_version: String,
    pub git_head: String,
    pub command: String,
    pub command_exit_code: i32,
    pub cells: Vec<ProofCellLite>,
    pub required_cells_missing: Vec<String>,
    pub overall_decision: String,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProofCellLite {
    pub cell_id: String,
    pub owner: String,
    pub lane: String,
    pub rules_covered: Vec<String>,
    pub changed_path_globs: Vec<String>,
    pub status: String,
    pub decision: String,
    pub assertions: usize,
    pub negative_cases: usize,
    pub risk_notes: Vec<String>,
    pub repair_hints: Vec<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProofCellVerification {
    pub schema_version: String,
    pub generated_at: String,
    pub index_path: String,
    pub verdict: String,
    pub issues: Vec<String>,
    pub cells_total: usize,
    pub cells_pass: usize,
    pub cells_review: usize,
    pub cells_block: usize,
    pub rules_covered: Vec<String>,
}

pub fn run_verify(args: ProofcellVerifyArgs) -> Result<()> {
    let index_path = resolve(&args.repo, &args.index);
    let text = fs::read_to_string(&index_path)
        .with_context(|| format!("read proofcell index {}", index_path.display()))?;
    let value: serde_json::Value = serde_json::from_str(&text)
        .with_context(|| format!("parse proofcell index {}", index_path.display()))?;

    // Add ArtifactSchema::ProofCellIndex in validation.rs before enabling this.
    validation::validate_value(&args.repo, ArtifactSchema::ProofCellIndex, &value)?;

    let index: ProofCellIndexLite = serde_json::from_value(value)?;
    let verification = verify_index(&args.index, &index);

    write_json(&args.out, &serde_json::to_string_pretty(&verification)?)?;
    write_markdown(&args.md, &render_md(&verification))?;

    if args.strict && verification.verdict == "block" {
        anyhow::bail!("proofcell verification blocked; see {}", args.md);
    }
    Ok(())
}

fn verify_index(index_path: &str, index: &ProofCellIndexLite) -> ProofCellVerification {
    let mut issues = Vec::new();
    let mut pass = 0;
    let mut review = 0;
    let mut block = 0;
    let mut rules = std::collections::BTreeSet::new();

    if index.command_exit_code != 0 {
        issues.push(format!(
            "proofcell command exited {}",
            index.command_exit_code
        ));
    }

    for missing in &index.required_cells_missing {
        issues.push(format!("changed path lacks proof cell: {missing}"));
    }

    for cell in &index.cells {
        match cell.decision.as_str() {
            "pass" => pass += 1,
            "review" => review += 1,
            "block" => block += 1,
            _ => block += 1,
        }
        if cell.assertions == 0 {
            issues.push(format!("cell `{}` has zero assertions", cell.cell_id));
        }
        for rule in &cell.rules_covered {
            rules.insert(rule.clone());
        }
    }

    let verdict = if index.overall_decision == "block" || block > 0 || !issues.is_empty() {
        "block"
    } else if index.overall_decision == "review" || review > 0 {
        "review"
    } else {
        "pass"
    };

    ProofCellVerification {
        schema_version: "1.0.0".to_string(),
        generated_at: chrono::Utc::now().to_rfc3339(),
        index_path: index_path.to_string(),
        verdict: verdict.to_string(),
        issues,
        cells_total: index.cells.len(),
        cells_pass: pass,
        cells_review: review,
        cells_block: block,
        rules_covered: rules.into_iter().collect(),
    }
}

fn render_md(report: &ProofCellVerification) -> String {
    let mut out = String::new();
    out.push_str("# Jankurai ProofCell Verification\n\n");
    out.push_str(&format!("- Verdict: `{}`\n", report.verdict));
    out.push_str(&format!("- Cells: `{}` pass, `{}` review, `{}` block, `{}` total\n", report.cells_pass, report.cells_review, report.cells_block, report.cells_total));
    out.push_str("\n## Rules covered\n\n");
    for rule in &report.rules_covered {
        out.push_str(&format!("- `{rule}`\n"));
    }
    if !report.issues.is_empty() {
        out.push_str("\n## Issues\n\n");
        for issue in &report.issues {
            out.push_str(&format!("- {issue}\n"));
        }
    }
    out
}

fn resolve(repo: &Path, path: &str) -> PathBuf {
    let path = PathBuf::from(path);
    if path.is_absolute() {
        path
    } else {
        repo.join(path)
    }
}
```

```rust
    AlreadyCancelled,

impl OrderError {
    pub fn reason_code(&self) -> &'static str {
        match self {
            Self::Forbidden => "ORDER_FORBIDDEN",
            Self::AlreadyCancelled => "ORDER_ALREADY_CANCELLED",
        }

pub fn cancel_order(role: Role, already_cancelled: bool) -> Result<(), OrderError> {
    if already_cancelled {

    }
    match role {
        Role::Owner | Role::Admin => Ok(()),
        Role::OtherTenant => Err(OrderError::Forbidden),
    }
```

````md
# Jankurai ProofCell

ProofCell turns Rust tests into behavior-linked proof receipts.

Use it when a Rust test is proving durable product truth, authorization, input validation, unsafe-code safety, idempotency, transaction behavior, or a regression that should be visible to agents.

## Minimal example

```rust

proof_cell! {
    id: "orders.cancel.rejects_cross_tenant_user",
    owner: "crates/application",

    rules: ["HLT-008-FALSE-GREEN-RISK", "HLT-022-AUTHZ-ISOLATION-GAP"],

    requires_negative_case: true,
    test: cancel_rejects_cross_tenant_user {
        let result = cancel_order_as_other_tenant();
        jk_negative!("cross_tenant_user_is_denied", result.is_err());
        jk_assert!("stable_reason_code", result.unwrap_err().reason_code() == "ORDER_FORBIDDEN");
    }
}
```

```bash
cargo jankurai-proofcell run \
  --changed-from origin/main \
  --nextest \
  --out target/jankurai/proof-cells/index.json \
  --receipt target/jankurai/proof-receipts/proofcell.json
```

## Decisions

- `pass`: test passed and emitted meaningful proof evidence.
- `review`: proof exists but has over-mocking, snapshots, or other residual risk.
- `block`: command failed, cell failed, zero assertions, missing negative case, or changed path lacks required proof.

## Rules covered

ProofCell primarily supports:

- `HLT-008-FALSE-GREEN-RISK`
- `HLT-018-PERF-CONCURRENCY-DRIFT`
- `HLT-022-AUTHZ-ISOLATION-GAP`
- `HLT-023-INPUT-BOUNDARY-GAP`
- `HLT-027-HUMAN-REVIEW-EVIDENCE-GAP`
````

|---|---|
| No Rust workspace | not applicable |
| Rust workspace, ProofCell disabled/advisory | no hard cap, tool adoption warning only |
| ProofCell required and artifact missing | high finding, cap `false-green-risk` |
| ProofCell present and `overall_decision=block` | high finding, cap `false-green-risk` |
| Authz/input rule has no negative cases | high finding, cap corresponding rule |
| ProofCell present and `review` only | medium finding, no hard cap unless policy says required |
| ProofCell pass with receipt digest in merge witness | improves witness completeness and proof coverage ratio |

### Suggested report section

```md
## ProofCell

- Artifact: `target/jankurai/proof-cells/index.json`
- Decision: `pass`
- Cells: 18 pass, 1 review, 0 block
- Rules covered: HLT-008, HLT-022, HLT-023
- Missing cells for changed paths: none
```

## 21. Security design

ProofCell itself must be boring and least-privilege.

### Safety properties

- **No network access** by default.
- **No mutation outside `target/jankurai/proof-cells/` and requested receipt path.**
- **No product runtime dependency.** It is a dev-dependency.
- **No secret capture.** Assertion expressions are stored, not runtime values.
- **No model judgment.** Decisions are deterministic.
- **No bypass of Jankurai allowlists.** It emits receipts, but `jankurai prove` still enforces proof command allowlists.
- **No auto-pass on malformed events.** Malformed events should block or review.

### Threats and mitigations

| Threat | Mitigation |
|---|---|
| Agent deletes weak test and claims lane passed | changed path missing cell appears in `required_cells_missing`; witness sees missing proof |
| Agent writes smoke test with no assertions | zero assertions block |
| Agent over-mocks external boundary | non-contract mock count routes review/block by policy |
| Agent changes authz and tests only success path | authz rule with no negative case blocks |
| Agent edits JSON artifact by hand | index and receipt digests mismatch under proof verification |
| Agent hides failure with `continue_on_error` | index records command exit code and overall decision remains block |
| Agent leaks secrets in assertion values | macros store expression text only, not debug values |

## 22. Optional phase-2 extensions

These are deliberately not MVP requirements.

### 22.1 Miri adapter

```bash
cargo jankurai-proofcell run --miri --package core-safety
```

Rules:

- unsafe code changes require Miri or documented exception
- any `unsafe` proof cell should include `rules: ["HLT-023-INPUT-BOUNDARY-GAP"]` or a future unsafe-specific rule

### 22.2 Kani adapter

```rust
#[cfg(kani)]
#[kani::proof]
fn cancel_order_never_allows_cross_tenant() {
    // proof harness
}
```

The ProofCell index should include a Kani report digest and classify the cell as `proof_kind = "kani"`.

### 22.3 cargo-semver-checks adapter

Public API cells can require:

```bash
cargo semver-checks check-release
```

ProofCell records this as `proof_kind = "semver"` and maps it to contract/API rules.

### 22.4 cargo-vet adapter

Dependency-sensitive changes should require `cargo vet` or `cargo deny` artifacts, but ProofCell should only point to those receipts. It should not duplicate supply-chain scanners.

## 25. Why not a different crate?

### Not chosen: a replacement test runner

Too large and duplicates `cargo-nextest`.

ProofCell should wrap or enrich existing runners, not compete with them.

### Not chosen: a secret scanner

Jankurai already has a security lane and should orchestrate tools like `gitleaks`, `cargo audit`, `cargo deny`, `zizmor`, SBOM, and provenance. Another scanner is lower leverage.

### Not chosen: a boundary runtime library

A runtime boundary crate could be valuable, but it would force product code changes and compete with frameworks. ProofCell is easier to auto-include and immediately valuable.

### Not chosen: a general AI code reviewer

Jankurai’s standard explicitly favors deterministic, artifact-backed proof over model opinions. ProofCell keeps that philosophy.

## 26. Expected value

### For Jankurai itself

- Makes `HLT-008-FALSE-GREEN-RISK` substantially more enforceable.
- Strengthens merge witness completeness.
- Creates a concrete artifact that bridges tests, owners, rules, and changed paths.
- Adds a showcase crate demonstrating Jankurai’s philosophy in Rust.
- Gives `jankurai init` a high-value automatic Rust dev-tool.

### For any Rust project using Jankurai

- Agents stop saying “tests pass” without naming what behavior was proven.
- Security-sensitive changes require negative proof, not only happy paths.
- Over-mocked tests become visible in CI.
- Proof can be reviewed as JSON/Markdown, not chat history.
- Test metadata survives across agents, commits, and pull requests.

### For open-source maintainers

- Reviews get a compact proof matrix.
- Contributors have a clear way to label regression tests.
- Security fixes can prove denied cases and input boundaries.
- CI artifacts are easier to audit after the fact.
