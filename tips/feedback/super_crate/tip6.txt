# Jankurai-ProofBind: the one Rust crate I would add to Jankurai

**Recommendation:** add one new Rust crate, `jankurai-proofbind`, and have `jankurai init --profile rust-ts-postgres --level full` include it automatically as the repository’s semantic proof router.

**One-line thesis:** Jankurai already knows *which paths changed* and *which proof lanes exist*; ProofBind teaches it *which behavior surfaces changed* and makes irrelevant green tests fail the merge.

**Crate name:** `jankurai-proofbind`  
**Cargo subcommand:** `cargo proofbind`  
**Jankurai subcommand:** `jankurai proofbind`  
**Primary artifact:** `target/jankurai/proofbind/surface-witness.json`  
**Primary policy file:** `agent/proofbind.toml`  
**Primary value:** turns “I ran tests” into “I ran the required proof for the exact API/route/authz/query/tool surface I changed.”

---

## 1. Why this is the #1 best crate for this repo

Jankurai’s current control plane is strong: the README frames the project as a repository conformance standard and local audit CLI for auditable AI-assisted merge, with the rule “no proof, no merge; no receipt, no trust.” The paper companion describes vibe-code drift as ownerless paths, unmapped proof, stale contracts, false-green tests, missing security evidence, overbroad agent permissions, unproven UI changes, and unreceipted claims. The standard requires machine-obvious ownership, generated zones, proof lanes, agent-friendly failures, and evidence receipts.

The key gap is not “more lint.” It is **semantic proof selection**.

Today, the existing Jankurai flow maps changed **paths** to owners and proof lanes through `agent/owner-map.json`, `agent/test-map.json`, `agent/generated-zones.toml`, and `agent/proof-lanes.toml`. That is necessary, but it is still possible for an agent to change a high-risk behavior surface inside a mapped file and run a proof lane that is technically mapped but semantically weak.

Examples:

- A route’s authorization branch changes, but the mapped lane only runs a smoke test.
- A public API signature changes, but no generated-client or SemVer witness runs.
- A `sqlx::query!` mutation changes tenant isolation, but the DB lane only applies migrations.
- A `clap` admin command gains destructive behavior, but no negative test or confirmation proof exists.
- An MCP tool exposes filesystem, process, network, or DB authority, but the only evidence is a passing unit test.
- A function returns `Ok(default)` in an error path; coverage exists, but no mutation/property proof proves the behavior.

`jankurai-proofbind` closes that gap by creating a **Proof-Carrying Surface Manifest**. It scans Rust semantic surfaces, computes stable surface IDs and hashes, classifies risk, and emits proof obligations that Jankurai can attach to proof plans, receipts, findings, context packs, and merge witnesses.

The result is a practical rule:

> If a changed semantic surface has no matching proof receipt, the merge is false green even when tests pass.

That single idea best matches Jankurai’s thesis because it attacks vibe coding at the point where agents most often fool themselves: “tests passed” without proving the changed behavior.

## 2. What I found in the repo that shapes the design

### Existing strengths to preserve

Jankurai already has the right architecture for this crate:

- `crates/jankurai` is the current Rust workspace member.
- `crates/jankurai/src/model.rs` defines report, finding, proof receipt, evidence index, manifest fingerprint, and agent-fix data models.
- `crates/jankurai/src/commands/proof.rs` builds proof plans, executes planned proof runs, writes receipts, and verifies evidence indexes.
- `crates/jankurai/src/commands/context_data.rs` loads `agent/owner-map.json`, `agent/test-map.json`, `agent/generated-zones.toml`, and `agent/proof-lanes.toml` into `RepoCatalog`.
- `RepoCatalog` currently routes by path prefix or exact path, which is simple and deterministic.
- `crates/jankurai/src/audit/rules.rs` already has stable rule IDs for false-green tests, authorization isolation, input-boundary gaps, agent tool supply gaps, destructive migrations, supply-chain drift, and unreceipted proof claims.
- `crates/jankurai/src/validation.rs` already validates artifacts through repo-local JSON schemas.
- `crates/jankurai/src/init/templates.rs` already scaffolds `Justfile`, `agent/proof-lanes.toml`, `agent/tool-adoption.toml`, CI, and other control-plane files.

ProofBind should therefore be a **small additive crate**, not a rewrite.

### Existing gap to target

The current proof planner answers:

> “Given a changed path, which owner/test-map/proof lane applies?”

ProofBind answers:

> “Given a changed symbol, route, migration, query, CLI command, or agent tool, which exact proof obligation must be satisfied?”

The latter is the missing semantic layer.

## 3. Ecosystem scan: why this is not redundant

The Rust ecosystem has excellent point tools, and ProofBind should orchestrate them rather than compete with them.

| Tool / standard | What it already does well | What it does not do for Jankurai |
|---|---|---|
| `cargo-vet` | Audits third-party dependencies against trusted audit criteria and highlights gaps. | Does not bind first-party behavior surfaces to proof lanes or receipts. |
| `cargo-deny` | Checks licenses, bans, advisories, duplicate crates, and sources in the crate graph. | Does not know whether a changed authz function has negative tests. |
| `cargo-nextest` | Fast, isolated Rust test runner with CI support, filtering, retries, recording, and JUnit export. | Runs tests; does not decide which semantic behavior must be proven. |
| `cargo-mutants` | Mutation testing finds code locations where bugs can be inserted without tests failing. | Great proof backend, but not path/surface/risk aware inside Jankurai. |
| `cargo-fuzz` | Recommended Rust fuzzing tool using libFuzzer. | Great proof backend, but does not know which changed external input surfaces require fuzzing. |
| `cargo-semver-checks` / public API tooling | Detects Rust public API compatibility breaks. | Public API proof is only one surface type; it does not cover authz, DB, tools, CLI, or business invariants. |
| SQLx checked queries | Can check SQL syntax and result types at compile time. | Does not prove tenant isolation, authorization semantics, destructive migration safety, or rollback policy. |
| OpenSSF Scorecard / SLSA / NIST SSDF | Strong supply-chain and secure-development framing. | They are not repo-local semantic proof routers for AI-agent merges. |
| MCP tool annotations | Provide tool behavior hints such as read-only, destructive, idempotent, and open-world. | The MCP spec says annotations are hints and untrusted unless from trusted servers; Jankurai needs independent evidence. |
| OWASP GenAI / LLM security guidance | Names risks such as prompt injection, supply chain, excessive agency, and insecure output handling. | Does not generate Rust workspace proof obligations or receipts. |

ProofBind’s role is therefore to be the **Jankurai-native semantic binding layer** above these tools. It selects and verifies proof obligations; it does not replace the proof engines.

## 5. What counts as a “surface”

A surface is any code or artifact boundary where a local edit can change externally observable behavior, security posture, or durable truth.

Initial MVP surface kinds:

| Surface kind | Examples | Default Jankurai rule mapping |
|---|---|---|
| `rust_public_api` | `pub fn`, `pub struct`, `pub enum`, exported traits | `HLT-007-HANDWRITTEN-CONTRACT`, `HLT-008-FALSE-GREEN-RISK` |
| `http_route` | `axum::Router::route`, handler functions | `HLT-022-AUTHZ-ISOLATION-GAP`, `HLT-023-INPUT-BOUNDARY-GAP` |
| `cli_command` | `clap::Parser`, `Subcommand`, command handlers | `HLT-023-INPUT-BOUNDARY-GAP`, `HLT-024-AGENT-TOOL-SUPPLY-GAP` when commands are agent-invoked |
| `sql_query` | `sqlx::query!`, `query_as!`, raw SQL strings | `HLT-006-DIRECT-DB-WRONG-LAYER`, `HLT-022-AUTHZ-ISOLATION-GAP` |
| `db_migration` | `db/migrations/*.sql` | `HLT-021-DESTRUCTIVE-MIGRATION` |
| `authz_boundary` | functions/types containing authz, authorize, policy, tenant, org, role, permission | `HLT-022-AUTHZ-ISOLATION-GAP` |
| `input_boundary` | deserialization, JSON, env, filesystem, network, process args, tool input schemas | `HLT-023-INPUT-BOUNDARY-GAP` |
| `mcp_tool` | MCP `Tool`, tool registry, `tools/list`, `tools/call`, annotations | `HLT-024-AGENT-TOOL-SUPPLY-GAP` |
| `unsafe_or_process_sink` | `unsafe`, `std::process::Command`, filesystem writes, network calls | `HLT-023-INPUT-BOUNDARY-GAP`, `HLT-016-SUPPLY-CHAIN-DRIFT` |
| `business_invariant` | pure decision functions, state transitions, pricing, workflow gates | `HLT-008-FALSE-GREEN-RISK` |

The surface manifest turns code into a machine-readable map:

```json
{
  "id": "surface:authz_boundary:crates/application/src/refunds.rs:AuthorizeRefund",
  "kind": "authz_boundary",
  "path": "crates/application/src/refunds.rs",
  "line": 41,
  "symbol": "AuthorizeRefund::check",
  "owner": "application",
  "risk": "critical",
  "lanes": ["fast", "db", "security"],
  "rules": ["HLT-022-AUTHZ-ISOLATION-GAP", "HLT-008-FALSE-GREEN-RISK"],
  "required_proofs": [
    "negative authorization matrix test",
    "tenant isolation regression test",
    "mutation proof for decision branch"
  ],
  "semantic_hash": "sha256:..."
}
```

## 6. User experience

### New repo adoption

```bash
jankurai init --profile rust-ts-postgres --level full --yes
just proofbind
```

The full Rust profile adds:

```toml
# agent/proofbind.toml
schema_version = "1.0.0"
mode = "standard"
fail_on_unbound_changed_surface = true
fail_on_missing_semantic_receipt = true
```

and this lane:

```toml
[[lane]]
name = "proofbind"
command = "jankurai proofbind verify . --changed-from origin/main --witness target/jankurai/proofbind/surface-witness.json --obligations target/jankurai/proofbind/obligations.json --md target/jankurai/proofbind/proofbind.md"
purpose = "Bind changed Rust semantic surfaces to required proof obligations."
kind = "semantic-proof"
rules_covered = ["HLT-008-FALSE-GREEN-RISK", "HLT-021-DESTRUCTIVE-MIGRATION", "HLT-022-AUTHZ-ISOLATION-GAP", "HLT-023-INPUT-BOUNDARY-GAP", "HLT-024-AGENT-TOOL-SUPPLY-GAP", "HLT-027-HUMAN-REVIEW-EVIDENCE-GAP"]
required_artifacts = ["target/jankurai/proofbind/surface-witness.json", "target/jankurai/proofbind/obligations.json", "target/jankurai/proofbind/proofbind.md"]
timeout_seconds = 60
requires_network = false
destructive = false
```

### Local loop for agents

```bash
# See what changed semantically.
jankurai proofbind map . --changed-from origin/main --md target/jankurai/proofbind/surface-witness.md

# Fail if any changed surface lacks the required proof evidence.
jankurai proofbind verify . --changed-from origin/main

# Then run only the required lanes.
jankurai proof --changed-from origin/main --out target/jankurai/proof-plan.json --md target/jankurai/proof-plan.md
jankurai prove --plan target/jankurai/proof-plan.json
```

### Output agents actually need

```md
## Changed high-risk surfaces

1. `AuthorizeRefund::check` in `crates/application/src/refunds.rs:41`
   - Risk: critical
   - Rules: HLT-022, HLT-008
   - Required proof:
     - `cargo nextest run -E 'test(authz_refund_matrix)'`
     - `cargo mutants --file crates/application/src/refunds.rs --in-place --timeout 30`
   - Missing receipt: negative authz matrix

2. `admin_wipe_user` in `crates/admin/src/cli.rs:88`
   - Risk: critical
   - Rules: HLT-023, HLT-024
   - Required proof:
     - CLI destructive confirmation test
     - no agent invocation without explicit human approval
```

This changes agent behavior. Instead of “run tests,” the agent gets a proof checklist tied to exact symbols and risk.

## 8. Policy file: `agent/proofbind.toml`

```toml
schema_version = "1.0.0"
mode = "standard"

# In advisory mode this emits findings only. In standard mode it fails high/critical gaps.
fail_on_unbound_changed_surface = true
fail_on_missing_semantic_receipt = true
fail_on_missing_owner = true
fail_on_missing_test_route = true

# Keep defaults conservative. Projects can lower risk only with reviewed exceptions.
default_external_input_risk = "high"
default_authz_risk = "critical"
default_mcp_tool_risk = "critical"
default_destructive_migration_risk = "critical"

# These strings are intentionally simple and explainable. No ML classifier is required.
authz_terms = ["authz", "authorize", "authorization", "permission", "policy", "tenant", "org", "role", "rbac", "acl"]
input_terms = ["deserialize", "parse", "from_json", "serde_json", "request", "payload", "args", "env", "stdin", "file", "path", "url"]
destructive_sql_terms = ["drop table", "drop column", "truncate", "delete from", "alter table", "rename column", "rename table"]
agent_tool_terms = ["mcp", "tool", "tools/list", "tools/call", "command", "browser", "terminal", "filesystem"]

[lanes]
fast = "cargo nextest run --workspace"
contract = "cargo semver-checks --workspace || cargo public-api diff latest"
db = "cargo test --workspace db_ -- --nocapture"
security = "cargo deny check && cargo audit"
mutation = "cargo mutants --no-shuffle --timeout 30"
fuzz = "cargo fuzz run proofbind_input -- -max_total_time=30"
audit = "jankurai audit . --mode standard --json agent/repo-score.json --md agent/repo-score.md"

[[surface_rule]]
kind = "authz_boundary"
required_rules = ["HLT-022-AUTHZ-ISOLATION-GAP", "HLT-008-FALSE-GREEN-RISK"]
required_lanes = ["fast", "db", "mutation"]
required_test_patterns = ["authz", "tenant", "negative", "denied", "role"]

[[surface_rule]]
kind = "input_boundary"
required_rules = ["HLT-023-INPUT-BOUNDARY-GAP"]
required_lanes = ["fast", "fuzz"]
required_test_patterns = ["invalid", "malformed", "reject", "boundary"]

[[surface_rule]]
kind = "mcp_tool"
required_rules = ["HLT-024-AGENT-TOOL-SUPPLY-GAP", "HLT-023-INPUT-BOUNDARY-GAP"]
required_lanes = ["security", "fast"]
required_test_patterns = ["tool", "permission", "confirmation", "deny", "schema"]

[[surface_rule]]
kind = "db_migration"
required_rules = ["HLT-021-DESTRUCTIVE-MIGRATION"]
required_lanes = ["db"]

[[surface_rule]]
kind = "rust_public_api"
required_rules = ["HLT-007-HANDWRITTEN-CONTRACT", "HLT-008-FALSE-GREEN-RISK"]
required_lanes = ["contract", "fast"]
required_test_patterns = ["api", "compat", "contract"]
```

### 10.3 `crates/jankurai-proofbind/src/model.rs`

```rust
use serde::{Deserialize, Serialize};

pub const SCHEMA_VERSION: &str = "1.0.0";

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq)]
#[serde(rename_all = "snake_case")]
pub enum SurfaceKind {
    RustPublicApi,
    HttpRoute,
    CliCommand,
    SqlQuery,
    DbMigration,
    AuthzBoundary,
    InputBoundary,
    McpTool,
    UnsafeOrProcessSink,
    BusinessInvariant,
}

impl SurfaceKind {
    pub fn as_str(&self) -> &'static str {
        match self {
            Self::RustPublicApi => "rust_public_api",
            Self::HttpRoute => "http_route",
            Self::CliCommand => "cli_command",
            Self::SqlQuery => "sql_query",
            Self::DbMigration => "db_migration",
            Self::AuthzBoundary => "authz_boundary",
            Self::InputBoundary => "input_boundary",
            Self::McpTool => "mcp_tool",
            Self::UnsafeOrProcessSink => "unsafe_or_process_sink",
            Self::BusinessInvariant => "business_invariant",
        }
    }
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, PartialOrd, Ord)]
#[serde(rename_all = "snake_case")]
pub enum RiskLevel {
    Low,
    Medium,
    High,
    Critical,
}

impl RiskLevel {
    pub fn as_str(&self) -> &'static str {
        match self {
            Self::Low => "low",
            Self::Medium => "medium",
            Self::High => "high",
            Self::Critical => "critical",
        }
    }
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Surface {
    pub id: String,
    pub kind: String,
    pub path: String,
    pub line: usize,
    pub symbol: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub owner: Option<String>,
    pub semantic_hash: String,
    pub risk: String,
    pub rules: Vec<String>,
    pub lanes: Vec<String>,
    pub required_proofs: Vec<String>,
    pub changed: bool,
    pub evidence: Vec<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct WitnessSummary {
    pub surface_count: usize,
    pub changed_surface_count: usize,
    pub high_risk_count: usize,
    pub critical_risk_count: usize,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SurfaceWitness {
    pub schema_version: String,
    pub generated_at: String,
    pub repo_root: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub git_head: Option<String>,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub base_ref: Option<String>,
    pub changed_paths: Vec<String>,
    pub summary: WitnessSummary,
    pub surfaces: Vec<Surface>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProofObligation {
    pub surface_id: String,
    pub rule_id: String,
```

```rust
pub fn rust_files(repo: &Path) -> Result<Vec<PathBuf>> {
    let ignores = ignore_set()?;
    let mut out = Vec::new();
    for entry in WalkDir::new(repo).follow_links(false) {
        let entry = entry?;
        let path = entry.path();
        let rel = path.strip_prefix(repo).unwrap_or(path);
        if ignores.is_match(rel) {
            continue;
        }
        if path.is_file() && path.extension().and_then(|s| s.to_str()) == Some("rs") {
            out.push(path.to_path_buf());
        }
    }
    out.sort();
    Ok(out)
}

pub fn sql_migrations(repo: &Path) -> Result<Vec<PathBuf>> {
    let mut out = Vec::new();
    for dir in ["db/migrations", "migrations"] {
        let root = repo.join(dir);
        if !root.exists() {
            continue;
        }
        for entry in WalkDir::new(root).follow_links(false) {
            let entry = entry?;
            let path = entry.path();
            if path.is_file() && path.extension().and_then(|s| s.to_str()) == Some("sql") {
                out.push(path.to_path_buf());
            }
        }
    }
    out.sort();
    Ok(out)
}

pub fn rel(repo: &Path, path: &Path) -> String {
    path.strip_prefix(repo)
        .unwrap_or(path)
        .to_string_lossy()
        .replace('\\', "/")
```

```rust
use anyhow::Result;

pub fn git_head(repo: &Path) -> Option<String> {

pub fn changed_paths(repo: &Path, changed_from: Option<&str>, explicit: &[String]) -> Result<Vec<String>> {
    if !explicit.is_empty() {
        return Ok(normalize(explicit.to_vec()));
    }
    let Some(base) = changed_from else {
        return Ok(Vec::new());
    };
    let Some(output) = git_output(repo, &["diff", "--name-only", base, "--"]) else {
        return Ok(Vec::new());
    };
    Ok(normalize(output.lines().map(str::to_string).collect()))
}

fn normalize(paths: Vec<String>) -> Vec<String> {
    let mut out = paths
        .into_iter()
        .map(|p| p.trim().trim_start_matches("./").replace('\\', "/"))
        .filter(|p| !p.is_empty())
        .collect::<Vec<_>>();
    out.sort();
    out.dedup();
    out
}

fn git_output(repo: &Path, args: &[&str]) -> Option<String> {
    Command::new("git")
        .args(args)
        .current_dir(repo)
        .output()
        .ok()
        .filter(|out| out.status.success())
        .and_then(|out| String::from_utf8(out.stdout).ok())
        .map(|s| s.trim().to_string())
        .filter(|s| !s.is_empty())
}
```

### 10.7 `crates/jankurai-proofbind/src/rust_scan.rs`

```rust
use crate::config::ProofbindConfig;
use crate::fs;
use crate::model::{RiskLevel, Surface, SurfaceKind};
use anyhow::{Context, Result};
use sha2::{Digest, Sha256};
use std::fs as stdfs;
use std::path::Path;
use syn::visit::{self, Visit};
use syn::{Attribute, Expr, ExprCall, ExprMacro, File, Item, ItemEnum, ItemFn, ItemStruct, ItemTrait, Macro, Visibility};

pub fn scan_rust_file(repo: &Path, path: &Path, cfg: &ProofbindConfig, changed_paths: &[String]) -> Result<Vec<Surface>> {
    let text = stdfs::read_to_string(path).with_context(|| format!("read {}", path.display()))?;
    let ast: File = syn::parse_file(&text).with_context(|| format!("parse Rust AST {}", path.display()))?;
    let rel = fs::rel(repo, path);
    let changed = changed_paths.iter().any(|p| p == &rel || rel.starts_with(&format!("{p}/")));
    let mut visitor = SurfaceVisitor {
        repo,
        rel: rel.clone(),
        source: text.clone(),
        cfg,
        changed,
        surfaces: Vec::new(),
    };
    visitor.visit_file(&ast);
    Ok(visitor.surfaces)
}

struct SurfaceVisitor<'a> {
    repo: &'a Path,
    rel: String,
    source: String,
    cfg: &'a ProofbindConfig,
    changed: bool,
    surfaces: Vec<Surface>,
}

impl<'ast, 'a> Visit<'ast> for SurfaceVisitor<'a> {
    fn visit_item_fn(&mut self, node: &'ast ItemFn) {
        let name = node.sig.ident.to_string();
        let vis_pub = matches!(node.vis, Visibility::Public(_));
        let lower = name.to_lowercase();

        if vis_pub {
            self.push_surface(SurfaceKind::RustPublicApi, &name, "public Rust function", RiskLevel::Medium);
        }
        if contains_any(&lower, &self.cfg.authz_terms) {
            self.push_surface(SurfaceKind::AuthzBoundary, &name, "authorization-looking function name", RiskLevel::Critical);
        }
        if contains_any(&lower, &self.cfg.input_terms) {
            self.push_surface(SurfaceKind::InputBoundary, &name, "input-looking function name", RiskLevel::High);
        }
        if looks_like_business_invariant(&lower) {
            self.push_surface(SurfaceKind::BusinessInvariant, &name, "business invariant or state transition", RiskLevel::High);
        }
        if has_test_attr(&node.attrs) {
            // Tests are evidence, not product surfaces.
        }
        visit::visit_item_fn(self, node);
    }

    fn visit_item_struct(&mut self, node: &'ast ItemStruct) {
        let name = node.ident.to_string();
        if matches!(node.vis, Visibility::Public(_)) {
            self.push_surface(SurfaceKind::RustPublicApi, &name, "public Rust struct", RiskLevel::Medium);
        }
        if has_attr_containing(&node.attrs, "Parser") || has_attr_containing(&node.attrs, "Args") || has_attr_containing(&node.attrs, "Subcommand") {
            self.push_surface(SurfaceKind::CliCommand, &name, "clap command/args surface", RiskLevel::High);
        }
        if name.to_lowercase().contains("tool") || name.to_lowercase().contains("mcp") {
            self.push_surface(SurfaceKind::McpTool, &name, "MCP/tool-like struct", RiskLevel::Critical);
        }
        visit::visit_item_struct(self, node);
    }

    fn visit_item_enum(&mut self, node: &'ast ItemEnum) {
        let name = node.ident.to_string();
        if matches!(node.vis, Visibility::Public(_)) {
            self.push_surface(SurfaceKind::RustPublicApi, &name, "public Rust enum", RiskLevel::Medium);
        }
        if has_attr_containing(&node.attrs, "Subcommand") || name.to_lowercase().contains("command") {
            self.push_surface(SurfaceKind::CliCommand, &name, "clap subcommand surface", RiskLevel::High);
        }
        visit::visit_item_enum(self, node);
    }

    fn visit_item_trait(&mut self, node: &'ast ItemTrait) {
        let name = node.ident.to_string();
        if matches!(node.vis, Visibility::Public(_)) {
            self.push_surface(SurfaceKind::RustPublicApi, &name, "public Rust trait", RiskLevel::Medium);
        }
        visit::visit_item_trait(self, node);
    }

    fn visit_macro(&mut self, node: &'ast Macro) {
        let path = node.path.segments.iter().map(|s| s.ident.to_string()).collect::<Vec<_>>().join("::");
        let tokens = node.tokens.to_string();
        let lower = format!("{} {}", path.to_lowercase(), tokens.to_lowercase());

        if lower.contains("sqlx") && lower.contains("query") {
            self.push_surface(SurfaceKind::SqlQuery, &path, "sqlx query macro", RiskLevel::High);
        }
        if lower.contains("route") || lower.contains("router") || lower.contains("axum") {
            self.push_surface(SurfaceKind::HttpRoute, &path, "route-like macro invocation", RiskLevel::High);
        }
        if lower.contains("tool") || lower.contains("tools/call") || lower.contains("mcp") {
            self.push_surface(SurfaceKind::McpTool, &path, "MCP/tool macro invocation", RiskLevel::Critical);
        }
        visit::visit_macro(self, node);
    }

    fn visit_expr_macro(&mut self, node: &'ast ExprMacro) {
        self.visit_macro(&node.mac);
        visit::visit_expr_macro(self, node);
    }

    fn visit_expr_call(&mut self, node: &'ast ExprCall) {
        let text = format!("{:?}", node.func);
        let lower = text.to_lowercase();
        if lower.contains("std :: process :: command") || lower.contains("command :: new") {
            self.push_surface(SurfaceKind::UnsafeOrProcessSink, "std::process::Command", "process execution sink", RiskLevel::Critical);
        }
        visit::visit_expr_call(self, node);
    }

    fn visit_expr(&mut self, node: &'ast Expr) {
        let text = format!("{:?}", node);
        let lower = text.to_lowercase();
        if lower.contains("unsafe") || lower.contains("unwrap") || lower.contains("expect") || lower.contains("panic !") {
            self.push_surface(SurfaceKind::UnsafeOrProcessSink, "unsafe-or-panic", "unsafe/panic/unwrap sink", RiskLevel::High);
        }
        if lower.contains("serde_json :: value") || lower.contains("from_str") || lower.contains("env :: var") {
            self.push_surface(SurfaceKind::InputBoundary, "dynamic-input", "dynamic input boundary", RiskLevel::High);
        }
        visit::visit_expr(self, node);
    }
}

impl<'a> SurfaceVisitor<'a> {
    fn push_surface(&mut self, kind: SurfaceKind, symbol: &str, evidence: &str, risk: RiskLevel) {
        let semantic = format!("{}|{}|{}|{}", kind.as_str(), self.rel, symbol, evidence);
        let hash = format!("sha256:{:x}", Sha256::digest(semantic.as_bytes()));
        let id = format!("surface:{}:{}:{}", kind.as_str(), self.rel, sanitize_symbol(symbol));
        let (rules, lanes, required_proofs) = defaults_for_kind(&kind);
        if self.surfaces.iter().any(|s| s.id == id) {
            return;
        }
        self.surfaces.push(Surface {
            id,
            kind: kind.as_str().into(),
            path: self.rel.clone(),
            line: 0,
            symbol: symbol.into(),
            owner: None,
            semantic_hash: hash,
            risk: risk.as_str().into(),
            rules,
            lanes,
            required_proofs,
            changed: self.changed,
            evidence: vec![evidence.into()],
        });
    }
}
```

```rust
use crate::model::{ObligationsReport, ProofObligation, SurfaceWitness, SCHEMA_VERSION};

pub fn build_obligations(repo: &Path, cfg: &ProofbindConfig, witness: &SurfaceWitness) -> ObligationsReport {

            let lane = lane_for_rule(rule_id, surface.lanes.first().map(String::as_str).unwrap_or("fast"));
            let command = cfg.lanes.get(&lane).cloned();
            let required_test_patterns = required_patterns_for_surface(cfg, &surface.kind);
            let required_artifacts = vec![
                "target/jankurai/proof-receipts/*.json".into(),
                "target/jankurai/logs/*.log".into(),
            ];
            let status = if has_matching_receipt(repo, rule_id, &lane) { "satisfied" } else { "missing" };
            if status == "missing" && cfg.fail_on_missing_semantic_receipt {
                issues.push(format!(
                    "surface `{}` requires `{}` in lane `{}` but no matching receipt was found",
                    surface.id, rule_id, lane
                ));
            }
            obligations.push(ProofObligation {
                surface_id: surface.id.clone(),
                rule_id: rule_id.clone(),
                severity: surface.risk.clone(),
                lane,
                command,
                required_test_patterns,
                required_artifacts,
                status: status.into(),
                reason: format!("changed {} surface `{}`", surface.kind, surface.symbol),
            });
        }
    }

    let verdict = if issues.iter().any(|issue| issue.contains("critical") || issue.contains("requires")) {
        "fail"
    } else if !issues.is_empty() {
        "warn"
    } else {
        "pass"
    };

    ObligationsReport {
        schema_version: SCHEMA_VERSION.into(),
        generated_at: Utc::now().to_rfc3339(),
        repo_root: repo.display().to_string(),
        verdict: verdict.into(),
        obligations,
        issues,
    }
}

fn lane_for_rule(rule_id: &str, fallback: &str) -> String {
    match rule_id {
        "HLT-021-DESTRUCTIVE-MIGRATION" => "db".into(),
        "HLT-022-AUTHZ-ISOLATION-GAP" => "db".into(),
        "HLT-023-INPUT-BOUNDARY-GAP" => "security".into(),
        "HLT-024-AGENT-TOOL-SUPPLY-GAP" => "security".into(),
        "HLT-007-HANDWRITTEN-CONTRACT" => "contract".into(),
        "HLT-008-FALSE-GREEN-RISK" => "mutation".into(),
        _ => fallback.into(),
    }
}

fn required_patterns_for_surface(cfg: &ProofbindConfig, kind: &str) -> Vec<String> {
    cfg.surface_rule
        .iter()
        .find(|rule| rule.kind == kind)
        .map(|rule| rule.required_test_patterns.clone())
        .unwrap_or_default()
}

fn has_matching_receipt(repo: &Path, rule_id: &str, lane: &str) -> bool {
    let dir = repo.join("target/jankurai/proof-receipts");
    let Ok(entries) = fs::read_dir(dir) else {
        return false;
    };
    for entry in entries.flatten() {
        let Ok(text) = fs::read_to_string(entry.path()) else {
            continue;
        };
        if text.contains(rule_id) && text.contains(&format!("\"lane\": \"{}\"", lane)) && text.contains("\"exit_code\": 0") {
            return true;
        }
    }
    false
}
```

### 10.10 `crates/jankurai-proofbind/src/lib.rs`

```rust
pub mod config;
pub mod fs;
pub mod git;
pub mod model;
pub mod obligations;
pub mod render;
pub mod rust_scan;

use anyhow::{Context, Result};
use chrono::Utc;
use config::ProofbindConfig;
use model::{Surface, SurfaceWitness, WitnessSummary, SCHEMA_VERSION};
use sha2::{Digest, Sha256};
use std::collections::BTreeMap;
use std::fs as stdfs;
use std::path::{Path, PathBuf};

#[derive(Debug, Clone)]
pub struct MapArgs {
    pub repo: PathBuf,
    pub changed_from: Option<String>,
    pub changed: Vec<String>,
    pub out: PathBuf,
    pub md: Option<PathBuf>,
}

#[derive(Debug, Clone)]
pub struct VerifyArgs {
    pub repo: PathBuf,
    pub changed_from: Option<String>,
    pub changed: Vec<String>,
    pub witness: PathBuf,
    pub obligations: PathBuf,
    pub md: Option<PathBuf>,
}

pub fn map(args: MapArgs) -> Result<SurfaceWitness> {
    let repo = args.repo.canonicalize().unwrap_or(args.repo);
    let cfg = ProofbindConfig::load(&repo)?;
    let changed_paths = git::changed_paths(&repo, args.changed_from.as_deref(), &args.changed)?;
    let mut surfaces = Vec::new();

    for path in fs::rust_files(&repo)? {
        surfaces.extend(rust_scan::scan_rust_file(&repo, &path, &cfg, &changed_paths)?);
    }
    for path in fs::sql_migrations(&repo)? {
        if let Some(surface) = rust_scan::scan_sql_migration(&repo, &path, &cfg, &changed_paths)? {
            surfaces.push(surface);
        }
    }

    attach_owner_hints(&repo, &mut surfaces);
    dedup_surfaces(&mut surfaces);
    let summary = summarize(&surfaces);
    let witness = SurfaceWitness {
        schema_version: SCHEMA_VERSION.into(),
        generated_at: Utc::now().to_rfc3339(),
        repo_root: repo.display().to_string(),
        git_head: git::git_head(&repo),
        base_ref: args.changed_from,
        changed_paths,
        summary,
        surfaces,
    };

    write_json(&args.out, &witness)?;
    if let Some(md) = args.md {
        write_text(&md, &render::witness_markdown(&witness))?;
    }
    Ok(witness)
}

pub fn verify(args: VerifyArgs) -> Result<model::ObligationsReport> {
    let repo = args.repo.canonicalize().unwrap_or(args.repo);
    let witness = if args.witness.is_file() {
        let text = stdfs::read_to_string(&args.witness).with_context(|| format!("read {}", args.witness.display()))?;
        serde_json::from_str::<SurfaceWitness>(&text).with_context(|| format!("parse {}", args.witness.display()))?
    } else {
        map(MapArgs {
            repo: repo.clone(),
            changed_from: args.changed_from.clone(),
            changed: args.changed.clone(),
            out: args.witness.clone(),
            md: None,
        })?
    };
    let cfg = ProofbindConfig::load(&repo)?;
    let report = obligations::build_obligations(&repo, &cfg, &witness);
    write_json(&args.obligations, &report)?;
    if let Some(md) = args.md {
        write_text(&md, &render::obligations_markdown(&report))?;
    }
    if report.verdict == "fail" && cfg.mode != "advisory" {
        anyhow::bail!("proofbind verification failed with {} issue(s)", report.issues.len());
    }
    Ok(report)
}

fn summarize(surfaces: &[Surface]) -> WitnessSummary {
    WitnessSummary {
        surface_count: surfaces.len(),
        changed_surface_count: surfaces.iter().filter(|s| s.changed).count(),
        high_risk_count: surfaces.iter().filter(|s| s.risk == "high").count(),
        critical_risk_count: surfaces.iter().filter(|s| s.risk == "critical").count(),
    }
}

fn dedup_surfaces(surfaces: &mut Vec<Surface>) {
    let mut seen = BTreeMap::new();
    for surface in std::mem::take(surfaces) {
        seen.entry(surface.id.clone()).or_insert(surface);
    }
    *surfaces = seen.into_values().collect();
}

fn attach_owner_hints(repo: &Path, surfaces: &mut [Surface]) {
    let owner_map_path = repo.join("agent/owner-map.json");
    let Ok(text) = stdfs::read_to_string(owner_map_path) else {
        return;
    };
    let Ok(value) = serde_json::from_str::<serde_json::Value>(&text) else {
        return;
    };
    let Some(owners) = value.get("owners").and_then(|v| v.as_object()) else {
        return;
    };
    for surface in surfaces {
        surface.owner = owners
            .iter()
            .filter(|(prefix, _)| surface.path == **prefix || surface.path.starts_with(&format!("{}/", prefix.trim_matches('/'))))
            .max_by_key(|(prefix, _)| prefix.matches('/').count())
            .and_then(|(_, owner)| owner.as_str().map(str::to_string));
    }
}

fn write_json<T: serde::Serialize>(path: &Path, value: &T) -> Result<()> {
    if let Some(parent) = path.parent() {
        stdfs::create_dir_all(parent)?;
    }
    let text = serde_json::to_string_pretty(value)?;
    stdfs::write(path, text)?;
    Ok(())
}

fn write_text(path: &Path, text: &str) -> Result<()> {
    if let Some(parent) = path.parent() {
        stdfs::create_dir_all(parent)?;
    }
    stdfs::write(path, text)?;
    Ok(())
}

pub fn content_sha256(text: &str) -> String {
    format!("sha256:{:x}", Sha256::digest(text.as_bytes()))
}
```

```rust
        md: Option<PathBuf>,
    },
}

fn main() -> Result<()> {
    let cli = Cli::parse();
    let command = match cli.command {
```

An agent says:

> I changed `refunds.rs`. I ran `cargo test`. All good.

But that does not answer:

- Did the tests cover denied roles?
- Did the tests cover cross-tenant access?
- Did the public API or generated client change?
- Did a mutation survive?
- Did the proof receipt bind to the changed behavior?

### After

The agent says:

> I changed `surface:authz_boundary:crates/application/src/refunds.rs:AuthorizeRefund`. ProofBind classified it as critical. It required `HLT-022` and `HLT-008`. I ran the authz matrix test and mutation lane. Receipts are `target/jankurai/proof-receipts/003-db-authz.json` and `004-mutation-refunds.json`.

That is agent-first code design: stable IDs, exact ownership, exact proof, exact receipt.

## 12. How ProofBind improves testing

ProofBind changes test selection from path-based to behavior-based.

| Changed surface | Required proof upgrade |
|---|---|
| `authz_boundary` | Negative authorization matrix + tenant isolation + mutation proof |
| `input_boundary` | Invalid/malformed input tests + fuzz target |
| `rust_public_api` | API compatibility / generated client / contract proof |
| `business_invariant` | Table/property tests + mutation proof |
| `db_migration` | Forward apply + rollback/backfill + lock safety |
| `mcp_tool` | Tool schema validation + permission/human confirmation proof |
| `cli_command` | Invalid args + destructive confirmation + no unintended agent execution |

The important shift is not “more tests.” It is **fewer irrelevant tests and more mandatory semantic tests**.

---

## 13. How ProofBind improves security

ProofBind directly addresses high-risk surfaces in the Jankurai ruleset:

- `HLT-021-DESTRUCTIVE-MIGRATION`: destructive SQL terms require DB safety evidence.
- `HLT-022-AUTHZ-ISOLATION-GAP`: authz/tenant/role surfaces require negative tests.
- `HLT-023-INPUT-BOUNDARY-GAP`: external inputs, dynamic JSON, env, process args, file paths, and URL-like boundaries require malformed-input proof.
- `HLT-024-AGENT-TOOL-SUPPLY-GAP`: agent/MCP/tool surfaces require permissions, schema, and confirmation evidence.
- `HLT-027-HUMAN-REVIEW-EVIDENCE-GAP`: review claims must attach reproducible receipts.

A crucial security design choice: ProofBind never trusts self-reported MCP annotations by themselves. MCP annotations are still useful metadata, but ProofBind independently classifies tool risk from code and requires proof receipts for destructive/open-world/high-authority tools.

---

## 14. How it improves popular standard surfaces

This crate makes common Rust surface crates safer without replacing them:

- **`clap`**: command surfaces become proof-bound. Destructive admin commands require confirmation and invalid-args tests.
- **`serde` / `serde_json`**: dynamic input surfaces require malformed-input and schema/validation proof.
- **`axum` / routing code**: route handlers become authz/input surfaces rather than anonymous functions in a file.
- **`sqlx`**: checked queries remain valuable, but ProofBind adds business/security obligations around tenant isolation and destructive migration safety.
- **MCP Rust servers**: tool names, schemas, annotations, and call handlers become security surfaces requiring tool-authority receipts.

That is a better target than inventing yet another linter: it strengthens the exact crates and patterns agents already touch.

## 17. Definition of done

The feature is done when these commands pass in the Jankurai repo:

```bash
cargo test -p jankurai-proofbind
cargo test -p jankurai proofbind_cli_smoke
jankurai proofbind map . --changed crates/jankurai/src/commands/proof.rs --md target/jankurai/proofbind/surface-witness.md
jankurai proofbind verify . --changed crates/jankurai/src/commands/proof.rs --md target/jankurai/proofbind/proofbind.md || true
jankurai audit . --mode advisory --json target/jankurai/repo-score.json --md target/jankurai/repo-score.md
```

And these fixture cases are covered:

1. A public Rust API change emits a `rust_public_api` obligation.
2. A function named `authorize_*` emits `HLT-022` and `HLT-008` obligations.
3. A `serde_json::from_str(...).unwrap()` boundary emits `HLT-023`.
4. A destructive SQL migration emits `HLT-021`.
5. An MCP tool surface emits `HLT-024` and requires security evidence.
6. A matching proof receipt changes an obligation from `missing` to `satisfied`.
7. Advisory mode reports but does not fail.
8. Standard mode fails high/critical missing obligations.

## 18. Why this beats alternative crate ideas

### Better than “Jankurai security scanner”

Jankurai already has security lanes and security evidence. A new scanner would duplicate cargo-deny, cargo-audit, RustSec, gitleaks, Scorecard, and SAST tools. ProofBind instead decides when those tools are mandatory and binds them to exact changed surfaces.

### Better than “Jankurai test generator”

Agents can generate tests, but generated tests can be shallow. ProofBind does not claim to write the perfect test. It tells the agent what behavior class must be proven and rejects unreceipted claims.

### Better than “Jankurai MCP guard”

MCP/tool security is important, but Jankurai’s scope is broader than MCP. ProofBind includes MCP tools as one surface kind while also covering authz, DB, APIs, CLI, SQL, and input boundaries.

### Better than “Jankurai SemVer tool”

Public API compatibility matters, but it is only one proof surface. Most product failures happen in authz, data isolation, migrations, command handlers, and input handling. ProofBind includes SemVer as one obligation type.

### Better than “Jankurai mutation wrapper”

Mutation testing is powerful but expensive. ProofBind routes mutation testing only to changed business/authz surfaces where it delivers high signal.

---

## 19. Final recommendation

Build `jankurai-proofbind` first.

It is the highest-leverage crate because it turns Jankurai from a path-and-lane conformance standard into a **semantic proof standard**. It preserves the current stack, uses existing proof lanes, improves popular Rust surfaces, hardens agent tools, catches false-green tests, and gives coding agents a concrete, machine-readable answer to the most important merge question:

> What exactly changed, and what proof is required before anyone should trust it?

That is the most direct possible reduction in vibe coding.
