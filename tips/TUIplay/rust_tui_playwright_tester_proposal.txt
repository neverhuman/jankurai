# Tuiwright: a Playwright-class Rust TUI test framework

**Goal:** build a Rust-native testing framework for terminal user interfaces that can drive a real application through a pseudo-terminal, wait for real rendered state, assert on user-visible behavior, capture deterministic high-resolution screenshots, generate web-compressed GIF recordings, and preserve a trace that makes failures debuggable.

This document is intentionally written as a single copy/pasteable Markdown blueprint. It contains the product proposal, architecture, API design, implementation plan, risk register, and a reference Rust workspace embedded as a bootstrap script.

---

## 1. Why this should exist

Rust TUI apps are now sophisticated enough that widget-buffer snapshots are not sufficient. They prove that a renderer produced a buffer, but not that a user can actually drive the app through a terminal. A Playwright-class TUI tester needs to sit outside the app, attach through a PTY, send the same keyboard/mouse bytes a terminal would send, parse terminal output, maintain a terminal screen model, and produce artifacts that look like a user’s terminal.

The core design is:

```text
Rust test code / cargo-tuiwright CLI
        |
        v
Tuiwright Page API  <---- locators, expectations, timeouts, tracing
        |
        v
PTY controller  <---- keyboard/mouse/paste/resize input bytes
        |
        v
real TUI process, e.g. ratatui+crossterm app
        |
        v
terminal byte stream, ANSI/VT/OSC sequences
        |
        v
terminal emulator model  <---- screen cells, cursor, title, modes, annotations
        |
        +---- text/semantic locators
        +---- PNG/SVG renderer
        +---- GIF recorder
        +---- trace/report artifacts
```

The key shift is that tests validate **actual terminal driving behavior**, not only library-level render output.

## 2. Inspiration from Playwright, adapted to TUIs

| Playwright concept | TUI equivalent in Tuiwright |
|---|---|
| Browser context isolation | Per-test PTY, temp directory, environment, fixed terminal size, fixed TERM/COLORTERM |
| Page | `Page`, a live PTY-backed terminal session |
| Locators | Text, regex, cell-region, cursor, semantic role, label, test id, style-aware queries |
| Auto-waiting | Wait until terminal output settles, screen matches, locator is visible/actionable |
| User actions | Key press, type text, bracketed paste, mouse click, drag, scroll, resize |
| Screenshots | Render terminal cell grid into deterministic high-DPI PNG/SVG |
| Videos | Capture screen frames and encode to high-quality GIF/APNG/WebP; GIF is first-class |
| Trace viewer | Timeline of actions, terminal snapshots, input/output bytes, screenshots, failures |
| Visual comparisons | Pixel screenshots plus terminal-cell diffs with redaction/masking |
| Codegen | Interactive proxy that records a human session into Rust flow code |

The result should feel like this:

```rust
use tuiwright::{expect, spawn, Key, RenderOptions};

#[test]
fn user_can_submit_settings() -> anyhow::Result<()> {
    let page = spawn(env!("CARGO_BIN_EXE_my-tui"))
        .size(100, 30)
        .env("TUIWRIGHT", "1")
        .record(true)
        .start()?;

    page.get_by_text("Settings").click()?;
    page.get_by_text("User name").click()?;
    page.type_text("ada")?;
    page.press(Key::Enter)?;

    expect(page.get_by_text("Saved"))
        .to_be_visible()?;

    page.screenshot("target/tuiwright/settings-saved.png", RenderOptions::ci())?;
    page.save_gif("target/tuiwright/settings-flow.gif", RenderOptions::ci(), Default::default())?;
    page.save_trace("target/tuiwright/settings-flow.trace")?;
    Ok(())
}
```

## 3. Principles

1. **Black-box first.** A test should work against any executable TUI, including Ratatui, Cursive, raw Crossterm, ncurses-like apps, or apps written in other languages.
2. **Rust-native runtime.** The driver, terminal model, render pipeline, recorder, and CLI are Rust. Bash below is only a repo bootstrap convenience.
3. **User-visible assertions.** Prefer assertions against what a user sees: text, cursor, style, menus, errors, selections, and screen snapshots.
4. **Semantic enhancement is optional.** Apps can emit hidden test annotations for robust `get_by_role` / `get_by_test_id`, but tests must not require instrumentation.
5. **Determinism is a feature.** Terminal size, pixel size, fonts, color theme, locale, animation policy, and environment are controlled.
6. **Artifacts must explain failures.** A failing assertion should produce: screen text, annotated screenshot, terminal diff, input/output trace, and optionally GIF.
7. **CI must be boring.** No GUI, no X server, no browser, no actual terminal window; only a PTY and Rust libraries.

## 4. Product surface

### 4.1 Rust API

```rust
let page = tuiwright::spawn("my-tui")
    .args(["--config", "fixtures/demo.toml"])
    .cwd(project_root)
    .env("TERM", "xterm-256color")
    .env("COLORTERM", "truecolor")
    .size(120, 40)
    .timeout(Duration::from_secs(5))
    .record(true)
    .start()?;

page.press(Key::Char('j'))?;
page.press(Key::Down)?;
page.type_text("hello")?;
page.paste("large pasted input")?;
page.mouse().click(10, 5)?;
page.get_by_text("Submit").click()?;
page.resize(160, 50)?;

expect(page.get_by_text("Success")).to_be_visible()?;
expect(page.locator(Query::regex("Saved in \\d+ms"))).to_be_visible()?;
expect(page.cursor()).to_be_at(12, 9)?;
expect(page.screen()).to_match_text_snapshot("settings-screen")?;
expect(page).to_have_screenshot("settings.png")?;
```

## 5. Architecture in detail

### 5.1 Crates

```text
tuiwright/
  crates/
    tuiwright/             # public API, PTY, terminal model, locators, renderer, recorder
    cargo-tuiwright/       # CLI runner, codegen, trace viewer launcher
    tuiwright-ratatui/     # optional semantic annotations for Ratatui apps
    tuiwright-viewer/      # optional richer static trace viewer generator
  examples/
    demo-tui/              # crossterm demo app used by integration tests
```

The first crate should be useful by itself. The CLI and Ratatui helper are additive.

### 5.2 The PTY layer

Responsibilities:

- Spawn a process in a pseudo-terminal.
- Set a deterministic terminal size and pixel size.
- Inject environment variables.
- Read output bytes continuously.
- Write input bytes exactly as a terminal would.
- Resize and signal the child correctly.
- Terminate child processes and clean up on panic.

Important default environment:

```text
TERM=xterm-256color
COLORTERM=truecolor
TUIWRIGHT=1
RUST_BACKTRACE=1
LC_ALL=C.UTF-8      # where available
TZ=UTC              # optional for deterministic snapshots
NO_COLOR unset      # unless a specific test wants no color
```

### 5.3 The terminal emulator layer

The terminal model must be able to answer:

- What text is currently visible?
- What is at row/col?
- What style does each cell have?
- What is the cursor position/style/visibility?
- Is alternate screen active?
- Are application cursor, keypad, bracketed paste, and mouse modes enabled?
- What did the app set as title/icon?
- Were there parse errors, bells, terminal queries, or unhandled graphics protocols?

A practical path:

1. **MVP:** use an existing Rust terminal parser with an in-memory screen.
2. **V1:** wrap that model with a `Screen` type that normalizes cells, colors, modes, cursor, and text search.
3. **V2:** build or embed a fuller terminal emulator model based on a low-level VTE parser so Tuiwright can capture custom OSC annotations, SGR mouse modes, Kitty keyboard protocol, OSC 8 hyperlinks, Sixel/Kitty graphics metadata, and terminal replies.

### 5.4 Locator model

Black-box locators:

```rust
page.get_by_text("Submit")
page.get_by_regex(r"Saved in \\d+ms")
page.get_by_line(12)
page.get_by_cell(10, 5)
page.get_by_region(Rect::new(0, 0, 50, 10))
page.get_by_style(|cell| cell.fg == Rgb(255,0,0))
page.cursor()
```

Semantic locators, enabled through hidden annotations:

```rust
page.get_by_role(Role::Button, "Submit")
page.get_by_label("User name")
page.get_by_test_id("save-button")
```

The semantic mechanism should not render anything visible. When `TUIWRIGHT=1`, an app or helper crate emits a private OSC marker describing a node:

```text
ESC ] 777 ; TUIWRIGHT ; {json} BEL
```

Example semantic node:

```json
{
  "id": "save-button",
  "role": "button",
  "name": "Save",
  "rect": { "x": 42, "y": 20, "width": 8, "height": 1 },
  "state": { "focused": true, "disabled": false }
}
```

Real terminals ignore unknown OSC sequences; Tuiwright captures them and attaches them to the current screen epoch. In production builds, annotations should be disabled unless `TUIWRIGHT=1` is present.

### 5.5 Actionability

Before clicking a locator, Tuiwright should auto-wait for:

- Exactly one match, unless `.first()` or `.nth()` is used.
- Match is within viewport.
- Match is not empty text.
- Screen is stable for `idle_ms`.
- If semantic state exists, node is not disabled/hidden.
- If mouse is required, app has enabled mouse tracking; otherwise warn or fail depending on strictness.

For terminal apps, “visible” means cells exist in the current visible grid and are not only in scrollback.

### 5.6 Input encoder

Keyboard:

- ASCII text.
- Enter, Tab, Backspace, Escape.
- Arrows, Home/End, Insert/Delete, PageUp/PageDown.
- Function keys.
- Ctrl/Alt/Shift modifiers where terminals support them.
- Application cursor/keypad mode detection.
- Bracketed paste, automatically using `ESC[200~` / `ESC[201~` when enabled.
- Kitty keyboard protocol support in V2.

Mouse:

- SGR 1006 mouse encoding as default.
- Click, double-click, drag, release, wheel scroll.
- Translate locator center to 1-based terminal coordinates.
- Do not silently send mouse bytes if app has not requested mouse reporting in strict mode.

Resize:

- `page.resize(cols, rows)` calls PTY resize and waits for the app to redraw.

### 5.7 Screenshots

Tuiwright screenshots are not OS-level screenshots. They are deterministic renderings of the terminal screen model.

Supported artifact types:

- **Text snapshot:** plain visible rows.
- **ANSI snapshot:** screen contents with style escape sequences.
- **Cell JSON:** rows, cols, cells, style, cursor, title, modes.
- **SVG screenshot:** vector terminal screenshot, useful for docs.
- **PNG screenshot:** high-resolution raster screenshot with configured font and cell metrics.
- **Element screenshot:** crop around a locator region.
- **Diff artifact:** baseline, actual, cell diff, pixel diff, masks.

`RenderOptions` controls:

```rust
RenderOptions {
    font_path: PathBuf,
    font_size_px: 16.0,
    cell_width_px: 10,
    cell_height_px: 20,
    scale: 2,
    theme: TerminalTheme::XtermDark,
    draw_cursor: true,
    trim_empty_edges: false,
    include_window_chrome: false,
}
```

For CI determinism, require an explicit `TUIWRIGHT_FONT` path or a repo-managed test font. Do not rely on whatever font happens to be installed.

### 5.8 GIF recordings

Recording pipeline:

1. Every terminal output chunk updates the terminal model.
2. The recorder samples frames after screen changes, capped by `max_fps`.
3. Duplicate frames are removed.
4. Small time deltas are coalesced.
5. Frames are rendered with the same PNG renderer used for screenshots.
6. Frames are encoded with a high-quality Rust GIF encoder.
7. Optional web compression settings reduce dimensions, FPS, and quality.

`GifOptions`:

```rust
GifOptions {
    max_fps: 12,
    speed: GifSpeed::Web,
    quality: 85,
    max_width_px: Some(1200),
    loop_forever: true,
    highlight_actions: true,
    action_subtitles: true,
    drop_duplicate_frames: true,
}
```

Important issue: the highest-quality Rust GIF encoder often has license implications. Keep GIF encoding pluggable: a permissive default encoder for broad adoption, and an optional `gifski` feature for teams that accept its license.

### 5.9 Trace format

A trace should be a directory or zip with:

```text
settings-flow.trace/
  trace.jsonl              # action, assertion, output, input, screenshot events
  meta.json                # OS, crate version, TERM, size, font, command, env allowlist
  frames/
    000001.png
    000002.png
  snapshots/
    000001.cells.json
    000001.txt
  attachments/
    failure.png
    diff.png
  index.html               # static viewer generated by Rust
```

Trace event example:

```json
{"ts_ms": 0, "type":"spawn", "cmd":["my-tui","--demo"], "cols":100, "rows":30}
{"ts_ms": 120, "type":"output", "bytes_b64":"G1s/..."}
{"ts_ms": 200, "type":"action", "kind":"click", "target":"text=Settings", "x":8, "y":2}
{"ts_ms": 410, "type":"assertion", "kind":"to_be_visible", "target":"text=Saved", "status":"passed"}
```

The viewer should support:

- Timeline of actions and output bursts.
- Screen text per frame.
- Screenshot per frame.
- Locator highlight overlays.
- Input/output byte inspection.
- Failure stack and assertion details.
- Side-by-side baseline/actual diffs.

A Vite/TypeScript viewer can be added, but the default should be a single static HTML file generated by Rust so CI artifacts are easy to open.

            pixel_width: 0,

    pub args: Vec<String>,

    pub record: bool,

        let mut env = BTreeMap::new();
        env.insert("TERM".to_string(), "xterm-256color".to_string());
        env.insert("COLORTERM".to_string(), "truecolor".to_string());

            record: false,
        }
    }

    pub fn arg(mut self, arg: impl Into<String>) -> Self {
        self.args.push(arg.into());
        self
    }

    pub fn args<I, S>(mut self, args: I) -> Self
    where
        I: IntoIterator<Item = S>,
        S: Into<String>,
    {
        self.args.extend(args.into_iter().map(Into::into));
        self
    }

    pub fn cwd(mut self, cwd: impl Into<PathBuf>) -> Self {
        self.cwd = Some(cwd.into());
        self
    }

    pub fn env(mut self, key: impl Into<String>, value: impl Into<String>) -> Self {
        self.env.insert(key.into(), value.into());
        self
    }

    pub fn size(mut self, cols: u16, rows: u16) -> Self {
        self.size = TerminalSize::new(cols, rows);
        self
    }

    pub fn pixel_size(mut self, width: u16, height: u16) -> Self {
        self.size.pixel_width = width;
        self.size.pixel_height = height;
        self
    }

    pub fn timeout(mut self, timeout: Duration) -> Self {
        self.default_timeout = timeout;
        self
    }

    pub fn idle_timeout(mut self, timeout: Duration) -> Self {
        self.idle_timeout = timeout;
        self
    }

    pub fn record(mut self, enabled: bool) -> Self {
        self.record = enabled;
        self
    }

    pub fn start(self) -> anyhow::Result<Page> {
        Page::start(self)
    }

#[derive(Debug, Clone, PartialEq, Eq)]
pub enum Key {
    Char(char),
    Text(String),
    Ctrl(char),
    Alt(char),
    Enter,
    Esc,
    Tab,
    BackTab,
    Backspace,
    Delete,
    Insert,
    Up,
    Down,
    Left,
    Right,
    Home,
    End,
    PageUp,
    PageDown,
    F(u8),
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum MouseButton {
    Left,
    Middle,
    Right,
}

/// Encode a key as bytes that a terminal would send to the application.
///
/// This intentionally starts with the common xterm-compatible subset. A production
/// implementation should negotiate Kitty keyboard protocol and other extended modes.
pub fn encode_key(key: &Key, application_cursor: bool) -> Vec<u8> {
    match key {
        Key::Char(c) => c.to_string().into_bytes(),
        Key::Text(s) => s.as_bytes().to_vec(),
        Key::Ctrl(c) => {
            let lower = c.to_ascii_lowercase() as u8;
            if (b'a'..=b'z').contains(&lower) {
                vec![lower - b'a' + 1]
            } else {
                vec![*c as u8]
            }
        }
        Key::Alt(c) => {
            let mut out = vec![0x1b];
            out.extend(c.to_string().bytes());
            out
        }
        Key::Enter => b"\r".to_vec(),
        Key::Esc => vec![0x1b],
        Key::Tab => b"\t".to_vec(),
        Key::BackTab => b"\x1b[Z".to_vec(),
        Key::Backspace => vec![0x7f],
        Key::Delete => b"\x1b[3~".to_vec(),
        Key::Insert => b"\x1b[2~".to_vec(),
        Key::Up => if application_cursor { b"\x1bOA".to_vec() } else { b"\x1b[A".to_vec() },
        Key::Down => if application_cursor { b"\x1bOB".to_vec() } else { b"\x1b[B".to_vec() },
        Key::Right => if application_cursor { b"\x1bOC".to_vec() } else { b"\x1b[C".to_vec() },
        Key::Left => if application_cursor { b"\x1bOD".to_vec() } else { b"\x1b[D".to_vec() },
        Key::Home => b"\x1b[H".to_vec(),
        Key::End => b"\x1b[F".to_vec(),
        Key::PageUp => b"\x1b[5~".to_vec(),
        Key::PageDown => b"\x1b[6~".to_vec(),
        Key::F(n) => encode_function_key(*n),
    }
}

fn encode_function_key(n: u8) -> Vec<u8> {
    match n {
        1 => b"\x1bOP".to_vec(),
        2 => b"\x1bOQ".to_vec(),
        3 => b"\x1bOR".to_vec(),
        4 => b"\x1bOS".to_vec(),
        5 => b"\x1b[15~".to_vec(),
        6 => b"\x1b[17~".to_vec(),
        7 => b"\x1b[18~".to_vec(),
        8 => b"\x1b[19~".to_vec(),
        9 => b"\x1b[20~".to_vec(),
        10 => b"\x1b[21~".to_vec(),
        11 => b"\x1b[23~".to_vec(),
        12 => b"\x1b[24~".to_vec(),
        _ => Vec::new(),
    }
}

/// Encode an SGR 1006 mouse event. Coordinates are 0-based cell coordinates here;
/// terminals send 1-based coordinates over the wire.
pub fn encode_sgr_mouse(button: MouseButton, x: u16, y: u16, down: bool) -> Vec<u8> {
    let cb = match button {
        MouseButton::Left => 0,
        MouseButton::Middle => 1,
        MouseButton::Right => 2,
    };
    let suffix = if down { 'M' } else { 'm' };
    format!("\x1b[<{};{};{}{}", cb, x + 1, y + 1, suffix).into_bytes()
}

pub fn encode_sgr_scroll(x: u16, y: u16, lines: i16) -> Vec<u8> {
    // Wheel up is 64, wheel down is 65. Repeat for multi-line scrolls.
    let cb = if lines < 0 { 64 } else { 65 };
    let count = lines.unsigned_abs().max(1);
    let mut out = Vec::new();
    for _ in 0..count {
        out.extend(format!("\x1b[<{};{};{}M", cb, x + 1, y + 1).bytes());
    }
    out
}

pub fn encode_bracketed_paste(text: &str) -> Vec<u8> {
    let mut out = b"\x1b[200~".to_vec();
    out.extend(text.as_bytes());
    out.extend(b"\x1b[201~");
    out
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct Cell {
    pub text: String,
    pub fg: Rgb,
    pub bg: Rgb,
    pub bold: bool,
    pub italic: bool,
    pub underline: bool,
    pub inverse: bool,
    pub wide: bool,
    pub wide_continuation: bool,
}

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct Screen {
    pub rows: u16,
    pub cols: u16,
    pub cells: Vec<Cell>,
    pub cursor_row: u16,
    pub cursor_col: u16,
    pub cursor_hidden: bool,
    pub alternate_screen: bool,
    pub application_cursor: bool,
    pub bracketed_paste: bool,
    pub title: String,
    pub parse_errors: usize,
}

impl Screen {
    pub fn blank(cols: u16, rows: u16) -> Self {
        let cell = Cell {
            text: " ".to_string(),
            fg: Rgb(229, 229, 229),
            bg: Rgb(0, 0, 0),
            bold: false,
            italic: false,
            underline: false,
            inverse: false,
            wide: false,
            wide_continuation: false,
        };
        Self {
            rows,
            cols,
            cells: vec![cell; rows as usize * cols as usize],
            cursor_row: 0,
            cursor_col: 0,
            cursor_hidden: false,
            alternate_screen: false,
            application_cursor: false,
            bracketed_paste: false,
            title: String::new(),
            parse_errors: 0,
        }
    }

    pub fn from_vt<Reply: vt100::TermReplySender>(vt: &vt100::Screen<Reply>) -> Self {
        let (rows, cols) = vt.size();
        let mut screen = Screen::blank(cols, rows);
        let (cursor_row, cursor_col) = vt.cursor_position();
        screen.cursor_row = cursor_row;
        screen.cursor_col = cursor_col;
        screen.cursor_hidden = vt.hide_cursor();
        screen.alternate_screen = vt.alternate_screen();
        screen.application_cursor = vt.application_cursor();
        screen.bracketed_paste = vt.bracketed_paste();
        screen.title = vt.title().to_string();
        screen.parse_errors = vt.errors();

        for row in 0..rows {
            for col in 0..cols {
                if let Some(cell) = vt.cell(row, col) {
                    let mut fg = color_to_rgb(cell.fgcolor(), Rgb(229, 229, 229));
                    let mut bg = color_to_rgb(cell.bgcolor(), Rgb(0, 0, 0));
                    if cell.inverse() {
                        std::mem::swap(&mut fg, &mut bg);
                    }
                    screen[(row, col)] = Cell {
                        text: if cell.has_contents() { cell.contents().to_string() } else { " ".to_string() },
                        fg,
                        bg,
                        bold: cell.bold(),
                        italic: cell.italic(),
                        underline: cell.underline(),
                        inverse: cell.inverse(),
                        wide: cell.is_wide(),
                        wide_continuation: vt.is_wide_continuation(row, col),
                    };
                }
            }
        }
        screen
    }

    pub fn line(&self, row: u16) -> String {
        if row >= self.rows {
            return String::new();
        }
        let mut out = String::new();
        for col in 0..self.cols {
            let c = &self[(row, col)];
            if c.wide_continuation {
                continue;
            }
            out.push_str(if c.text.is_empty() { " " } else { &c.text });
        }
        out
    }

    pub fn text(&self) -> String {
        (0..self.rows).map(|r| self.line(r)).collect::<Vec<_>>().join("\n")
    }

    pub fn trimmed_text(&self) -> String {
        (0..self.rows)
            .map(|r| self.line(r).trim_end().to_string())
            .collect::<Vec<_>>()
            .join("\n")
            .trim_end()
            .to_string()
    }

    pub fn find_text(&self, needle: &str) -> Vec<Rect> {
        let mut rects = Vec::new();
        for row in 0..self.rows {
            let line = self.line(row);
            let mut start = 0;
            while let Some(pos) = line[start..].find(needle) {
                let byte_idx = start + pos;
                let x = line[..byte_idx].chars().count() as u16;
                let width = needle.chars().count().max(1) as u16;
                rects.push(Rect::new(x.min(self.cols.saturating_sub(1)), row, width.min(self.cols), 1));
                start = byte_idx + needle.len().max(1);
                if start >= line.len() {
                    break;
                }
            }
        }
        rects
    }

    pub fn find_regex(&self, regex: &Regex) -> Vec<Rect> {
        let mut rects = Vec::new();
        for row in 0..self.rows {
            let line = self.line(row);
            for mat in regex.find_iter(&line) {
                let x = line[..mat.start()].chars().count() as u16;
                let width = mat.as_str().chars().count().max(1) as u16;
                rects.push(Rect::new(x.min(self.cols.saturating_sub(1)), row, width.min(self.cols), 1));
            }
        }
        rects
    }

    pub fn get(&self, row: u16, col: u16) -> Option<&Cell> {
        if row < self.rows && col < self.cols {
            Some(&self[(row, col)])
        } else {
            None
        }
    }
}

impl std::ops::Index<(u16, u16)> for Screen {
    type Output = Cell;

    fn index(&self, index: (u16, u16)) -> &Self::Output {
        let (row, col) = index;
        &self.cells[row as usize * self.cols as usize + col as usize]
    }
}

impl std::ops::IndexMut<(u16, u16)> for Screen {
    fn index_mut(&mut self, index: (u16, u16)) -> &mut Self::Output {
        let (row, col) = index;
        &mut self.cells[row as usize * self.cols as usize + col as usize]
    }
}

impl fmt::Display for Screen {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {

    }
}

fn color_to_rgb(color: vt100::Color, default: Rgb) -> Rgb {
    match color {
        vt100::Color::Default => default,
        vt100::Color::Rgb(r, g, b) => Rgb(r, g, b),
        vt100::Color::Idx(idx) => xterm_index_to_rgb(idx),
    }
}

fn xterm_index_to_rgb(idx: u8) -> Rgb {
    const BASE16: [Rgb; 16] = [
        Rgb(0, 0, 0),
        Rgb(205, 0, 0),
        Rgb(0, 205, 0),
        Rgb(205, 205, 0),
        Rgb(0, 0, 238),
        Rgb(205, 0, 205),
        Rgb(0, 205, 205),
        Rgb(229, 229, 229),
        Rgb(127, 127, 127),
        Rgb(255, 0, 0),
        Rgb(0, 255, 0),
        Rgb(255, 255, 0),
        Rgb(92, 92, 255),
        Rgb(255, 0, 255),
        Rgb(0, 255, 255),
        Rgb(255, 255, 255),
    ];
    match idx {
        0..=15 => BASE16[idx as usize],
        16..=231 => {
            let n = idx - 16;
            let r = n / 36;
            let g = (n % 36) / 6;
            let b = n % 6;
            let map = |v: u8| if v == 0 { 0 } else { 55 + 40 * v };
            Rgb(map(r), map(g), map(b))
        }
        232..=255 => {
            let v = 8 + (idx - 232) * 10;
            Rgb(v, v, v)
        }
    }
}

pub(crate) fn resolve_in_screen(screen: &crate::Screen, query: &Query) -> Result<ResolvedLocator> {
    match query {
        Query::Text(text) => {
            let matches = screen.find_text(text);
            match matches.len() {
                0 => bail!("no visible text match for {text:?}\n\nCurrent screen:\n{screen}"),
                1 => Ok(ResolvedLocator { query_label: format!("text={text:?}"), rect: matches[0] }),
                n => bail!("ambiguous text locator {text:?}: {n} matches; use a stricter query or .nth() in the production API"),
            }
        }
        Query::Regex(regex) => {
            let matches = screen.find_regex(regex);
            match matches.len() {
                0 => bail!("no visible regex match for /{}/\n\nCurrent screen:\n{screen}", regex.as_str()),
                1 => Ok(ResolvedLocator { query_label: format!("regex=/{}/", regex.as_str()), rect: matches[0] }),
                n => bail!("ambiguous regex locator /{}/: {n} matches", regex.as_str()),
            }
        }
        Query::Rect(rect) => {
            if rect.x < screen.cols && rect.y < screen.rows {
                Ok(ResolvedLocator { query_label: format!("rect={rect:?}"), rect: *rect })
            } else {
                Err(anyhow!("rect {rect:?} is outside the screen {}x{}", screen.cols, screen.rows))
            }
        }
    }

    pub fn render_region_png(&self, screen: &Screen, region: Rect) -> Result<RgbaImage> {

            bail!("region {region:?} outside screen {}x{}", screen.cols, screen.rows);

            }

        }

fn fill_rect(img: &mut RgbaImage, x: u32, y: u32, w: u32, h: u32, color: Rgba<u8>) {
    for yy in y..(y + h).min(img.height()) {
        for xx in x..(x + w).min(img.width()) {
            img.put_pixel(xx, yy, color);
        }
    }
}

fn stroke_rect(img: &mut RgbaImage, x: u32, y: u32, w: u32, h: u32, color: Rgba<u8>, thickness: u32) {
    fill_rect(img, x, y, w, thickness, color);
    fill_rect(img, x, y + h.saturating_sub(thickness), w, thickness, color);
    fill_rect(img, x, y, thickness, h, color);
    fill_rect(img, x + w.saturating_sub(thickness), y, thickness, h, color);
}

fn blit_alpha(img: &mut RgbaImage, bitmap: &[u8], width: usize, height: usize, x0: i32, y0: i32, color: Rgba<u8>) {
    for y in 0..height {
        for x in 0..width {
            let alpha = bitmap[y * width + x];
            if alpha == 0 {
                continue;
            }
            let dx = x0 + x as i32;
            let dy = y0 + y as i32;
            if dx < 0 || dy < 0 || dx >= img.width() as i32 || dy >= img.height() as i32 {
                continue;
            }
            let dst = img.get_pixel_mut(dx as u32, dy as u32);
            let a = alpha as f32 / 255.0;
            for i in 0..3 {
                dst.0[i] = ((color.0[i] as f32 * a) + (dst.0[i] as f32 * (1.0 - a))).round() as u8;
            }
            dst.0[3] = 255;
        }
    }
}

    candidates.iter().map(PathBuf::from).find(|p| p.exists())

use std::{path::Path, sync::Mutex, time::{Duration, Instant}};

use anyhow::{bail, Result};

use crate::{render::{RenderOptions, Renderer}, screen::Screen};

#[derive(Debug, Clone)]
pub struct RecordedFrame {
    pub timestamp: Duration,
    pub screen: Screen,
}

#[derive(Debug)]
pub struct Recording {
    enabled: bool,
    start: Instant,
    min_frame_gap: Duration,
    last_capture: Mutex<Option<Instant>>,
    last_screen: Mutex<Option<Screen>>,
    frames: Mutex<Vec<RecordedFrame>>,
}

impl Recording {
    pub fn new(enabled: bool) -> Self {
        Self {
            enabled,
            start: Instant::now(),
            min_frame_gap: Duration::from_millis(80),
            last_capture: Mutex::new(None),
            last_screen: Mutex::new(None),
            frames: Mutex::new(Vec::new()),
        }
    }

    pub fn enabled(&self) -> bool {
        self.enabled
    }

    pub fn maybe_capture(&self, screen: &Screen) {
        if !self.enabled {
            return;
        }
        let now = Instant::now();
        let mut last_capture = self.last_capture.lock().unwrap();
        if last_capture.map(|t| now.duration_since(t) < self.min_frame_gap).unwrap_or(false) {
            return;
        }
        let mut last_screen = self.last_screen.lock().unwrap();
        if last_screen.as_ref() == Some(screen) {
            return;
        }
        *last_capture = Some(now);
        *last_screen = Some(screen.clone());
        self.frames.lock().unwrap().push(RecordedFrame { timestamp: now.duration_since(self.start), screen: screen.clone() });
    }

    pub fn frames(&self) -> Vec<RecordedFrame> {
        self.frames.lock().unwrap().clone()
    }
}

#[derive(Debug, Clone)]
pub struct GifOptions {
    pub quality: u8,
    pub max_width: Option<u32>,
    pub max_height: Option<u32>,
    pub fast: bool,
}

impl Default for GifOptions {
    fn default() -> Self {
        Self { quality: 85, max_width: Some(1200), max_height: None, fast: false }
    }
}

#[cfg(feature = "gif")]
pub fn save_gif(path: impl AsRef<Path>, frames: &[RecordedFrame], render: RenderOptions, gif: GifOptions) -> Result<()> {
    use std::{fs, thread};

    if frames.is_empty() {
        bail!("no recorded frames; call .record(true) when starting the page");
    }
    let path = path.as_ref();
    if let Some(parent) = path.parent() {
        fs::create_dir_all(parent)?;
    }

    let temp = tempfile::tempdir()?;
    let renderer = Renderer::new(render)?;
    let mut png_paths = Vec::with_capacity(frames.len());
    for (idx, frame) in frames.iter().enumerate() {
        let png = temp.path().join(format!("frame-{idx:06}.png"));
        renderer.write_png(&frame.screen, &png)?;
        png_paths.push((png, frame.timestamp.as_secs_f64()));
    }

    let settings = gifski::Settings {
        width: gif.max_width,
        height: gif.max_height,
        quality: gif.quality,
        fast: gif.fast,
        ..Default::default()
    };
    let (collector, writer) = gifski::new(settings)?;

    thread::scope(|scope| -> Result<()> {
        let producer = scope.spawn(move || -> Result<()> {
            for (idx, (png, ts)) in png_paths.into_iter().enumerate() {
                collector.add_frame_png_file(idx, png, ts)?;
            }
            drop(collector);
            Ok(())
        });
        writer.write(std::fs::File::create(path)?, &mut gifski::progress::NoProgress {})?;
        producer.join().unwrap()?;
        Ok(())
    })
}

#[cfg(not(feature = "gif"))]

    bail!("GIF encoding requires building tuiwright with the `gif` feature. Note: review the GIF encoder license before enabling it.");
}

    master: Mutex<Box<dyn MasterPty + Send>>,

impl Page {
    pub(crate) fn start(spec: CommandSpec) -> Result<Self> {
        let pty_system = native_pty_system();
        let pty_size = PtySize {
            rows: spec.size.rows,
            cols: spec.size.cols,
            pixel_width: spec.size.pixel_width,
            pixel_height: spec.size.pixel_height,
        };
        let pair = pty_system.openpty(pty_size)?;

        let mut cmd = CommandBuilder::new(&spec.program);
        cmd.args(&spec.args);

            cmd.cwd(cwd);
        }
        for (k, v) in &spec.env {
            cmd.env(k, v);
        }

        let mut reader = pair.master.try_clone_reader()?;
        let writer = pair.master.take_writer()?;
        let child = pair.slave.spawn_command(cmd)
            .with_context(|| format!("spawning {}", spec.program))?;

        let inner = Arc::new(Inner {
            writer: Mutex::new(writer),
            master: Mutex::new(pair.master),
            child: Mutex::new(child),
            parser: Mutex::new(vt100::Parser::new(spec.size.rows, spec.size.cols, 10_000)),
            last_output: Mutex::new(Instant::now()),
            default_timeout: spec.default_timeout,
            idle_timeout: spec.idle_timeout,
            trace: TraceLog::new(),
            recording: Recording::new(spec.record),
        });

        inner.trace.push(TraceEvent::Spawn {
            ts_ms: 0,
            cmd: std::iter::once(spec.program).chain(spec.args).collect(),
            cols: spec.size.cols,
            rows: spec.size.rows,
        });

        let reader_inner = inner.clone();
        thread::spawn(move || read_loop(&mut reader, &reader_inner));

        let page = Page { inner };
        // Give the app a moment to paint initial content, but do not require it.
        let _ = page.wait_for_idle_timeout(Duration::from_millis(500));
        Ok(page)

    pub fn default_timeout(&self) -> Duration {
        self.inner.default_timeout
    }

    pub fn screen(&self) -> Result<Screen> {
        let parser = self.inner.parser.lock().unwrap();
        Ok(Screen::from_vt(parser.screen()))
    }

    pub fn wait_for_idle(&self) -> Result<()> {
        self.wait_for_idle_timeout(self.inner.default_timeout)
    }

    pub fn wait_for_idle_timeout(&self, timeout: Duration) -> Result<()> {
        let deadline = Instant::now() + timeout;
        loop {
            let last = *self.inner.last_output.lock().unwrap();
            if last.elapsed() >= self.inner.idle_timeout {
                return Ok(());
            }
            if Instant::now() > deadline {
                bail!("terminal did not become idle within {timeout:?}");
            }
            thread::sleep(Duration::from_millis(10));
        }
    }

    pub fn wait_for_text(&self, text: &str) -> Result<Screen> {
        let deadline = Instant::now() + self.inner.default_timeout;
        loop {
            let _ = self.wait_for_idle_timeout(Duration::from_millis(250));
            let screen = self.screen()?;
            if screen.text().contains(text) {
                self.inner.trace.push(TraceEvent::Assertion {
                    ts_ms: self.inner.trace.ts_ms(),
                    label: format!("screen contains {text:?}"),
                    status: "passed".to_string(),
                });
                return Ok(screen);
            }
            if Instant::now() > deadline {
                self.inner.trace.push(TraceEvent::Assertion {
                    ts_ms: self.inner.trace.ts_ms(),
                    label: format!("screen contains {text:?}"),
                    status: "failed".to_string(),
                });
                bail!("timed out waiting for text {text:?}\n\nCurrent screen:\n{screen}");
            }
            thread::sleep(Duration::from_millis(20));
        }
    }

        let application_cursor = self.screen().map(|s| s.application_cursor).unwrap_or(false);

        self.write_input(format!("press {key:?}"), &bytes)

        self.write_input(format!("type {text:?}"), text.as_bytes())

        let bytes = if bracketed { encode_bracketed_paste(text) } else { text.as_bytes().to_vec() };

    }

        let mut bytes = encode_sgr_mouse(button, x, y, true);
        bytes.extend(encode_sgr_mouse(button, x, y, false));
        self.inner.trace.push(TraceEvent::Action { ts_ms: self.inner.trace.ts_ms(), label: format!("click {button:?} at {x},{y}") });
        self.write_input(format!("mouse click {x},{y}"), &bytes)
    }

    pub fn scroll_at(&self, x: u16, y: u16, lines: i16) -> Result<()> {
        let bytes = encode_sgr_scroll(x, y, lines);
        self.write_input(format!("scroll {lines} at {x},{y}"), &bytes)
    }

    pub fn resize(&self, cols: u16, rows: u16) -> Result<()> {
        self.inner.master.lock().unwrap().resize(PtySize { rows, cols, pixel_width: 0, pixel_height: 0 })?;

        // so production should call the terminal model's resize method here. As a conservative fallback, process
        // the common CSI resize-triggered clear only after the app redraws. This scaffold relies on app redraw.
        self.inner.trace.push(TraceEvent::Action { ts_ms: self.inner.trace.ts_ms(), label: format!("resize {cols}x{rows}") });
        self.wait_for_idle()
    }

        Locator::new(self, Query::Text(text.into()))
    }

    pub fn get_by_regex(&self, pattern: &str) -> Result<Locator<'_>> {
        Ok(Locator::new(self, Query::Regex(Regex::new(pattern)?)))
    }

    pub fn locator(&self, query: Query) -> Locator<'_> {
        Locator::new(self, query)
    }

    pub(crate) fn resolve_query(&self, query: &Query, timeout: Duration) -> Result<ResolvedLocator> {
        let deadline = Instant::now() + timeout;
        loop {
            let _ = self.wait_for_idle_timeout(Duration::from_millis(250));
            let screen = self.screen()?;

                Ok(resolved) => return Ok(resolved),
                Err(err) if Instant::now() <= deadline => {
                    let _ = err;
                    thread::sleep(Duration::from_millis(20));
                }
                Err(err) => return Err(err),
            }
        }
    }

        self.wait_for_idle()?;
        let screen = self.screen()?;
        let renderer = Renderer::new(options)?;
        renderer.write_png(&screen, path.as_ref())?;
        self.inner.trace.push(TraceEvent::Screenshot { ts_ms: self.inner.trace.ts_ms(), path: path.as_ref().display().to_string() });
        Ok(())
    }

    pub fn screenshot_region(&self, path: impl AsRef<Path>, options: RenderOptions, region: Rect) -> Result<()> {
        self.wait_for_idle()?;
        let screen = self.screen()?;
        let renderer = Renderer::new(options)?;
        renderer.write_region_png(&screen, region, path.as_ref())?;
        self.inner.trace.push(TraceEvent::Screenshot { ts_ms: self.inner.trace.ts_ms(), path: path.as_ref().display().to_string() });

    }

    pub fn save_gif(&self, path: impl AsRef<Path>, render: RenderOptions, gif: GifOptions) -> Result<()> {
        let mut frames = self.inner.recording.frames();
        if frames.is_empty() {
            frames.push(crate::record::RecordedFrame { timestamp: self.inner.trace.elapsed(), screen: self.screen()? });
        }

    }

    pub fn save_trace(&self, path: impl AsRef<Path>) -> Result<()> {
        let screen = self.screen()?;
        self.inner.trace.write(path, &screen.trimmed_text())
    }

    pub fn try_wait(&self) -> Result<Option<portable_pty::ExitStatus>> {
        Ok(self.inner.child.lock().unwrap().try_wait()?)
    }

    pub fn kill(&self) -> Result<()> {
        Ok(self.inner.child.lock().unwrap().kill()?)
    }

    fn write_input(&self, label: String, bytes: &[u8]) -> Result<()> {
        self.inner.trace.push_input(label, bytes);
        let mut writer = self.inner.writer.lock().unwrap();
        writer.write_all(bytes)?;
        writer.flush()?;
        Ok(())
    }
}

impl Drop for Page {
    fn drop(&mut self) {
        let _ = self.inner.child.lock().map(|mut child| child.kill());
    }
}

fn read_loop(reader: &mut Box<dyn Read + Send>, inner: &Arc<Inner>) {
    let mut buf = [0u8; 8192];
    loop {
        match reader.read(&mut buf) {
            Ok(0) => break,
            Ok(n) => {
                let chunk = &buf[..n];
                inner.trace.push_output(chunk);
                let screen = {
                    let mut parser = inner.parser.lock().unwrap();
                    parser.process(chunk);
                    Screen::from_vt(parser.screen())
                };
                *inner.last_output.lock().unwrap() = Instant::now();
                inner.recording.maybe_capture(&screen);
            }
            Err(_) => break,
        }
    }
}

use anyhow::{bail, Result};
use clap::{Args, Parser, Subcommand};
use tuiwright::{spawn, GifOptions, RenderOptions};

#[derive(Debug, Parser)]
#[command(name = "cargo-tuiwright")]
#[command(about = "Playwright-style tests and artifacts for TUI apps")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Debug, Subcommand)]

    Run(RunArgs),
}

#[derive(Debug, Args)]
struct RunArgs {
    #[arg(long, default_value_t = 80)]
    cols: u16,
    #[arg(long, default_value_t = 24)]
    rows: u16,
    #[arg(long)]
    wait_text: Option<String>,
    #[arg(long)]
    screenshot: Option<PathBuf>,
    #[arg(long)]

    #[arg(long)]
    trace: Option<PathBuf>,
    #[arg(long)]

    #[arg(long, default_value_t = 5)]
    timeout_secs: u64,
    #[arg(last = true, required = true)]
    cmd: Vec<String>,
}

fn main() -> Result<()> {
    let cli = Cli::parse();
    match cli.command {
        Commands::Run(args) => run(args),
    }
}

fn run(args: RunArgs) -> Result<()> {
    if args.cmd.is_empty() {
        bail!("missing command after --");
    }
    let mut spec = spawn(&args.cmd[0])
        .args(args.cmd.iter().skip(1).cloned().collect::<Vec<_>>())
        .size(args.cols, args.rows)
        .timeout(Duration::from_secs(args.timeout_secs))
        .record(args.gif.is_some());

    // Make cargo-run examples less noisy in traces.
    spec = spec.env("RUST_BACKTRACE", "1");

    let page = spec.start()?;
    if let Some(text) = &args.wait_text {
        page.wait_for_text(text)?;
    } else {
        page.wait_for_idle()?;
    }

    let mut render = RenderOptions::ci();
    if let Some(font) = args.font {
        render = render.font_path(font);
    }

    if let Some(path) = args.screenshot {
        page.screenshot(path, render.clone())?;
    }
    if let Some(path) = args.gif {
        page.save_gif(path, render.clone(), GifOptions::default())?;
    }
    if let Some(path) = args.trace {
        page.save_trace(path)?;
    }

    Ok(())
}

    let mut name = String::new();

                        }

                    KeyCode::Char(c) if selected == 1 => name.push(c),
                    _ => {}
                },
                Event::Mouse(m) => {
                    if let MouseEventKind::Down(_) = m.kind {
                        let row = m.row as usize;
                        if (3..=6).contains(&row) {
                            selected = row - 3;
                            status = format!("Clicked {}", buttons[selected]);
                        }
                    }
                }
                Event::Resize(_, _) => status = "Resized".into(),
                _ => {}
            }
        }
    }
    Ok(())
}

    queue!(out, SetForegroundColor(Color::Cyan), Print("Demo TUI"), ResetColor, MoveTo(0, 1))?;

    for (idx, label) in buttons.iter().enumerate() {
        queue!(out, MoveTo(2, 3 + idx as u16))?;
        if idx == selected {
            queue!(out, SetForegroundColor(Color::Yellow), SetAttribute(Attribute::Bold), Print("> "))?;
        } else {
            queue!(out, Print("  "))?;
        }
        match *label {
            "User name" => queue!(out, Print(format!("User name: {name}")))?,
            _ => queue!(out, Print(label))?,
        }
        if idx == selected {
            queue!(out, ResetColor, SetAttribute(Attribute::Reset))?;
        }
    }

    queue!(out, MoveTo(0, 10), SetForegroundColor(Color::Green), Print(format!("Status: {status}")), ResetColor)?;
    out.flush()?;

## 7. Production hardening plan

The reference scaffold proves the shape of the solution. The production version should be built in layers so each feature remains testable.

### 7.1 Milestone 0: black-box PTY smoke tester

**Acceptance criteria**

- Spawn a real process under PTY on Linux, macOS, and Windows ConPTY.
- Drive Enter, text, arrows, Escape, Ctrl-C.
- Capture visible text.
- Wait for text with timeout.
- Kill child on panic or drop.
- Run in CI without a real terminal window.

**Deliverables**

- `Page::start`, `Page::press`, `Page::type_text`, `Page::wait_for_text`.
- Text failure dumps.
- JSON trace with input/output bytes.

### 7.2 Milestone 1: locators and auto-waiting

**Acceptance criteria**

- `get_by_text`, `get_by_regex`, `get_by_cell`, `get_by_region`.
- Retry locator resolution until timeout.
- Fail clearly on zero matches or multiple matches.
- Stable-screen detection with configurable idle timeout.
- Optional `.first()`, `.last()`, `.nth(i)`, `.filter(...)`.

**Production API sketch**

```rust
page.get_by_text("Save")
    .exact(true)
    .timeout(Duration::from_secs(2))
    .click()?;

page.get_by_regex(r"error: .+")
    .first()
    .expect()
    .to_have_style(|style| style.fg == Rgb(255, 0, 0))?;
```

### 7.3 Milestone 2: high-fidelity terminal model

The reference scaffold uses an existing parser. The production system should define its own stable abstraction:

```rust
pub trait TerminalEmulator: Send {
    fn process(&mut self, bytes: &[u8]) -> TerminalDelta;
    fn resize(&mut self, cols: u16, rows: u16);
    fn screen(&self) -> Screen;
    fn modes(&self) -> TerminalModes;
    fn replies_to_send(&mut self) -> Vec<Vec<u8>>;
}
```

`TerminalDelta` should include changed rows, cursor changes, title changes, mode changes, semantic annotations, bells, and parse warnings. The PTY read loop can then capture frames only when relevant deltas occur.

**Terminal sequences to support before 1.0**

- CSI cursor movement and erase sequences.
- SGR 16/256/truecolor styling.
- Alternate screen.
- Bracketed paste.
- Application cursor and keypad.
- Mouse tracking modes, especially SGR 1006.
- Title/icon OSC.
- OSC 8 hyperlinks as metadata.
- Terminal query responses: device status report, cursor position report, primary/secondary DA.
- Private Tuiwright OSC semantic annotations.

### 7.4 Milestone 3: deterministic renderer

The renderer must become a serious terminal renderer, not just a convenient fontdue demo.

**Required fidelity features**

- Font metrics: baseline, ascender, descender, line gap.
- Grapheme clusters and combining marks.
- Unicode width, East Asian wide characters, zero-width joiners.
- Emoji fallback and color emoji strategy.
- Bold, dim, italic, underline, double underline, curly underline, strikethrough.
- Inverse, hidden, blink policy.
- Cursor shapes: block, bar, underline, blinking disabled for screenshots.
- Selection and search highlights.
- Terminal theme config: default fg/bg and indexed palette.
- Cropping by locator rect.
- Pixel-diff masks and redaction.

**Renderer architecture**

```text
Screen cells
  -> GlyphLayout rows
  -> PaintCommands { rects, glyph runs, decorations, cursor }
  -> Backend
       - PNG raster backend
       - SVG backend
       - Diff overlay backend
```

The PNG backend can remain 100% Rust with `cosmic-text`/`swash` or a similar text stack plus `tiny-skia`/`image`. The MVP `fontdue` renderer is useful, but production should shape text properly.

### 7.5 Milestone 4: screenshots and visual assertions

**API**

```rust
expect(&page).to_have_screenshot("main-screen", ScreenshotOptions {
    mask_text: vec![Regex::new(r"\\d{2}:\\d{2}:\\d{2}")?],
    mask_rects: vec![Rect::new(70, 0, 10, 1)],
    max_pixel_diff_ratio: 0.001,
    max_cell_diff_count: 0,
    ..Default::default()
})?;
```

**Artifacts on failure**

```text
actual.png
expected.png
diff.png
actual.txt
expected.txt
cell-diff.txt
trace/
```

Cell diffs should be primary for TUI screenshots because they are more explainable than pixel diffs:

```text
row 10 col 14
expected: "Saved" fg=#00ff00 bg=#000000 bold=false
actual:   "Error" fg=#ff0000 bg=#000000 bold=true
```

### 7.6 Milestone 5: GIF/video recorder

**Recording modes**

```rust
page.recording().mode(RecordingMode::Off);
page.recording().mode(RecordingMode::On);
page.recording().mode(RecordingMode::RetainOnFailure);
page.recording().mode(RecordingMode::OnFirstRetry);
```

**Frame selection**

- Capture after actions.
- Capture after output bursts.
- Capture final screen.
- Cap to `max_fps`.
- Coalesce repeated spinner frames unless specifically requested.
- Preserve action moments with subtitles and pointer highlights.

**Web compression**

- Max width/height.
- FPS cap.
- Palette quality.
- Drop duplicate frames.
- Prefer short frame durations only when the screen actually changed.
- Add “poster frame” PNG for docs pages.

**License-aware encoder strategy**

```rust
pub trait AnimationEncoder {
    fn encode(&self, frames: &[RenderedFrame], output: &Path) -> Result<()>;
}

pub struct PermissiveGifEncoder;  // lower quality, broad license compatibility
pub struct GifskiEncoder;         // optional high quality, feature-gated
pub struct ApngEncoder;           // better quality than GIF, optional artifact
pub struct WebpEncoder;           // smaller web artifact where acceptable
```

### 7.7 Milestone 6: semantic annotations

Black-box testing is essential, but semantic locators make large TUI suites maintainable. The helper crate should be tiny and optional.

#### 7.7.1 Semantic marker format

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SemanticNode {
    pub test_id: Option<String>,
    pub role: Option<String>,
    pub name: Option<String>,
    pub label: Option<String>,
    pub rect: Rect,
    pub disabled: bool,
    pub focused: bool,
}

pub fn osc_marker(node: &SemanticNode) -> String {
    let json = serde_json::to_string(node).expect("semantic node serializes");
    format!("\x1b]777;TUIWRIGHT;{}\x07", json)
}
```

#### 7.7.2 Ratatui integration concept

```rust
use ratatui::{layout::Rect as RatRect, Frame};

pub struct SemanticFrame<'a, 'b> {
    frame: &'a mut Frame<'b>,
    nodes: Vec<SemanticNode>,
}

impl<'a, 'b> SemanticFrame<'a, 'b> {
    pub fn button(&mut self, rect: RatRect, name: impl Into<String>, test_id: impl Into<String>) {
        self.nodes.push(SemanticNode {
            test_id: Some(test_id.into()),
            role: Some("button".into()),
            name: Some(name.into()),
            label: None,
            rect: Rect::new(rect.x, rect.y, rect.width, rect.height),
            disabled: false,
            focused: false,
        });
    }

    pub fn flush_to_stdout_if_enabled(&self) {
        if std::env::var_os("TUIWRIGHT").is_none() {
            return;
        }
        for node in &self.nodes {
            print!("{}", osc_marker(node));
        }
    }
}
```

The production Ratatui helper should avoid `print!` from inside render paths when apps use buffered backends. Better options:

- Provide a `SemanticBackend<B>` wrapper that injects markers into the same output stream as the terminal backend.
- Provide a side-channel JSON socket/file when terminal OSC injection is undesirable.
- Allow apps to register stable IDs in their own component tree and emit one marker batch after each draw.

#### 7.7.3 Locator syntax

```rust
page.get_by_test_id("save-button").click()?;
page.get_by_role(Role::Button).name("Save").click()?;
page.get_by_label("User name").fill("ada")?;
```

### 7.8 Milestone 7: codegen

Codegen should proxy a real session:

```text
user terminal <-> tuiwright interactive proxy <-> PTY <-> app
```

The proxy records:

- Keystrokes.
- Mouse clicks.
- Wait points inferred from output quiescence.
- Text locators near click coordinates.
- Semantic locators when annotations exist.
- Screenshots at meaningful milestones.

Generated Rust example:

```rust
#[test]
fn generated_flow() -> anyhow::Result<()> {
    let page = tuiwright::spawn(env!("CARGO_BIN_EXE_my_tui"))
        .size(100, 30)
        .record(true)
        .start()?;

    page.wait_for_text("Ready")?;
    page.get_by_text("Settings").click()?;
    page.press(Key::Down)?;
    page.type_text("ada")?;
    expect(page.get_by_text("Saved")).to_be_visible()?;
    Ok(())
}
```

Selector generation priority:

1. Semantic `test_id`.
2. Semantic role + accessible name.
3. Exact visible text.
4. Regex around stable text.
5. Region/cell fallback with warning.

**Report features**

- Test list with pass/fail/flaky status.
- Per-test duration and retry count.
- Failure screenshot and final screen.
- Open trace link.
- Open GIF link.
- Copy reproduction command.
- CI-friendly JUnit and JSON output.

**Trace viewer panels**

## 9. CI recipe

A robust CI setup should:

```yaml
name: tuiwright
on: [push, pull_request]
jobs:
  tui:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - name: Install deterministic font
        run: sudo apt-get update && sudo apt-get install -y fonts-dejavu-core
      - name: Run TUI tests
        env:
          TUIWRIGHT_FONT: /usr/share/fonts/truetype/dejavu/DejaVuSansMono.ttf
          TERM: xterm-256color
          COLORTERM: truecolor
          LC_ALL: C.UTF-8
          TZ: UTC
        run: cargo test --workspace
      - name: Upload artifacts on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: tuiwright-artifacts
          path: target/tuiwright
```

Best practices:

- Pin terminal size in every test.
- Pin font path.
- Disable animations or set deterministic tick rate under `TUIWRIGHT=1`.
- Redact clocks, durations, random IDs, temp paths, and hashes.
- Treat screenshots as product-level tests; keep most tests text/cell-based.
- Store traces only on failure or retry to reduce CI storage.

## 10. Anticipated problems and mitigations

### 10.1 “Equivalent to a user terminal” is not one terminal

Different terminals render fonts, Unicode, emoji, cursor style, line height, ambiguous-width characters, and ligatures differently.

**Mitigation**

- Define a default deterministic Tuiwright terminal profile.
- Let users choose profiles: `xterm`, `wezterm`, `alacritty`, `windows-terminal`.
- Use cell-level assertions for logic and PNG screenshots for product-review baselines.
- Make font and metrics explicit.

### 10.2 Unicode is hard

Combining marks, CJK wide characters, emoji sequences, and zero-width joiners can break naïve cell-to-pixel rendering.

**Mitigation**

- Store both raw cell text and display width.
- Use a real shaping stack for production rendering.
- Add a Unicode torture-test fixture.
- Provide `unicode_mode = strict | compatibility`.

### 10.3 PTY behavior differs on Windows

ConPTY differs from POSIX PTYs in process groups, signals, resize behavior, and control events.

**Mitigation**

- Keep PTY behind a trait.
- Maintain Windows-specific integration tests.
- Provide `page.interrupt()` abstraction instead of exposing signals.
- Normalize CRLF in text snapshots.

### 10.4 Terminal replies can deadlock apps

Some apps query the terminal for cursor position, device attributes, or color support. If the emulator does not reply, the app may wait forever.

**Mitigation**

- Detect common queries in the output stream.
- Respond with xterm-compatible answers.
- Add trace warnings for unhandled queries.
- Allow profile-specific replies.

### 10.5 Mouse bytes can corrupt input

Real terminals only send mouse sequences when the app enables mouse tracking. If a test sends mouse bytes to an app that did not enable mouse mode, the app may interpret them as keyboard input.

**Mitigation**

- Track mouse mode from terminal model.
- In strict mode, fail if mouse is disabled.
- In permissive mode, warn in trace.

### 10.6 Flaky animations and spinners

Spinners and clocks make screenshots unstable.

**Mitigation**

- Encourage apps to check `TUIWRIGHT=1` and freeze animations.
- Provide redactions and masks.
- Add `page.wait_for_stable_screen(stable_for)`.
- Prefer event-driven waits over sleeps.

### 10.7 Hidden semantic markers could leak

If annotations are emitted outside tests, users may see strange OSC sequences in logs or unsupported terminals.

**Mitigation**

- Only emit when `TUIWRIGHT=1`.
- Keep marker parser tolerant.
- Allow a non-terminal side channel for semantic trees.

### 10.8 GIF encoder licensing

High-quality GIF encoding in Rust may bring a copyleft license depending on the chosen crate.

**Mitigation**

- Keep GIF encoding behind a feature.
- Offer a permissive fallback encoder.
- Document license implications clearly.
- Support APNG/WebP as alternatives.

### 10.9 Font licensing and reproducibility

Bundling a font may create license obligations; relying on system fonts causes drift.

**Mitigation**

- Do not ship font files in the framework.
- Require users to point at a font path for snapshot tests.
- Recommend they commit an appropriately licensed font to their own test assets after reviewing its license.

### 10.10 Trace artifacts may contain secrets

Terminal sessions can include tokens, passwords, file paths, or private data.

**Mitigation**

- Redact input values by default for `fill_secret` / `paste_secret`.
- Support trace redaction filters.
- Allow env allowlists instead of dumping all env vars.
- Store traces only on failure by default.

### 10.11 Test runner integration

Rust teams use `cargo test`, `cargo nextest`, custom harnesses, and CI-specific reporters.

**Mitigation**

- Keep the main API as ordinary Rust functions that work in any test harness.
- Add optional macros later.
- Emit JUnit/JSON from the CLI.

### 10.12 Performance

Rendering full PNG frames for every output burst can be expensive.

**Mitigation**

- Record cell snapshots first, render artifacts lazily on failure.
- Deduplicate frames by cell hash.
- Capture at action boundaries and capped FPS.
- Use row-level deltas in the terminal model.

### 10.13 Terminal graphics protocols

Modern TUIs may use Sixel, iTerm2 images, or Kitty graphics.

**Mitigation**

- V1: record unsupported graphics sequences and warn.
- V2: implement metadata and placeholders.
- V3: render protocols into screenshot artifacts where feasible.

## 13. What “incredible” looks like at 1.0

A 1.0-quality Tuiwright should let a team do this confidently:

```rust
#[tuiwright::test(cols = 120, rows = 36, record = "retain-on-failure")]
fn admin_can_rotate_api_key(page: Page) -> anyhow::Result<()> {
    page.goto_bin(env!("CARGO_BIN_EXE_admin_tui"))?;

    page.get_by_role(Role::MenuItem, "Secrets").click()?;
    page.get_by_test_id("api-key-row-prod").press(Key::Enter)?;
    page.get_by_role(Role::Button, "Rotate").click()?;
    page.get_by_label("Confirm rotation").fill_secret("rotate prod key")?;
    page.get_by_role(Role::Button, "Confirm").click()?;

    expect(page.get_by_text("Key rotated")).to_be_visible()?;
    expect(page.get_by_test_id("api-key-row-prod"))
        .to_contain_text("Last rotated: <redacted>")?;

    expect(&page).to_have_screenshot("secrets/rotated", ScreenshotOptions::default()
        .mask_regex(r"sk_live_[a-zA-Z0-9]+")
        .mask_regex(r"\\d{4}-\\d{2}-\\d{2} \\d{2}:\\d{2}:\\d{2}"))?;

    Ok(())
}
```

On failure, CI should show:

- The failing assertion.
- The final terminal screen as text.
- `failure.png` with locator highlight.
- `flow.gif` showing what happened.
- `trace/index.html` for time-travel debugging.
- Raw input/output bytes if needed.
