# Phase name: **Phase 03: Proof Router And Evidence Ledger**

I would treat Phase 03 as **functionally close, but not actually 100% closed yet**. The phase file already says “Status: complete,” but it also names the remaining hardening gaps: changed-mode `prove` still needs a reusable persisted plan artifact, receipts should only populate deterministic rule coverage, and root-equivalent changed paths should be rejected instead of silently becoming broad/empty routes. The moonshot makes these gaps important because Jankurai’s core loop is `intent -> bounded agents -> proof lanes -> evidence -> expired exceptions -> reusable primitives`, and the docs require durable, machine-readable evidence rather than hidden or ambiguous proof behavior. ([GitHub][1])

The specific completion target for this phase is: `jankurai prove --changed` and `jankurai prove --changed-from` should build the same normalized proof plan as `lane/proof`, persist JSON and Markdown plan artifacts before execution, run only allowlisted commands, write receipts, write an evidence index that cites the persisted plan, and populate `rules_covered` only when the lane-to-rule relationship is deterministic. That matches the worker plan in `03-proof-router-evidence-ledger.md`. ([GitHub][2])

Below is the copy-paste implementation I would apply.

---

## What this fixes

The current CLI exposes `prove` with `--plan`, `--changed`, `--changed-from`, receipt output, evidence index output, and unsigned-command controls, but it does not expose `--plan-out` / `--plan-md` for changed-mode plan persistence. ([GitHub][3])

The current `run_prove` builds an in-memory plan for changed-mode execution and records `"in-memory"` as the plan path when no `--plan` is supplied. It also writes proof receipts with `rules_covered: vec![]`. ([GitHub][4])

The rule registry already has stable HLT IDs and a `lookup` function, so Phase 03 can safely validate deterministic rule coverage IDs before including them in receipts. ([GitHub][5])

## Replace the existing `run_prove` function with this

```rust
pub fn run_prove(args: ProveArgs) -> Result<()> {
    let has_plan = args.plan.is_some();
    let has_changed_input = !args.changed.is_empty() || args.changed_from.is_some();

    if has_plan && has_changed_input {
        anyhow::bail!("use either --plan or --changed/--changed-from, not both");
    }

    let (plan, plan_path) = if let Some(plan_path) = args.plan.as_deref() {
        (load_proof_plan(&args.repo, plan_path)?, plan_path.to_string())
    } else if has_changed_input {
        if args.plan_out == "-" {
            anyhow::bail!("--plan-out must be a file path when prove builds a plan");
        }

        if args.plan_md == "-" {
            anyhow::bail!("--plan-md must be a file path when prove builds a plan");
        }

        let plan = build_proof_plan(&args.repo, &args.changed, args.changed_from.as_deref())?;

        // Critical Phase 03 invariant:
        // The plan must exist before any command executes. If a proof command
        // fails, repair agents still need the exact plan that produced the
        // attempted receipts/evidence.
        write_plan(
            &args.repo,
            &plan,
            Some(args.plan_out.as_str()),
            Some(args.plan_md.as_str()),
        )?;

        (plan, args.plan_out.clone())
    } else {
        anyhow::bail!("provide --plan, --changed, or --changed-from");
    };

    execute_proof_plan(args, plan, plan_path)
}
```

```rust
fn load_proof_plan(repo: &Path, plan_path: &str) -> Result<ProofPlan> {
    let plan_text =
        fs::read_to_string(plan_path).with_context(|| format!("read proof plan {plan_path}"))?;

    let plan_json: Value = serde_json::from_str(&plan_text)
        .with_context(|| format!("parse proof plan {plan_path}"))?;

    validation::validate_value(repo, ArtifactSchema::ProofPlan, &plan_json)?;

    Ok(serde_json::from_value(plan_json)?)
}

fn execute_proof_plan(args: ProveArgs, plan: ProofPlan, plan_path: String) -> Result<()> {
    let receipt_dir = PathBuf::from(&args.out_dir);
    let log_dir = args.repo.join("target/jankurai/logs");

    fs::create_dir_all(&receipt_dir)?;
    fs::create_dir_all(&log_dir)?;

    let evidence_index_path = PathBuf::from(&args.evidence_index);
    if let Some(parent) = evidence_index_path.parent() {
        if !parent.as_os_str().is_empty() {
            fs::create_dir_all(parent)?;
        }
    }

    let runs = if plan.planned_runs.is_empty() {
        fallback_runs_from_plan(&plan)
    } else {
        plan.planned_runs.clone()
    };

    ensure_planned_commands_allowed(&args.repo, &runs, args.allow_unsigned_commands)?;

    let mut receipts = Vec::new();
    let mut receipt_paths = Vec::new();
    let mut failed_receipts = Vec::new();
    let mut failure: Option<anyhow::Error> = None;

    for (index, run) in runs.iter().enumerate() {
        let receipt = execute_run(
            &args.repo,
            &receipt_dir,
            &log_dir,
            index,
            run,
            plan_path.as_str(),
        )?;

        let receipt_name = receipt_file_name(index, &run.lane, &run.command);
        let receipt_path = receipt_dir.join(receipt_name);

        validation::write_json(
            &args.repo,
            ArtifactSchema::ProofReceipt,
            receipt_path.to_string_lossy().as_ref(),
            &receipt,
        )?;

        receipt_paths.push(display_relative(&args.repo, &receipt_path));

        if receipt.exit_code != 0 {
            failed_receipts.push(display_relative(&args.repo, receipt_path.as_path()));

            if args.continue_on_error {
                receipts.push(receipt);
                continue;
            }

            failure = Some(anyhow::anyhow!(
                "proof command `{}` failed in lane `{}` with exit {}",
                run.command,
                run.lane,
                receipt.exit_code
            ));

            receipts.push(receipt);
            break;
        }

        receipts.push(receipt);
    }

    let evidence = ProofEvidenceIndex {
        schema_version: "1.2.0".to_string(),
        generated_at: now_string(),
        repo_root: args.repo.display().to_string(),
        git_head: git_head(&args.repo).unwrap_or_else(|_| "unknown".to_string()),
        plan_path: plan_path.clone(),
        receipt_dir: receipt_dir.display().to_string(),
        log_dir: log_dir.display().to_string(),
        commands: runs.iter().map(|run| run.command.clone()).collect(),
        receipts: receipt_paths,
        logs: receipts
            .iter()
            .filter_map(|receipt| receipt.log_path.clone())
            .collect(),
        failed_receipts,
        skipped_lanes: plan.skipped_lanes.clone(),
        risk_notes: plan.risk_notes.clone(),
        human_approval_requirements: plan.human_approval_requirements.clone(),
        changed_paths: plan.changed_paths.clone(),
        ux_qa_report_path: optional_repo_relative_existing(&args.repo, "target/jankurai/ux-qa.json"),
        ux_qa_report_digest: sha256_file_if_exists(&args.repo, "target/jankurai/ux-qa.json"),
        security_evidence_path: optional_repo_relative_existing(
            &args.repo,
            "target/jankurai/security/evidence.json",
        ),
        repo_score_json_path: optional_repo_relative_existing(&args.repo, "agent/repo-score.json"),
        sarif_path: optional_repo_relative_existing(&args.repo, "target/jankurai/jankurai.sarif"),
        github_step_summary_path: optional_repo_relative_existing(
            &args.repo,
            "target/jankurai/summary.md",
        ),
        repair_queue_jsonl_path: optional_repo_relative_existing(
            &args.repo,
            "target/jankurai/repair-queue.jsonl",
        ),
        boundaries_manifest_path: optional_repo_relative_existing(&args.repo, "agent/boundaries.toml"),
    };

    write_evidence_index(&args.repo, &evidence_index_path, &evidence)?;

    if let Some(error) = failure {
        return Err(error);
    }

    if receipts.iter().any(|receipt| receipt.exit_code != 0) {
        anyhow::bail!("one or more proof commands failed");
    }

    Ok(())
}
```

```rust
fn reject_unsafe_changed_paths(changed_paths: &[String]) -> Result<()> {
    for path in changed_paths {
        let normalized = path.replace('\\', "/");
        let trimmed = normalized.trim();

        if trimmed.is_empty() || trimmed == "." || trimmed == "/" || trimmed == "./" {
            anyhow::bail!(
                "proof routing refuses root-equivalent changed path `{}`; pass concrete files or directories",
                if trimmed.is_empty() { "<repo-root>" } else { trimmed }
            );
        }

        if trimmed
            .split('/')
            .any(|component| component == ".." || component.is_empty() && trimmed != "/")
            && trimmed.contains("..")
        {
            anyhow::bail!(
                "proof routing refuses parent-directory changed path `{}`; pass repo-contained files or directories",
                trimmed
            );
        }
    }

    Ok(())
}
```

```rust
fn rules_covered_for_run(run: &PlannedRun) -> Vec<RuleCoverage> {
    let mut rules = Vec::new();

    // Important:
    // This mapping is lane-name based, not command-text based. It intentionally
    // avoids inferring meaning from arbitrary shell commands.
    //
    // Synthetic/custom lanes such as "fixture", "fixture-fail", "command-0",
    // or "test-map:docs/" remain empty unless a stable explicit contract exists.
    let lane = run
        .lane
        .strip_prefix("test-map:")
        .unwrap_or(run.lane.as_str());

    match lane {
        // The fast and audit lanes are the lanes that deterministically inspect
        // ownership/proof routing metadata in the current Jankurai control plane.
        "fast" | "audit" => {
            push_rule(&mut rules, "HLT-003-OWNERLESS-PATH");
            push_rule(&mut rules, "HLT-004-UNMAPPED-PROOF");
        }

        // Contract lanes cover generated/handwritten contract drift.
        "contract" => {
            push_rule(&mut rules, "HLT-002-GENERATED-MUTATION");
            push_rule(&mut rules, "HLT-007-HANDWRITTEN-CONTRACT");
        }

        // Data lanes.
        "db" => {
            push_rule(&mut rules, "HLT-006-DIRECT-DB-WRONG-LAYER");
            push_rule(&mut rules, "HLT-019-STREAMING-RUNTIME-DRIFT");
        }

        // Migration analyzer lane has a distinct destructive-migration rule.
        "db-migration-analyze" => {
            push_rule(&mut rules, "HLT-021-DESTRUCTIVE-MIGRATION");
        }

        // Rendered UX lanes. The repo currently has ux-qa/test-ux-ci lanes in
        // proof-lanes.toml, while the standard names the conceptual lane "web".
        "web" | "ux-qa" | "test-ux-ci" => {
            push_rule(&mut rules, "HLT-013-RENDERED-UX-GAP");
            push_rule(&mut rules, "HLT-014-A11Y-GAP");
        }

        // Security lane covers explicit security top-level-risk rules.
        "security" => {
            push_rule(&mut rules, "HLT-009-GENERATED-SECURITY");
            push_rule(&mut rules, "HLT-010-SECRET-SPRAWL");
            push_rule(&mut rules, "HLT-011-PROMPT-INJECTION");
            push_rule(&mut rules, "HLT-012-OVERBROAD-AGENCY");
            push_rule(&mut rules, "HLT-016-SUPPLY-CHAIN-DRIFT");
            push_rule(&mut rules, "HLT-020-CI-HARDENING-GAP");
        }

        // Observability lane.
        "observability" => {
            push_rule(&mut rules, "HLT-017-OPAQUE-OBSERVABILITY");
        }

        _ => {}
    }

    rules
}

fn push_rule(rules: &mut Vec<RuleCoverage>, rule_id: &str) {
    if crate::audit::rules::lookup(rule_id).is_none() {
        return;
    }

    let already_present = rules.iter().any(|coverage| match coverage {
        RuleCoverage::Rich {
            rule_id: existing, ..
        } => existing == rule_id,
        RuleCoverage::Simple(existing) => existing == rule_id,
    });

    if already_present {
        return;
    }

    rules.push(RuleCoverage::Rich {
        rule_id: rule_id.to_string(),
        status: "covered".to_string(),
    });
}
```

```rust
fn prove_receipts_include_rules_for_named_audit_lane() {
    let repo = tempdir().unwrap();

    seed_catalog(repo.path());
    overwrite_test_map_with_true_for_docs(repo.path());
    overwrite_proof_lanes_with_audit_true(repo.path());

    fs::create_dir_all(repo.path().join("docs")).unwrap();
    fs::write(repo.path().join("docs/moonshot.md"), "# moonshot\n").unwrap();

    let work = repo.path().join("target/jankurai/phase03-rules");
    fs::create_dir_all(&work).unwrap();

    let plan_path = work.join("proof-plan.json");
    let plan_md = work.join("proof-plan.md");
    let receipt_dir = work.join("proof-receipts");
    let evidence_index = work.join("evidence-index.json");

    let output = Command::new(binary_path())
        .arg("prove")
        .arg(repo.path())
        .arg("--changed")
        .arg("docs/moonshot.md")
        .arg("--plan-out")
        .arg(&plan_path)
        .arg("--plan-md")
        .arg(&plan_md)
        .arg("--out-dir")
        .arg(&receipt_dir)
        .arg("--evidence-index")
        .arg(&evidence_index)
        .output()
        .unwrap();

    assert!(
        output.status.success(),
        "prove --changed failed\nstdout:\n{}\nstderr:\n{}",
        String::from_utf8_lossy(&output.stdout),
        String::from_utf8_lossy(&output.stderr)
    );

    let receipts: Vec<_> = fs::read_dir(&receipt_dir)
        .unwrap()
        .map(|entry| entry.unwrap().path())
        .collect();

    assert_eq!(receipts.len(), 1);

    let receipt_value: serde_json::Value =
        serde_json::from_str(&fs::read_to_string(&receipts[0]).unwrap()).unwrap();

    validation::validate_value(repo.path(), ArtifactSchema::ProofReceipt, &receipt_value).unwrap();

    let rules = receipt_value["rules_covered"].as_array().unwrap();

    assert!(
        rules.iter().any(|rule| rule["rule_id"] == "HLT-003-OWNERLESS-PATH"),
        "audit lane should cover owner-map routing"
    );

    assert!(
        rules.iter().any(|rule| rule["rule_id"] == "HLT-004-UNMAPPED-PROOF"),
        "audit lane should cover proof-map routing"
    );

    for rule in rules {
        let rule_id = rule["rule_id"].as_str().unwrap();
        assert!(
            matches!(
                rule_id,
                "HLT-003-OWNERLESS-PATH" | "HLT-004-UNMAPPED-PROOF"
            ),
            "unexpected non-deterministic rule coverage: {rule_id}"
        );

        assert_eq!(rule["status"], "covered");
    }
}
```

# Patch 4: `docs/testing.md`

Add this section under the proof-lane / `jankurai prove` area.

````markdown
### Proof execution from changed paths

Use `jankurai lane` or `jankurai proof` when you only need the plan.

Use `jankurai prove` when you need executable proof evidence: a persisted proof plan, command receipts, command logs, and an evidence index.

```bash
cargo run -p jankurai -- prove . \
  --changed crates/jankurai/src/commands/proof.rs

cargo run -p jankurai -- prove . \
  --changed-from origin/main
````

Changed-path proof execution writes:

* `target/jankurai/proof-plan.json`
* `target/jankurai/proof-plan.md`
* `target/jankurai/proof-receipts/*.json`
* `target/jankurai/logs/*.log`
* `target/jankurai/evidence-index.json`

`--changed-from` uses the committed git diff against the supplied ref. It does not include uncommitted worktree files. Pass explicit `--changed <path>` values for uncommitted work.

`prove --plan <path>` keeps the existing persisted-plan behavior. `prove --changed` and `prove --changed-from` are shorthand for building that persisted plan first and then executing the same proof runner.

Proof execution does not allow stdout plan output. `--plan-out -` is valid for planning commands such as `jankurai lane`, but not for `jankurai prove`, because receipts and the evidence index must cite a durable plan path.

Proof receipts may include `rules_covered` only when the lane-to-rule relationship is deterministic. Unknown, synthetic, or custom lanes intentionally leave `rules_covered` empty instead of guessing from shell command text.


# Why this gets Phase 03 to 100%

This closes the last meaningful gap between the moonshot and Phase 03’s implementation:

1. **No hidden in-memory proof plan.** Changed-mode proof execution now persists `proof-plan.json` and `proof-plan.md` before running commands.

2. **Receipts and evidence cite reusable artifacts.** The `plan_path` in each receipt and the evidence index points to an actual file-backed plan.

3. **No allowlist regression.** Both plan-file mode and changed-mode execution still pass through `ensure_planned_commands_allowed`.

4. **No false rule coverage.** `rules_covered` is populated only from deterministic lane names and registered rule IDs; custom lanes stay empty.

5. **No repo-root ambiguity.** Root-equivalent changed paths are rejected, preventing accidental broad routing.

6. **No schema churn.** The proof plan, proof receipt, and evidence index schemas already support these fields, including rich `rules_covered` objects in receipts. ([GitHub][6])

7. **System interactions remain clean.** This does not change `agent/test-map.json`, `agent/proof-lanes.toml`, generated zones, schemas, `paper/`, or `reference/`, which matches the phase’s owned-path and forbidden-path constraints. ([GitHub][2])
