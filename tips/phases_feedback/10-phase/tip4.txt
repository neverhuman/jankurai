## Phase name: Phase 10 — Reuse Registry Certified Cells

I would finish and harden **Phase 10** by adding the next certified cell in the documented order: **`auth-session`**.

This matches the repo’s moonshot loop: `intent -> bounded agents -> proof lanes -> evidence -> expired exceptions -> reusable primitives`, where repeated fixes should collapse into reusable cells/templates instead of copy-paste folklore. ([GitHub][1]) Phase 10 already has the registry/cell command surface, three certified cells, dry-run-only install semantics, and `cell --mode prove` evidence emission; its own follow-up says the next registry cell is **auth/session shell**. ([GitHub][2]) The Phase 00 router also explicitly prioritizes “Phase 10 auth/session or cell hardening” before Phase 12 and Phase 13. ([GitHub][3])

Below is a **single pasteable implementation script**. It adds the `auth-session` certified cell, hardens certification lifecycle honesty, expands the smoke tests, updates the Phase 10 receipt/log, and runs the correct proof/validation commands.

I have **not executed this in your checkout**; the script itself runs validation and writes the evidence artifacts.

### Why this finishes Phase 10 instead of drifting into Phase 13

This keeps Phase 10 inside the intended cell-registry boundary. The installer remains **dry-run-only** and **never-overwrite**, which matches the existing Phase 10 contract. ([GitHub][2]) It does not create branches, mutate real projects, or add provider-backed auth behavior, because that would collide with the Phase 13 safety model, where real repo mutation and PR creation are still deferred behind stronger gates. ([GitHub][4])

The important hardening addition is not just “one more manifest.” It makes certification status evidence-bound: if a built-in cell’s required paths or proof lanes are missing, it can no longer keep lifecycle `certified`; it becomes `experimental` + `candidate`. That directly supports the moonshot’s demand that evidence be durable and machine-readable instead of a doc label. ([GitHub][1])
