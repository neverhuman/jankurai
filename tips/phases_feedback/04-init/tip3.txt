**Phase name:** `04 Init Profiles And Golden Repos`

I’m treating this as the completion package for `tips/phases/04-init-profiles-golden-repos.md`.

The key issue is that Phase 04’s written objective is broader than the current implementation: the phase says init should become a profile-driven repo generator, with public profiles including `rust-api`, `react-web`, `rust-ts-postgres`, `b2b-saas`, `ai-product`, `regulated-saas`, and `migration-target`; it also requires golden fixtures for idempotency and generated-adapter refresh. ([GitHub][1]) The repo currently exposes only four bundled profiles in `profiles.rs`, and `docs/install.md` still says only `rust-ts-postgres` is bundled. ([GitHub][2])

The solution below finishes the phase by making init profile coverage match the phase contract, making generated repos align with the moonshot standard, and adding golden tests that prove the behavior. The moonshot requires a repo where agents can infer ownership, safe commands, proof lanes, generated-vs-source boundaries, and exception/evidence flows without tribal knowledge. ([GitHub][3]) The generated skeletons below intentionally avoid creating fake app package manifests because the existing doctor flags manifests without lockfiles; instead, they create explicit architecture slots and validation lanes that can mature in later phases without causing false-green CI. The existing agent-native standard also requires files like `AGENTS.md`, `agent/JANKURAI_STANDARD.md`, owner/test maps, generated zones, proof lanes, score artifacts, boundaries, and security policy, which this patch preserves and expands. ([GitHub][4])

So this is a copy-paste implementation package designed from the live repo contents, but it still needs to be run in your checkout with the validation commands at the bottom.

pub fn resolve_profile(repo: &Path, profile: &str) -> Result<ProfileManifest> {
    let trimmed = profile.trim();

    if let Some(id) = normalize_profile_id(trimmed) {
        let json = match id {
            "rust-ts-postgres" => RUST_TS_POSTGRES_JSON,
            "rust-api" => RUST_API_JSON,
            "react-web" => REACT_WEB_JSON,
            "b2b-saas" => B2B_SAAS_JSON,
            "ai-product" => AI_PRODUCT_JSON,
            "regulated-saas" => REGULATED_SAAS_JSON,
            "migration-target" => MIGRATION_TARGET_JSON,
            _ => unreachable!("normalize_profile_id returned unsupported id"),
        };
        return load_profile_manifest(json, true)
            .with_context(|| format!("failed to load bundled init profile `{id}`"));
    }

    if let Some(path) = profile_json_path(repo, trimmed) {
        return load_profile_file(repo, &path)
            .with_context(|| format!("failed to load init profile `{}`", path.display()));
    }

    bail!(
        "unknown init profile `{}`.\nsupported bundled profiles: {}\ncustom profiles: pass a repo-relative or absolute path to an init-profile JSON manifest",
        profile,
        supported_profile_summary()
    )
}

pub fn supported_profile_summary() -> String {
    SUPPORTED_PROFILE_IDS.join(", ")
}

fn load_profile_file(_repo: &Path, path: &Path) -> Result<ProfileManifest> {
    let text = fs::read_to_string(path)
        .with_context(|| format!("failed to read init profile `{}`", path.display()))?;
    load_profile_manifest(&text, false)
}

fn load_profile_manifest(text: &str, require_supported_id: bool) -> Result<ProfileManifest> {
    let value: serde_json::Value =
        serde_json::from_str(text).context("init profile is not valid JSON")?;

    validation::validate_value(ArtifactSchema::InitProfile, &value)
        .context("init profile failed schemas/init-profile.schema.json")?;

    let file: ProfileManifestFile =
        serde_json::from_value(value).context("failed to deserialize init profile")?;
    let manifest: ProfileManifest = file.into();

    validate_manifest_contract(&manifest, require_supported_id)?;

    Ok(manifest)
}

fn profile_json_path(repo: &Path, profile: &str) -> Option<PathBuf> {
    if profile.is_empty() {
        return None;
    }

    let candidate = Path::new(profile);

    if candidate.is_absolute() && candidate.is_file() {
        return Some(candidate.to_path_buf());
    }

    let repo_candidate = repo.join(candidate);
    if repo_candidate.is_file() {
        return Some(repo_candidate);
    }

    None
}

fn normalize_profile_id(profile: &str) -> Option<&'static str> {
    match profile {
        "" | "default" | "fullstack" | "rust-ts" | "rust-ts-postgres" => Some("rust-ts-postgres"),
        "rust" | "api" | "rust-api" => Some("rust-api"),
        "web" | "frontend" | "react" | "react-web" => Some("react-web"),
        "saas" | "b2b" | "b2b-saas" => Some("b2b-saas"),
        "ai" | "ml" | "ai-product" => Some("ai-product"),
        "regulated" | "compliance" | "regulated-saas" => Some("regulated-saas"),
        "migration" | "brownfield" | "migration-target" => Some("migration-target"),
        _ => None,
    }
}

fn validate_manifest_contract(manifest: &ProfileManifest, require_supported_id: bool) -> Result<()> {
    if require_supported_id && !SUPPORTED_PROFILE_IDS.contains(&manifest.id.as_str()) {
        bail!(
            "bundled init profile `{}` is not listed in SUPPORTED_PROFILE_IDS",
            manifest.id
        );
    }

    if manifest.id.trim().is_empty() {
        bail!("init profile id cannot be empty");
    }
    if manifest.display_name.trim().is_empty() {
        bail!("init profile displayName cannot be empty");
    }
    if manifest.target_stack_id.trim().is_empty() {
        bail!("init profile targetStackId cannot be empty");
    }

    ensure_list_has_no_duplicates("generatedPaths", &manifest.generated_paths)?;
    ensure_list_has_no_duplicates("requiredLanes", &manifest.required_lanes)?;
    ensure_list_has_no_duplicates("optionalLanes", &manifest.optional_lanes)?;
    ensure_list_has_no_duplicates("agentAdapters", &manifest.agent_adapters)?;
    ensure_list_has_no_duplicates("ciTemplates", &manifest.ci_templates)?;
    ensure_list_has_no_duplicates("docs", &manifest.docs)?;
    ensure_list_has_no_duplicates("securityControls", &manifest.security_controls)?;
    ensure_list_has_no_duplicates("uxControls", &manifest.ux_controls)?;
    ensure_list_has_no_duplicates("contractSystem", &manifest.contract_system)?;
    ensure_list_has_no_duplicates("dbPolicy", &manifest.db_policy)?;
    ensure_list_has_no_duplicates("validationCommands", &manifest.validation_commands)?;

    for path in &manifest.generated_paths {
        ensure_relative_template_path("generatedPaths", path)?;
    }

    ensure_category_subset("agentAdapters", &manifest.agent_adapters, &manifest.generated_paths)?;
    ensure_category_subset("ciTemplates", &manifest.ci_templates, &manifest.generated_paths)?;
    ensure_category_subset("docs", &manifest.docs, &manifest.generated_paths)?;
    ensure_category_subset(
        "securityControls",
        &manifest.security_controls,
        &manifest.generated_paths,
    )?;
    ensure_category_subset("uxControls", &manifest.ux_controls, &manifest.generated_paths)?;
    ensure_category_subset("contractSystem", &manifest.contract_system, &manifest.generated_paths)?;
    ensure_category_subset("dbPolicy", &manifest.db_policy, &manifest.generated_paths)?;

    let required: BTreeSet<&str> = manifest.required_lanes.iter().map(String::as_str).collect();
    for lane in &manifest.optional_lanes {
        if required.contains(lane.as_str()) {
            bail!("lane `{lane}` cannot appear in both requiredLanes and optionalLanes");
        }
    }

    ensure_non_empty_commands(&manifest.validation_commands)?;

    Ok(())
}

fn ensure_list_has_no_duplicates(name: &str, values: &[String]) -> Result<()> {
    let mut seen = BTreeSet::new();

    for value in values {
        let trimmed = value.trim();
        if trimmed.is_empty() {
            bail!("{name} cannot contain empty values");
        }
        if !seen.insert(trimmed.to_owned()) {
            bail!("{name} contains duplicate value `{trimmed}`");
        }
    }

    Ok(())
}

fn ensure_relative_template_path(field: &str, path: &str) -> Result<()> {
    let path_ref = Path::new(path);

    if path_ref.is_absolute() {
        bail!("{field} path `{path}` must be relative");
    }

    for component in path_ref.components() {
        match component {
            Component::Normal(_) => {}
            Component::CurDir | Component::ParentDir | Component::RootDir | Component::Prefix(_) => {
                bail!("{field} path `{path}` must be a clean relative path")
            }
        }
    }

    if path.contains('\\') {
        bail!("{field} path `{path}` must use forward slashes");
    }

    Ok(())
}

fn ensure_category_subset(field: &str, values: &[String], generated_paths: &[String]) -> Result<()> {
    let generated: BTreeSet<&str> = generated_paths.iter().map(String::as_str).collect();

    for value in values {
        if !generated.contains(value.as_str()) {
            bail!("{field} path `{value}` must also be present in generatedPaths");
        }
    }

    Ok(())
}

fn ensure_non_empty_commands(commands: &[String]) -> Result<()> {
    for command in commands {
        if command.trim().is_empty() {
            bail!("validationCommands cannot contain empty commands");
        }
    }

    Ok(())
}

cat > docs/install.md <<'EOF'

# Install and initialize

## Install

From a checkout:

```bash
cargo install --path crates/jankurai --locked
```

Or run from the workspace:

```bash
cargo run -p jankurai -- --help
```

## Initialize a repository

```bash
jankurai init . --profile rust-ts-postgres
```

Use `--dry-run` first when initializing an existing repository:

```bash
jankurai init . --profile migration-target --dry-run
```

Write a machine-readable plan:

```bash
jankurai init . --profile regulated-saas --plan-json target/jankurai/init-plan.json --dry-run
```

Show a diff-style plan:

```bash
jankurai init . --profile b2b-saas --diff --dry-run
```

## Bundled profiles

| `rust-api`         | The repository is primarily a Rust API/service with contracts, database policy, and security lanes.     |
| `react-web`        | The repository is primarily a React/web frontend with UX and contract readiness lanes.                  |
| `rust-ts-postgres` | The repository is a full-stack Rust + TypeScript + Postgres product. This is also the default profile.  |
| `b2b-saas`         | The product needs tenant, org, auth, admin, privacy, and SaaS evidence slots.                           |
| `ai-product`       | The product includes AI, model, prompt, retrieval, or eval workflows.                                   |
| `regulated-saas`   | The product needs stronger compliance, data classification, backup/restore, and durable evidence slots. |
| `migration-target` | The repository is brownfield or being migrated into the Jankurai standard.                              |

* `default`, `fullstack`, `rust-ts` -> `rust-ts-postgres`
* `api`, `rust` -> `rust-api`
* `web`, `frontend`, `react` -> `react-web`
* `saas`, `b2b` -> `b2b-saas`
* `ai`, `ml` -> `ai-product`
* `regulated`, `compliance` -> `regulated-saas`
* `migration`, `brownfield` -> `migration-target`

## Local custom profiles

A custom profile is a JSON file that matches `schemas/init-profile.schema.json`.

```bash
jankurai init . --profile ./profiles/my-profile.json --dry-run
```

Custom profile paths may be absolute or repo-relative. Every path listed in profile categories such as `docs`, `securityControls`, `contractSystem`, or `dbPolicy` must also appear in `generatedPaths`.

## Safe re-run behavior

`jankurai init` is designed to be re-run.

* Missing files are created.
* Existing user-owned files are preserved.
* `AGENTS.md` and `agent/JANKURAI_STANDARD.md` use controlled merge behavior.
* Generated adapter files are not overwritten by default.
* Passing `--force-generated-adapters` refreshes only adapter files that contain the Jankurai generated-adapter marker.

Generated adapters include files such as:

* `CLAUDE.md`
* `GEMINI.md`
* `.cursor/rules/jankurai.mdc`
* `.claude/skills/jankurai/SKILL.md`
* `.agents/skills/jankurai/SKILL.md`
* `.github/instructions/*.md`

## After init

Run the baseline proof lanes:

```bash
jankurai audit . --json agent/repo-score.json --md agent/repo-score.md
jankurai doctor . --fail-on high
jankurai security run . --script tools/security-lane.sh --out target/jankurai/security/evidence.json
```

Optional helper:

```bash
just proof
```

## What init intentionally does not do

Init does not create fake application package manifests, dependency lockfiles, deployment targets, or product code. It creates the repo contract and architecture slots. Add real package manifests only when the lockfile, owner map, test map, and proof lanes are ready.
