**Phase name: Phase 04 — Init Profiles And Golden Repos**

I would treat Phase 04 as “not actually complete” until `jankurai init` is a real creation layer: manifests are strict, all promised profiles exist, generated repo scaffolds can run audit/doctor, existing repo adoption is safe, refreshable generated adapters are explicit, and tests prove idempotency plus profile coverage. That aligns with the repo’s stated loop of `intent -> bounded agents -> proof lanes -> evidence -> expired exceptions -> reusable primitives`, and with the README’s framing of Jankurai as a control plane for ownership, proof, generated zones, and audit-ready repair evidence. ([GitHub][1])

The current Phase 04 file says the objective is to turn `jankurai init` from a control-file installer into a profile-driven repo generator, and it names the exit state as a robust generator contract, one excellent default profile, and golden repo fixtures. It also lists follow-on gaps: additional profiles, deeper merge policy, and optional local profile loading. ([GitHub][2]) The repo has already advanced beyond the original first slice because `rust-api`, `react-web`, and `b2b-saas` manifests now exist alongside `rust-ts-postgres`, and tests cover some greenfield profile application. ([GitHub][3]) The patch below closes the remaining gap set for Phase 04: it adds the missing profile family, makes manifests stricter and more expressive, adds local profile loading, upgrades template content into a real scaffold, adds refresh markers for adapters, strengthens merge-marker/idempotency behavior, enriches init receipts, expands tests, and updates install/phase docs.

+/// Resolves a bundled profile or an explicit local JSON manifest.
+///
+/// Local profile forms:
+/// - `--profile file:agent/init-profiles/custom.json`
+/// - `--profile ./agent/init-profiles/custom.json`
+/// - `--profile local:custom` => `agent/init-profiles/custom.json`
+pub fn resolve_profile(repo: &Path, profile: &str) -> Result<ProfileManifest> {
+    if let Some(path) = resolve_profile_file(repo, profile) {
+        let json = fs::read_to_string(&path)
+            .with_context(|| format!("read init profile {}", path.display()))?;
+        return load_profile(repo, &json, path.display().to_string());
+    }
+
+    let normalized = normalize_profile_id(profile);
+    for bundled in BUNDLED_PROFILES {
+        if bundled.id == normalized {
+            return load_profile(
+                repo,
+                bundled.json,
+                format!("bundled:{}", bundled.id),
+            );
+        }
+    }
+
+    bail!(
+        "unknown init profile `{}`. supported: {} (plus aliases and explicit file:/local: profiles)",
+        profile,
+        supported_profile_list()
+    )
+}

+fn resolve_profile_file(repo: &Path, profile: &str) -> Option<PathBuf> {
+    if let Some(name) = profile.strip_prefix("local:") {
+        let file_name = if name.ends_with(".json") {
+            name.to_string()
+        } else {
+            format!("{name}.json")
+        };
+        return Some(repo.join("agent/init-profiles").join(file_name));
+    }
+
+    let raw = profile.strip_prefix("file:").unwrap_or(profile);
+    if !looks_like_path(raw) {
+        return None;
+    }
+    let path = PathBuf::from(raw);
+    Some(if path.is_absolute() { path } else { repo.join(path) })
+}

+fn looks_like_path(raw: &str) -> bool {
+    raw.ends_with(".json")
+        || raw.starts_with("./")
+        || raw.starts_with("../")
+        || raw.starts_with('/')
+        || raw.contains('/')
+        || raw.contains('\\')
+}

+fn load_profile(repo: &Path, json_str: &str, source: String) -> Result<ProfileManifest> {
+    let value: serde_json::Value = serde_json::from_str(json_str)
+        .with_context(|| format!("parse init profile manifest from {source}"))?;
+    validation::validate_value(repo, ArtifactSchema::InitProfile, &value)
+        .with_context(|| format!("validate init profile manifest from {source}"))?;
+    let file: ProfileManifestFile = serde_json::from_value(value)
+        .with_context(|| format!("decode init profile manifest from {source}"))?;
+    let mut manifest: ProfileManifest = file.into();
+    manifest.source = source;
+    if manifest.refreshable_paths.is_empty() {
+        manifest.refreshable_paths = manifest.agent_adapters.clone();
+    }
+    validate_manifest_contract(&manifest)?;
+    Ok(manifest)
+}

+    validate_path_list("generatedPaths", &manifest.generated_paths)?;
+    validate_path_list("agentAdapters", &manifest.agent_adapters)?;
+    validate_path_list("ciTemplates", &manifest.ci_templates)?;
+    validate_path_list("docs", &manifest.docs)?;
+    validate_path_list("securityControls", &manifest.security_controls)?;
+    validate_path_list("uxControls", &manifest.ux_controls)?;
+    validate_path_list("contractSystem", &manifest.contract_system)?;
+    validate_path_list("dbPolicy", &manifest.db_policy)?;
+    validate_path_list("refreshablePaths", &manifest.refreshable_paths)?;
+    validate_path_list("mergePolicy.userOwnedPaths", &manifest.merge_policy.user_owned_paths)?;
+    validate_path_list("mergePolicy.neverOverwrite", &manifest.merge_policy.never_overwrite)?;
+
+    let generated: BTreeSet<_> = manifest.generated_paths.iter().cloned().collect();
+    ensure_subset(manifest, "agentAdapters", &manifest.agent_adapters, &generated)?;
+    ensure_subset(manifest, "ciTemplates", &manifest.ci_templates, &generated)?;
+    ensure_subset(manifest, "docs", &manifest.docs, &generated)?;
+    ensure_subset(
+        manifest,
+        "securityControls",
+        &manifest.security_controls,
+        &generated,
+    )?;
+    ensure_subset(manifest, "uxControls", &manifest.ux_controls, &generated)?;
+    ensure_subset(
+        manifest,
+        "contractSystem",
+        &manifest.contract_system,
+        &generated,
+    )?;
+    ensure_subset(manifest, "dbPolicy", &manifest.db_policy, &generated)?;
+    ensure_subset(
+        manifest,
+        "refreshablePaths",
+        &manifest.refreshable_paths,
+        &generated,
+    )?;
+
+    if manifest.validation_commands.is_empty() {
+        bail!("profile `{}` must declare validationCommands", manifest.id);
+    }
+    Ok(())
+}

+fn validate_path_list(field: &str, paths: &[String]) -> Result<()> {
+    let mut seen = BTreeSet::new();
+    for path in paths {
+        if path.trim().is_empty() {
+            bail!("{field} contains an empty path");
+        }
+        if path != &path.replace('\\', "/") {
+            bail!("{field} path `{path}` must use forward slashes");
+        }
+        let parsed = Path::new(path);
+        if parsed.is_absolute() {
+            bail!("{field} path `{path}` must be repo-relative");
+        }
+        for component in parsed.components() {
+            if matches!(
+                component,
+                Component::ParentDir | Component::RootDir | Component::Prefix(_)
+            ) {
+                bail!("{field} path `{path}` must not escape the repository");
+            }
+        }
+        if !seen.insert(path) {
+            bail!("{field} contains duplicate path `{path}`");
+        }
+    }
+    Ok(())
+}

+fn ensure_subset(
+    manifest: &ProfileManifest,
+    field: &str,

+                "profile `{}` declares `{}` in {} but it is absent from generatedPaths",
+                manifest.id,
+                path,
+                field
+            );
+        }
+    }
+    Ok(())

+# Installing jankurai
+
+## Init Profiles
+
+Bundled init profiles are JSON manifests validated against `schemas/init-profile.schema.json`.
+They are creation-layer contracts, not loose template lists. A profile declares generated paths, required and optional lanes, adapters, CI templates, docs, security controls, UX controls, contract/DB policy, validation commands, refreshable generated paths, and golden assertions.
+
+Supported bundled profiles:
+
+- `rust-ts-postgres` — default fullstack COLD scaffold: Rust API, TypeScript/React/Vite slot, Postgres policy, contracts, UX QA, security evidence, and agent control files.
+- `rust-api` — Rust API with contracts, DB policy, and security evidence; no web UX lane by default.
+- `react-web` — TypeScript/React/Vite web profile with UX QA and generated contract slots; no DB ownership.
+- `b2b-saas` — fullstack SaaS profile with auth/org/admin docs and SOC-ready evidence shell.
+- `ai-product` — bounded Python AI/eval profile with prompt/eval policy and no product-truth claim.
+- `regulated-saas` — stricter SaaS evidence shell with PII, backup/restore, incident-response, and compliance evidence slots.
+- `migration-target` — strangler-migration target with slice/equivalence/cutover evidence slots.
+
+Aliases include `default`, `fullstack`, `api`, `web`, `saas`, `ai`, `regulated`, and `migration`.
+
+## Dry-Run First
+
+```bash
+jankurai init \
+  --profile rust-ts-postgres \
+  --ide all \
+  --mode advisory \
+  --dry-run \
+  --plan-json target/jankurai/init-plan.json
+```
+
+The plan JSON contains the selected profile manifest and the planned action for every `generatedPaths` entry.
+Missing templates are a hard error at plan time.
+
+## Apply
+
+```bash
+jankurai init --profile rust-ts-postgres --ide all --mode advisory --yes
+jankurai doctor --fail-on high
+jankurai audit . --json agent/repo-score.json --md agent/repo-score.md
+```
+
+`init --yes` creates missing files and preserves existing user content.
+Paths listed in `mergePolicy.userOwnedPaths` receive a merge marker if they already exist and are not already Jankurai-controlled.
+Paths listed in `mergePolicy.neverOverwrite` are preserved.
+
+## Refresh Generated Adapters
+
+IDE adapters created by init include the `jankurai generated adapter` marker.
+They can be refreshed explicitly:
+
+```bash
+jankurai init --profile rust-ts-postgres --ide all --yes --force-generated-adapters
+```
+
+Files without the generated marker are treated as user-owned and are not overwritten.
+
+## Local Profiles
+
+Use explicit local manifests when adopting a special repo:
+
+```bash
+jankurai init --profile file:agent/init-profiles/internal.json --dry-run
+jankurai init --profile local:internal --dry-run
+```
+
+`local:internal` resolves to `agent/init-profiles/internal.json`.
+
+## Existing Repo Adoption
+
+For brownfield repos:
+
+```bash
+jankurai init --profile migration-target --dry-run --plan-json target/jankurai/init-plan.json
+jankurai init --profile migration-target --yes
+jankurai context-pack --task "adopt existing repo into jankurai control plane" --out target/jankurai/context-pack.json
+jankurai repair-plan --from agent/repo-score.json --out target/jankurai/repair-plan.json
+```
+
+The goal is bounded adoption: add the control plane, map ownership and proof, preserve user-owned files, and replace scaffold placeholders with real product intent before production.
