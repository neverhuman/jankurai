This is the right phase to finish because the moonshot requires explicit intent, bounded agent authority, ownership/proof mapping, declared generated outputs, durable evidence, expiring exceptions, and reusable primitives. The Phase 04 target is exactly the creation-layer version of that loop: a robust generator contract, an excellent default profile, and golden fixtures that prove generated repos are audit-ready. pub fn write_adapters(repo: &Path, ide: &str, force_generated: bool) -> Result<Vec<AdapterAction>> {
    let selected = selected_adapter_paths(ide);
    let mut actions = Vec::new();

    for template in crate::init::templates::TEMPLATES
        .iter()
        .filter(|template| selected.contains(&template.path))
    {
        let path = repo.join(template.path);
        if let Some(parent) = path.parent() {
            fs::create_dir_all(parent).with_context(|| format!("create {}", parent.display()))?;
        }

        if path.exists() {
            let text = fs::read_to_string(&path).unwrap_or_default();
            if force_generated && text.contains(GENERATED_MARKER) {
                fs::write(&path, template.body)
                    .with_context(|| format!("write {}", path.display()))?;
                actions.push(AdapterAction {
                    path: template.path.into(),
                    action: "overwrote-generated".into(),
                });
            } else {
                actions.push(AdapterAction {
                    path: template.path.into(),
                    action: "keep-existing".into(),
                });
            }
            continue;
        }

        fs::write(&path, template.body).with_context(|| format!("write {}", path.display()))?;
        actions.push(AdapterAction {
            path: template.path.into(),
            action: "created".into(),
        });
    }

    Ok(actions)
}

pub fn verify_adapters(repo: &Path) -> Result<Vec<AdapterFailure>> {
    let mut failures = Vec::new();

    for path in ADAPTER_PATHS {
        let full = repo.join(path);
        if !full.exists() {
            continue;
        }

        let text = fs::read_to_string(&full).with_context(|| format!("read {}", full.display()))?;

        if !text.contains("AGENTS.md")
            || !text.contains("agent/JANKURAI_STANDARD.md")
            || !text.contains(master_plan_pointer())
            || !text.contains(planner_protocol_pointer())
            || !text.contains("tips/phases/00-phase-index.md")
            || !text.contains("tips/phases/logs/")
        {
            failures.push(AdapterFailure {
                path: (*path).into(),
                problem: "adapter lacks canonical AGENTS.md, standard, MASTER_PLAN, planner protocol, phase index, and phase log pointers".into(),
            });
        }

        let lower = text.to_ascii_lowercase();
        if lower.contains("ignore agents.md")
            || lower.contains("ignore `agents.md`")
            || lower.contains("do not read agents.md")
            || lower.contains("do not use agent/jankurai_standard.md")
            || lower.contains("do not use agent/master_plan.md")
        {
            failures.push(AdapterFailure {
                path: (*path).into(),
                problem: "adapter contains contradictory command/path policy".into(),
            });
        }

        if text.lines().count() > 90 {
            failures.push(AdapterFailure {
                path: (*path).into(),
                problem: "adapter is too long; adapters must not duplicate the full standard".into(),
            });
        }
    }

    Ok(failures)
}

pub fn selected_adapter_paths(ide: &str) -> Vec<&'static str> {
    let mut paths = Vec::new();
    let mut none_selected = false;

    for token in ide.split(',').map(str::trim).filter(|token| !token.is_empty()) {
        match token {
            "none" | "off" | "false" => {
                none_selected = true;
                paths.clear();
            }
            "all" => {
                paths.extend_from_slice(ADAPTER_PATHS);
                break;
            }
            "claude" => {
                paths.push("CLAUDE.md");
                paths.push(".claude/skills/jankurai/SKILL.md");
            }
            "cursor" => paths.push(".cursor/rules/jankurai.mdc"),
            "copilot" | "github" => {
                paths.push(".github/copilot-instructions.md");
                paths.push(".github/instructions/jankurai.instructions.md");
                paths.push(".github/instructions/jankurai-rust.instructions.md");
                paths.push(".github/instructions/jankurai-web.instructions.md");
                paths.push(".github/instructions/jankurai-python-ai.instructions.md");
            }
            "gemini" => paths.push("GEMINI.md"),
            "antigravity" | "agents" => {
                paths.push(".agents/agents.md");
                paths.push(".agents/skills/jankurai/SKILL.md");
                paths.push(".agents/workflows/jankurai-audit.md");
            }
            "aider" => {
                // Reserved for future adapter support. Keep this token accepted so
                // profile UX can evolve without breaking scripts.
            }
            _ => {}
        }
    }

    if none_selected {
        return Vec::new();
    }

    paths.sort_unstable();
    paths.dedup();
    paths
}

/// Resolves a bundled, local, or composite init profile.
///
/// Supported forms:
/// - `rust-ts-postgres`
/// - `rust-api,react-web`
/// - `.jankurai/profiles/my-profile.json`
/// - `my-profile` when `.jankurai/profiles/my-profile.json` exists.
pub fn resolve_profile(repo: &Path, profile: &str) -> Result<ProfileManifest> {
    resolve_profiles(repo, profile)
}

pub fn resolve_profiles(repo: &Path, selector: &str) -> Result<ProfileManifest> {
    let tokens = split_profile_selector(selector);
    if tokens.is_empty() {
        bail!(
            "empty init profile selector. supported: {}",
            supported_profile_ids().join(", ")
        );
    }

    if tokens.len() == 1 {
        return resolve_one_profile(repo, &tokens[0]);
    }

    let mut manifests = Vec::new();
    for token in &tokens {
        manifests.push(resolve_one_profile(repo, token)?);
    }

    let combined = combine_profiles(&manifests)?;
    validate_profile_manifest(&combined)?;
    Ok(combined)
}

pub fn supported_profile_ids() -> Vec<&'static str> {

        "react-web",
        "rust-ts-postgres",
        "b2b-saas",
        "ai-product",
        "regulated-saas",
        "migration-target",
    ]
}

pub fn apply_init_options(mut manifest: ProfileManifest, ide: &str, ci: &str) -> ProfileManifest {
    let selected_adapters = crate::init::adapters::selected_adapter_paths(ide);
    let selected: BTreeSet<&str> = selected_adapters.into_iter().collect();

    manifest.generated_paths.retain(|path| {
        !crate::init::adapters::is_adapter_path(path) || selected.contains(path.as_str())
    });
    manifest.agent_adapters.retain(|path| selected.contains(path.as_str()));
    manifest.generated_refresh_paths.retain(|path| {
        !crate::init::adapters::is_adapter_path(path) || selected.contains(path.as_str())
    });

    if matches!(ci.trim(), "none" | "off" | "false") {
        let ci_paths: BTreeSet<&str> = manifest.ci_templates.iter().map(String::as_str).collect();
        manifest
            .generated_paths
            .retain(|path| !ci_paths.contains(path.as_str()));
        manifest.ci_templates.clear();
    }

    sort_manifest_lists(&mut manifest);
    manifest
}

fn resolve_one_profile(repo: &Path, profile: &str) -> Result<ProfileManifest> {
    let raw = profile.trim();
    let normalized = normalize_profile_id(raw);

    if let Some(json) = bundled_profile_json(normalized) {
        return load_profile(repo, json, format!("bundled:{normalized}"));
    }

    if let Some(path) = local_profile_path(repo, raw)? {
        return load_profile_file(repo, &path);
    }

    bail!(
        "unknown init profile `{raw}`. supported: {} (plus aliases). composite selectors may use commas, for example `rust-api,react-web`. local profile JSON may be supplied as `--profile .jankurai/profiles/<id>.json`",
        supported_profile_ids().join(", ")
    );
}

fn bundled_profile_json(profile: &str) -> Option<&'static str> {
    match profile {
        "rust-ts-postgres" => Some(RUST_TS_POSTGRES_JSON),
        "rust-api" => Some(RUST_API_JSON),
        "react-web" => Some(REACT_WEB_JSON),
        "b2b-saas" => Some(B2B_SAAS_JSON),
        "ai-product" => Some(AI_PRODUCT_JSON),
        "regulated-saas" => Some(REGULATED_SAAS_JSON),
        "migration-target" => Some(MIGRATION_TARGET_JSON),
        _ => None,
    }
}

fn local_profile_path(repo: &Path, profile: &str) -> Result<Option<PathBuf>> {
    let candidate = Path::new(profile);
    let path_like = profile.ends_with(".json") || profile.contains('/') || profile.contains('\\');

    if path_like {
        let path = if candidate.is_absolute() {
            candidate.to_path_buf()
        } else {
            repo.join(candidate)
        };

        if path.is_file() {
            return Ok(Some(path));
        }

        bail!("init profile file `{}` not found", path.display());
    }

    let path = repo
        .join(".jankurai")
        .join("profiles")
        .join(format!("{profile}.json"));

    if path.is_file() {
        return Ok(Some(path));
    }

    Ok(None)
}

fn load_profile_file(repo: &Path, path: &Path) -> Result<ProfileManifest> {
    let json = fs::read_to_string(path).with_context(|| format!("read {}", path.display()))?;
    load_profile(repo, &json, format!("local:{}", path.display()))
}

fn load_profile(repo: &Path, json_str: &str, source: impl Into<String>) -> Result<ProfileManifest> {
    let source = source.into();
    let value: serde_json::Value = serde_json::from_str(json_str)
        .with_context(|| format!("parse init profile manifest from {source}"))?;

    validation::validate_value(repo, ArtifactSchema::InitProfile, &value)
        .with_context(|| format!("validate init profile manifest from {source}"))?;

    let file: ProfileManifestFile = serde_json::from_value(value)
        .with_context(|| format!("deserialize init profile manifest from {source}"))?;

    let manifest = file.into_manifest(source);
    validate_profile_manifest(&manifest)?;
    Ok(manifest)
}

fn split_profile_selector(selector: &str) -> Vec<String> {
    selector
        .split(|ch| ch == ',' || ch == '+')
        .map(str::trim)
        .filter(|token| !token.is_empty())
        .map(ToOwned::to_owned)
        .collect()
}

fn normalize_profile_id(profile: &str) -> &str {

        | "rust-ts-vite-react-postgres"

fn combine_profiles(manifests: &[ProfileManifest]) -> Result<ProfileManifest> {
    if manifests.is_empty() {
        bail!("cannot combine an empty profile set");
    }

    let mut combined = ProfileManifest {
        profile_version: "1.0.0".into(),
        id: format!(
            "composite-{}",
            manifests
                .iter()
                .map(|manifest| manifest.id.as_str())
                .collect::<Vec<_>>()
                .join("-")
        ),
        display_name: format!(
            "Composite ({})",
            manifests
                .iter()
                .map(|manifest| manifest.display_name.as_str())
                .collect::<Vec<_>>()
                .join(" + ")
        ),
        description: format!(
            "Composite init profile selected from: {}",
            manifests
                .iter()
                .map(|manifest| manifest.id.as_str())
                .collect::<Vec<_>>()
                .join(", ")
        ),
        target_stack_id: manifests
            .iter()
            .map(|manifest| manifest.target_stack_id.as_str())
            .collect::<Vec<_>>()
            .join("+"),
        generated_paths: Vec::new(),
        generated_refresh_paths: Vec::new(),
        reserved_paths: Vec::new(),
        required_lanes: Vec::new(),
        optional_lanes: Vec::new(),
        agent_adapters: Vec::new(),
        ci_templates: Vec::new(),
        docs: Vec::new(),
        security_controls: Vec::new(),
        ux_controls: Vec::new(),
        contract_system: Vec::new(),
        db_policy: Vec::new(),
        validation_commands: Vec::new(),
        source: format!(
            "composite:{}",
            manifests
                .iter()
                .map(|manifest| manifest.source.as_str())
                .collect::<Vec<_>>()
                .join(",")
        ),
    };

    for manifest in manifests {
        extend_unique(&mut combined.generated_paths, &manifest.generated_paths);
        extend_unique(
            &mut combined.generated_refresh_paths,
            &manifest.generated_refresh_paths,
        );
        extend_unique(&mut combined.reserved_paths, &manifest.reserved_paths);
        extend_unique(&mut combined.required_lanes, &manifest.required_lanes);
        extend_unique(&mut combined.optional_lanes, &manifest.optional_lanes);
        extend_unique(&mut combined.agent_adapters, &manifest.agent_adapters);
        extend_unique(&mut combined.ci_templates, &manifest.ci_templates);
        extend_unique(&mut combined.docs, &manifest.docs);
        extend_unique(
            &mut combined.security_controls,
            &manifest.security_controls,
        );
        extend_unique(&mut combined.ux_controls, &manifest.ux_controls);
        extend_unique(&mut combined.contract_system, &manifest.contract_system);
        extend_unique(&mut combined.db_policy, &manifest.db_policy);
        extend_unique(
            &mut combined.validation_commands,
            &manifest.validation_commands,
        );
    }

    let required: BTreeSet<String> = combined.required_lanes.iter().cloned().collect();
    combined
        .optional_lanes
        .retain(|lane| !required.contains(lane));

    sort_manifest_lists(&mut combined);
    Ok(combined)
}

fn validate_profile_manifest(manifest: &ProfileManifest) -> Result<()> {
    if manifest.id.trim().is_empty() {
        bail!("init profile id may not be empty");
    }
    if manifest.display_name.trim().is_empty() {
        bail!("init profile `{}` displayName may not be empty", manifest.id);
    }
    if manifest.target_stack_id.trim().is_empty() {
        bail!(
            "init profile `{}` targetStackId may not be empty",
            manifest.id
        );
    }
    if manifest.generated_paths.is_empty() {
        bail!("init profile `{}` generatedPaths may not be empty", manifest.id);
    }
    if manifest.required_lanes.is_empty() {
        bail!("init profile `{}` requiredLanes may not be empty", manifest.id);
    }
    if manifest.validation_commands.is_empty() {
        bail!(
            "init profile `{}` validationCommands may not be empty",
            manifest.id
        );
    }

    validate_path_list("generatedPaths", &manifest.generated_paths)?;
    validate_path_list("generatedRefreshPaths", &manifest.generated_refresh_paths)?;
    validate_path_list("reservedPaths", &manifest.reserved_paths)?;
    validate_path_list("agentAdapters", &manifest.agent_adapters)?;
    validate_path_list("ciTemplates", &manifest.ci_templates)?;
    validate_path_list("docs", &manifest.docs)?;
    validate_path_list("securityControls", &manifest.security_controls)?;
    validate_path_list("uxControls", &manifest.ux_controls)?;
    validate_path_list("contractSystem", &manifest.contract_system)?;
    validate_path_list("dbPolicy", &manifest.db_policy)?;

    let generated: BTreeSet<&str> = manifest.generated_paths.iter().map(String::as_str).collect();

    for required in CORE_CONTROL_PATHS {
        if !generated.contains(required) {
            bail!(
                "init profile `{}` must include core control path `{required}`",
                manifest.id
            );
        }
    }

    require_subset(
        &manifest.id,
        "generatedRefreshPaths",
        &manifest.generated_refresh_paths,
        &generated,
    )?;
    require_subset(
        &manifest.id,
        "agentAdapters",
        &manifest.agent_adapters,
        &generated,
    )?;
    require_subset(&manifest.id, "ciTemplates", &manifest.ci_templates, &generated)?;
    require_subset(&manifest.id, "docs", &manifest.docs, &generated)?;
    require_subset(
        &manifest.id,
        "securityControls",
        &manifest.security_controls,
        &generated,
    )?;
    require_subset(&manifest.id, "uxControls", &manifest.ux_controls, &generated)?;
    require_subset(
        &manifest.id,
        "contractSystem",
        &manifest.contract_system,
        &generated,
    )?;
    require_subset(&manifest.id, "dbPolicy", &manifest.db_policy, &generated)?;

    let required_lanes: BTreeSet<&str> = manifest.required_lanes.iter().map(String::as_str).collect();
    let optional_lanes: BTreeSet<&str> = manifest.optional_lanes.iter().map(String::as_str).collect();
    for lane in required_lanes.intersection(&optional_lanes) {
        bail!(
            "init profile `{}` declares lane `{lane}` as both required and optional",
            manifest.id
        );
    }

    if required_lanes.contains("ux") && manifest.ux_controls.is_empty() {
        bail!(
            "init profile `{}` requires UX lane but declares no uxControls",
            manifest.id
        );
    }

    Ok(())
}

fn validate_path_list(field: &str, paths: &[String]) -> Result<()> {
    let mut seen = BTreeSet::new();

    for path in paths {
        validate_relative_path(field, path)?;

        if !seen.insert(path.as_str()) {
            bail!("{field} contains duplicate path `{path}`");
        }
    }

    Ok(())
}

fn validate_relative_path(field: &str, path: &str) -> Result<()> {
    if path.trim().is_empty() {
        bail!("{field} contains an empty path");
    }
    if path.starts_with('/') {
        bail!("{field} path `{path}` must be relative");
    }
    if path.contains('\\') {
        bail!("{field} path `{path}` must use forward slashes");
    }
    if path == "."
        || path.split('/').any(|part| part.is_empty() || part == "." || part == "..")
    {
        bail!("{field} path `{path}` contains an unsafe segment");
    }

    for forbidden in ["reference", "target", ".git"] {
        if path == forbidden || path.starts_with(&format!("{forbidden}/")) {
            bail!("{field} path `{path}` is reserved and may not be generated by init");
        }
    }

    Ok(())
}

fn require_subset(
    profile_id: &str,
    field: &str,
    values: &[String],
    generated: &BTreeSet<&str>,
) -> Result<()> {
    for value in values {
        if !generated.contains(value.as_str()) {
            bail!(
                "init profile `{profile_id}` declares `{value}` in {field}, but it is absent from generatedPaths"
            );
        }
    }

    Ok(())
}

fn sorted_dedup(mut values: Vec<String>) -> Vec<String> {
    values.sort();
    values.dedup();
    values
}

fn sort_manifest_lists(manifest: &mut ProfileManifest) {
    manifest.generated_paths = sorted_dedup(std::mem::take(&mut manifest.generated_paths));
    manifest.generated_refresh_paths =
        sorted_dedup(std::mem::take(&mut manifest.generated_refresh_paths));

    manifest.agent_adapters = sorted_dedup(std::mem::take(&mut manifest.agent_adapters));

fn extend_unique(target: &mut Vec<String>, source: &[String]) {
    for item in source {
        if !target.iter().any(|existing| existing == item) {
            target.push(item.clone());
        }
    }

fn apply_templates(
    repo: &Path,
    manifest: &crate::init::profiles::ProfileManifest,
    force_generated_adapters: bool,
) -> Result<Vec<InitAction>> {
    let mut paths = manifest.generated_paths.clone();
    paths.sort();

    let refreshable: BTreeSet<&str> = manifest
        .generated_refresh_paths
        .iter()
        .map(String::as_str)
        .collect();

    let mut actions = Vec::new();

    for rel in paths {
        let template = crate::init::templates::template_for_path(&rel)
            .with_context(|| format!("no template registered for profile path `{rel}`"))?;

        let path = repo.join(&rel);

        if let Some(parent) = path.parent() {
            fs::create_dir_all(parent).with_context(|| format!("create {}", parent.display()))?;
        }

        if path.exists() {
            let text = fs::read_to_string(&path).unwrap_or_default();
            let can_refresh = refreshable.contains(rel.as_str())
                && crate::init::adapters::is_adapter_path(&rel);

            if force_generated_adapters && can_refresh {
                if text.contains(crate::init::adapters::GENERATED_MARKER) {
                    fs::write(&path, template.body)
                        .with_context(|| format!("write {}", path.display()))?;
                    maybe_make_executable(&path, &rel)?;
                    actions.push(InitAction {
                        path: rel,
                        action: "refreshed-generated-adapter".into(),
                        detail: Some("existing adapter carried the generated marker".into()),
                    });
                    continue;
                }

                actions.push(InitAction {
                    path: rel,
                    action: "kept-existing-unmarked-adapter".into(),
                    detail: Some(
                        "adapter was not refreshed because it lacks the generated marker".into(),
                    ),
                });
                continue;
            }

            if should_mark_for_merge(&rel) {
                if is_jankurai_controlled(&text) {
                    actions.push(InitAction {
                        path: rel,
                        action: "kept-existing".into(),
                        detail: Some("existing canonical file already points at jankurai".into()),
                    });
                } else {
                    append_merge_marker(&path, &rel, &text)?;
                    actions.push(InitAction {
                        path: rel,
                        action: "merge-marker".into(),
                        detail: Some("existing user-owned file preserved with a merge marker".into()),
                    });
                }
            } else {
                actions.push(InitAction {
                    path: rel,
                    action: "kept-existing".into(),
                    detail: Some("existing user-owned file preserved".into()),
                });
            }

            continue;
        }

        fs::write(&path, template.body).with_context(|| format!("write {}", path.display()))?;
        maybe_make_executable(&path, &rel)?;

        actions.push(InitAction {
            path: rel,
            action: "created".into(),
            detail: None,
        });
    }

    Ok(actions)
}

        }
    }

# Installing and Bootstrapping jankurai

## Init profiles

Bundled init profiles are JSON manifests validated against `schemas/init-profile.schema.json`.

Supported profiles:

| Profile            | Use when                                              | Includes                                                           |
| ------------------ | ----------------------------------------------------- | ------------------------------------------------------------------ |
| `rust-api`         | Rust API with contracts and DB policy, no web surface | Rust workspace skeleton, contracts, DB docs, security lane         |
| `react-web`        | Web-only React/Vite surface                           | web placeholder, UX policy, contracts, no DB ownership             |
| `rust-ts-postgres` | Default fullstack COLD scaffold                       | Rust API, web slot, Postgres policy, contracts, UX, security       |
| `b2b-saas`         | SaaS product foundation                               | default fullstack plus auth/org/admin/observability docs           |
| `ai-product`       | Bounded AI product                                    | Python AI containment, prompts, evals, AI/product docs             |
| `regulated-saas`   | Evidence-heavy SaaS                                   | fullstack plus security, privacy, compliance, backup/restore shell |
| `migration-target` | Legacy modernization target                           | migration boundary map, slices, contracts, proof lanes             |

Aliases:

* `default`, `fullstack`, `rust-ts-vite-react-postgres`, and `rust-ts-vite-react-postgres-bounded-python` resolve to `rust-ts-postgres`.
* `react`, `web`, and `vite-react` resolve to `react-web`.
* `ai` resolves to `ai-product`.
* `regulated` resolves to `regulated-saas`.
* `migration` resolves to `migration-target`.

## Dry-run first

```bash
jankurai init --profile rust-ts-postgres --ide all --mode advisory --dry-run --plan-json target/jankurai/init-plan.json
```

Review the plan. The planned actions are exactly the selected manifest paths after `--ide` and `--ci` filtering.

## Apply

```bash
jankurai init --profile rust-ts-postgres --ide all --mode advisory --yes
jankurai audit . --json agent/repo-score.json --md agent/repo-score.md
jankurai doctor . --fail-on high
```

## Composite profiles

Profiles can be composed with commas:

```bash
jankurai init --profile rust-api,react-web --dry-run
```

The generator unions paths, proof lanes, docs, controls, and validation commands. Required lanes win over optional lanes.

## Local profile JSON

A repo can carry a local profile:

```bash
mkdir -p .jankurai/profiles
cp path/to/profile.json .jankurai/profiles/my-profile.json
jankurai init --profile my-profile --dry-run
```

A direct path also works:

```bash
jankurai init --profile .jankurai/profiles/my-profile.json --dry-run
```

Local profiles are validated with the same schema and safety checks as bundled profiles. Unsafe paths such as `reference/`, `target/`, `.git/`, absolute paths, backslashes, or `..` segments are rejected.

## IDE adapters

Use `--ide` to control generated adapter files:

```bash
jankurai init --profile rust-ts-postgres --ide claude --dry-run
jankurai init --profile rust-ts-postgres --ide cursor,copilot --dry-run
jankurai init --profile rust-ts-postgres --ide none --dry-run
```

Generated adapters carry a marker. They are refreshed only when both conditions are true:

1. the existing file contains `jankurai generated adapter`
2. `--force-generated-adapters` is passed

User-owned adapter files without the marker are preserved.

## CI templates

The bundled CI selector supports:

```bash
--ci github
--ci none
```

`--ci none` omits `.github/workflows/jankurai.yml`.

## Merge behavior

`jankurai init` never overwrites existing user files.

For canonical guidance files (`AGENTS.md`, `agent/JANKURAI_STANDARD.md`, `agent/MASTER_PLAN.md`), init appends a merge marker when the file exists but does not already point at jankurai. This makes adoption visible without destroying prior guidance.

For all other paths, existing files are kept unchanged.

## Generated versus source files

Generated evidence belongs under `target/jankurai/` unless a generated zone declares another path.

Source files include:

* `contracts/`
* `db/migrations/`
* `docs/`
* Rust source
* web source
* prompt and eval source

Generated outputs must have a source and regeneration command in `agent/generated-zones.toml`.
