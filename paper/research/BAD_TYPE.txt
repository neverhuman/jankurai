\section{Inexcusable TYPESCRIPT Anti-Patterns}
\label{sec:bad_type}

The following section categorizes the most severe, inexcusable TYPESCRIPT anti-patterns observed in production environments and AI-generated code. These patterns compromise safety, maintainability, and performance.\

\subsection{Known Best Practices}
To counteract the anti-patterns, ensure adherence to these core TYPESCRIPT best practices:
\begin{itemize}
  \item Eliminate \texttt{any} types: Use \texttt{unknown} for unvalidated data and enforce narrowing with type guards.
  \item Validate data at the boundaries: Use schema validation libraries (e.g., Zod) for API responses and inputs.
  \item Prefer strict mode: Enable \texttt{"strict": true} and \texttt{"noUncheckedIndexedAccess": true}.
  \item Keep types aligned with runtime reality: Avoid lying to the compiler with type assertions or non-null assertions.
  \item Model illegal states out: Use discriminated unions to represent mutually exclusive states.
\end{itemize}

\subsection{1. Silencing TypeScript instead of fixing the problem}
\begin{itemize}
  \item Committing \texttt{// @ts-nocheck} in source files.
  \item Using \texttt{// @ts-ignore} to make a red squiggle disappear.
  \item Using \texttt{// @ts-expect-error} without a precise explanation.
  \item Leaving \texttt{@ts-expect-error} in place after the underlying error is gone.
  \item Disabling a lint rule at file level because one line is hard.
  \item Disabling type-aware linting because it exposes too many real problems.
  \item Changing \texttt{.ts} / \texttt{.tsx} files back to \texttt{.js} just to escape type errors.
  \item Excluding broken source folders from \texttt{tsconfig.json}.
  \item Making CI run tests but not \texttt{tsc --noEmit}.
  \item Allowing deployments when \texttt{tsc} reports errors.
  \item Using \texttt{noEmitOnError: false} in a deploy/build path so broken code still emits JavaScript.
  \item Turning off strict compiler options after the project has already adopted TypeScript.
  \item Changing types until the compiler shuts up without understanding why.
  \item Asking an AI agent to “fix TypeScript errors” and accepting a diff full of casts, \texttt{any}, or suppressions.
  \item Hiding errors inside wrapper functions named things like \texttt{unsafeCast}, \texttt{forceType}, \texttt{fixType}, or \texttt{asWhatever}.
  \item Treating TypeScript errors as “compiler drama” rather than design feedback.
\end{itemize}

\subsection{1. Type-system escape hatches}
\begin{itemize}
  \item Using \texttt{any} in handwritten production code.
  \item Using \texttt{as any}.
  \item Using \texttt{as unknown as T} to force a value into a type.
  \item Using \texttt{unknown} as a cosmetic replacement for \texttt{any}, then immediately casting it away.
  \item Returning \texttt{any} from public functions.
  \item Accepting \texttt{any} in public APIs.
  \item Storing \texttt{any} in state, context, caches, services, stores, Redux/Zustand/Pinia/etc.
  \item Using \texttt{Record<string, any>}.
  \item Using \texttt{Array<any>}, \texttt{any[]}, \texttt{Promise<any>}, \texttt{Map<string, any>}, \texttt{Set<any>}.
  \item Passing \texttt{any} into generics, such as \texttt{UseQueryResult<any>} or \texttt{Response<any>}.
  \item Using \texttt{Function} instead of a real function signature.
  \item Using \texttt{{}} as “any object.”
  \item Using \texttt{object} when the code needs a real object shape.
  \item Using boxed primitive types: \texttt{String}, \texttt{Number}, \texttt{Boolean}, \texttt{Symbol}, \texttt{Object}.
  \item Using \texttt{T = any} as a generic default.
  \item Using \texttt{T extends any}.
  \item Using unconstrained generics when the implementation assumes properties exist.
  \item Using unused generic parameters.
  \item Using a generic only to avoid writing a real type.
  \item Using \texttt{never} to silence impossible-state errors instead of modeling the state correctly.
  \item Typing a value as \texttt{unknown} but never narrowing it.
  \item Typing parsed JSON as \texttt{SomeType} without validation.
  \item Typing \texttt{fetch().json()} as a trusted domain object without schema validation.
  \item Typing environment variables as present without checking them.
  \item Typing feature flags, cookies, route params, search params, localStorage, sessionStorage, postMessage, WebSocket messages, queue messages, or database rows as trusted without validation.
  \item Using type assertions to access properties on external input.
  \item Using type assertions to bypass discriminated-union narrowing.
  \item Using type assertions to silence excess property errors.
  \item Using type assertions to bypass readonly/immutability.
  \item Using type assertions to bypass \texttt{null}/\texttt{undefined}.
  \item Using type assertions to make tests pass without proving behavior.
  \item Using type assertions in reducers/state transitions instead of modeling the transition.
  \item Casting an API response to a generated type while ignoring version drift.
  \item Casting third-party library results instead of wrapping/narrowing them.
  \item Casting \texttt{Error} values in \texttt{catch} instead of narrowing from \texttt{unknown}.
  \item Casting DOM elements without checking they exist and have the expected type.
  \item Casting \texttt{querySelector} results and immediately dereferencing them.
  \item Casting dates, decimals, IDs, or branded values from strings without parsing/validating.
  \item Casting string literals into union members instead of deriving/validating them.
  \item Casting object keys to \texttt{keyof T} when the object was not proven to contain the key.
\end{itemize}

\subsection{10. Module, import, and build misconduct}
\begin{itemize}
  \item Using \texttt{require} in modern TypeScript modules without a clear interop reason.
  \item Mixing CJS and ESM casually.
  \item Using \texttt{esModuleInterop}/synthetic default imports to paper over incorrect runtime assumptions.
  \item Importing types as values and causing runtime side effects.
  \item Importing values only used as types.
  \item Not using \texttt{import type} where required by the project’s module settings.
  \item Relying on path aliases that only TypeScript understands.
  \item Relying on \texttt{ts-node} behavior that production Node does not share.
  \item Relying on bundler-only resolution in library code.
  \item Publishing packages without checking the emitted JS actually resolves.
  \item Publishing packages whose \texttt{.d.ts} files reference private/internal paths.
  \item Publishing packages without declarations when consumers need types.
  \item Publishing broken \texttt{exports}/\texttt{types} mappings.
  \item Publishing ESM/CJS dual packages without testing both consumption modes.
  \item Importing from package internals that are not public API.
  \item Creating circular dependencies that produce partially initialized values.
  \item Creating barrel files with hidden side effects.
  \item Using barrels that destroy tree-shaking or create circular imports.
  \item Depending on import order for side effects.
  \item Using global augmentations when explicit imports would work.
  \item Polluting global namespaces.
  \item Using triple-slash references instead of normal imports except where specifically required.
  \item Mixing test-only imports into production code.
  \item Accidentally including test files in production builds.
  \item Accidentally excluding source files from type checking.
  \item Emitting JS into \texttt{src}.
  \item Shipping source maps that reveal secrets or private source when that is not intended.
  \item Using stale module-resolution modes in new projects.
  \item Ignoring TypeScript’s host/runtime module-model guidance.
\end{itemize}

\subsection{10. Unsafe async and promise handling}
\begin{itemize}
  \item Floating promises.
  \item Missing \texttt{await}.
  \item \texttt{if (somePromise)} instead of \texttt{if (await somePromise)}.
  \item Unhandled promise rejections.
  \item \texttt{array.forEach(async item => ...)}.
  \item Async callbacks passed where return value is ignored.
  \item Async React event/effect code with unhandled errors.
  \item Fire-and-forget promises without explicit \texttt{void}, logging, cancellation, and error handling.
  \item \texttt{Promise.all} used when partial failure must be handled.
  \item \texttt{Promise.all} over unbounded user-sized arrays.
  \item Swallowed \texttt{.catch(() => {})}.
  \item \texttt{try/catch} around code that starts promises but does not await them.
  \item Tests that forget to \texttt{await expect(...).resolves/rejects}.
  \item API handlers that start background work and return success before work is durably queued.
  \item Mixing callbacks and promises in the same code path.
  \item Async constructors or initialization hidden behind half-initialized objects.
  \item Using sleeps/timeouts instead of awaiting deterministic signals.
  \item Ignoring \texttt{AbortSignal}.
  \item Not cleaning timers, subscriptions, sockets, or event listeners.
\end{itemize}

\subsection{11. Bad error handling}
\begin{itemize}
  \item \texttt{catch (e: any)}.
  \item Empty \texttt{catch {}} blocks.
  \item Logging and continuing after unrecoverable errors.
  \item Throwing strings.
  \item Throwing plain objects with no stack.
  \item Replacing specific errors with \texttt{throw new Error("failed")} and losing context.
  \item Ignoring \texttt{cause}.
  \item Returning \texttt{null} for errors in one function, throwing in another, and returning \texttt{{ error }} elsewhere with no convention.
  \item Treating validation errors as impossible.
  \item Swallowing parse errors and using defaults silently.
  \item Catching errors only to satisfy TypeScript.
  \item Logging secrets/tokens/PII in error objects.
  \item Sending raw internal errors to clients.
  \item Hiding async errors in detached promises.
  \item Converting all errors to \texttt{any}.
  \item Assuming \texttt{catch} values are \texttt{Error}.
\end{itemize}

  \item Declaring \texttt{module "*"} as \texttt{any}.
  \item Declaring missing packages as \texttt{any} instead of minimal safe types.
  \item Publishing declarations not generated from source.
  \item Publishing declarations from code that failed type checking.
  \item Publishing declarations that expose private implementation details.
  \item Publishing declarations with broad \texttt{any}.
  \item Publishing declarations with wrong nullability.
  \item Publishing declarations with wrong optionality.
  \item Publishing declarations with wrong overload order.
  \item Publishing declarations with callback arity mistakes.
  \item Publishing declarations with unused generics.
  \item Publishing declarations that use boxed primitive types.
  \item Publishing declarations that hide thrown/rejected error behavior.
  \item Publishing declarations that omit side effects.
  \item Publishing declarations that describe a different version of the runtime package.
  \item Modifying global interfaces from a library without an extremely explicit reason.
  \item Using declaration merging to hide design problems.
  \item Using ambient declarations where explicit modules would be safer.
  \item Letting generated declarations drift from implementation.
  \item Not testing public type behavior with type-level tests.
\end{itemize}

\subsection{12. Fake domain modeling}
\begin{itemize}
  \item \texttt{type Status = string} when only specific statuses are allowed.
  \item \texttt{type Role = string}.
  \item \texttt{type UserId = string} interchangeable with \texttt{OrgId}, \texttt{PostId}, \texttt{SessionId}.
  \item Boolean flag explosions like \texttt{{ isAdmin, isOwner, isEditor, isViewer }}.
  \item Optional-field blobs representing multiple states.
  \item \texttt{interface Thing { type: string }} instead of discriminated unions.
  \item State machines represented as loose objects.
  \item Magic strings copied across files.
  \item Dates represented inconsistently as \texttt{Date}, ISO string, timestamp number, and nullable string.
  \item Money represented as floating-point numbers without currency/minor-unit rules.
  \item Permissions represented only in UI components.
  \item Domain invariants only in comments.
  \item \texttt{Partial<T>} used as a substitute for proper create/update DTOs.
  \item \texttt{Record<string, T>} used when keys are a known union.
  \item \texttt{object} / \texttt{{}} used instead of real shapes.
  \item Optional properties used because the AI does not know which state owns which fields.
  \item One “MegaType” shared across frontend, backend, DB, and API even though each layer has different invariants.
\end{itemize}

\subsection{13. Non-exhaustive logic}
\begin{itemize}
  \item \texttt{switch} over a union without exhaustiveness checking.
  \item \texttt{default: return null} hiding unhandled cases.
  \item Adding a new union member without updating all handlers.
  \item \texttt{if/else} ladders over literal states with no final \texttt{never} check.
  \item Reducers that ignore unknown actions.
  \item State machines that silently fall back to initial state.
  \item Permission checks with catch-all allow behavior.
  \item Feature flag variants with no default-deny path.
  \item Enum/string union conversions with unchecked casts.
  \item \texttt{as never} to make exhaustive checks pass.
\end{itemize}

\subsection{14. Bad public API types}
\begin{itemize}
  \item Exported functions with inferred accidental return types when the return shape is part of the contract.
  \item Public APIs returning \texttt{any}, \texttt{unknown}, \texttt{{}}, \texttt{object}, or \texttt{Record<string, unknown>} without a reason.
  \item Package \texttt{.d.ts} files that lie about runtime behavior.
  \item \texttt{declare module "foo";} to silence missing types.
  \item \texttt{declare module "*";}.
  \item Global augmentations that affect the whole app unnecessarily.
  \item Re-exporting unstable internal types as public API.
  \item Changing exported types without versioning.
  \item Publishing generated types that were not produced by CI/codegen.
  \item Mismatched runtime exports and type declarations.
  \item Type-only breaking changes treated as harmless.
  \item Exporting broad DTOs that let callers construct invalid states.
  \item Public generics that accept impossible combinations.
  \item \texttt{Function}, \texttt{Object}, \texttt{String}, \texttt{Number}, or \texttt{Boolean} in public types.
\end{itemize}

\subsection{15. Generic type abuse}
\begin{itemize}
  \item Generics that do not relate input to output.
  \item \texttt{<T>(x: unknown) => x as T}.
  \item \texttt{<T = any>}.
  \item \texttt{<T extends object>} when primitives are valid or when the shape is known.
  \item Fake type-safe fetch helpers: \texttt{api.get<T>(url): Promise<T>} with no decoder.
  \item Generic repositories/services that erase domain constraints.
  \item Deep conditional types no one can debug.
  \item Recursive mapped types that destroy compiler performance.
  \item Over-abstracted utility types instead of a simple interface.
  \item Type-level programming to avoid writing runtime logic.
  \item \texttt{keyof any} where \texttt{PropertyKey} or a narrower key type is intended.
  \item \texttt{T as unknown as U} inside generic helpers.
  \item Generic “form builders” that allow impossible field/value pairs.
  \item Using \texttt{Partial<T>}, \texttt{Pick<T>}, and \texttt{Omit<T>} mechanically instead of designing boundary types.
  \item Overusing \texttt{ReturnType<typeof fn>} across module boundaries where an explicit contract is clearer.
  \item Hiding breaking changes behind utility types.
\end{itemize}

  \item \texttt{arr[0].id} without checking length.
  \item \texttt{record[key].value} when \texttt{key} may be absent.
  \item \texttt{map.get(key).value}.
  \item \texttt{object[someString]} without an index signature or key guard.
  \item Treating \texttt{process.env.X} as definitely present.
  \item Treating URL params as definitely present.
  \item Using \texttt{!} after every indexed access.
  \item Turning off \texttt{noUncheckedIndexedAccess} because it is “annoying.”
  \item Using \texttt{Record<string, T>} for maps where absence is possible but not modeled.
  \item Assuming non-empty arrays without a \texttt{NonEmptyArray} type or runtime check.
  \item Indexing tuple-like arrays as if every position exists.
  \item Relying on array order from external APIs.
\end{itemize}

  \item Truthy checks on arrays to mean “non-empty.”
  \item Truthy checks on objects to mean “has keys.”
  \item Optional chaining that masks missing required data.
  \item Nullish and falsy checks mixed casually.
