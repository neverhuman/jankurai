\section{Inexcusable GIT Anti-Patterns}
\label{sec:bad_git}

The following section categorizes the most severe, inexcusable GIT anti-patterns observed in production environments and AI-generated code. These patterns compromise safety, maintainability, and performance.\

\subsection{Known Best Practices}
To counteract the anti-patterns, ensure adherence to these core GIT best practices:
\begin{itemize}
  \item Write semantic commit messages: Provide clear context and use conventional commits.
  \item Never commit secrets: Use \texttt{.gitignore} strictly and rely on secret management tools.
  \item Keep history linear and clean: Rebase local branches before merging, and squash WIP commits.
  \item Branch with intent: Use feature branches for isolated work and delete them post-merge.
  \item Review before merging: Enforce pull requests, CI checks, and peer review for all shared branches.
\end{itemize}

\subsection{0. The root bad behavior: treating dirtiness as trash}
\begin{itemize}
  \item Treating a dirty worktree as a problem to “fix” instead of state to preserve.
  \item Assuming untracked files are disposable.
  \item Assuming ignored files are disposable.
  \item Assuming local commits ahead of origin are accidental.
  \item Assuming a branch that differs from \texttt{origin/main} must be force-synced.
  \item Treating “clean up” as permission to delete files.
  \item Treating “commit your changes” as permission to stage unrelated changes.
  \item Treating “make tests pass” as permission to revert, delete, or rewrite user work.
  \item Treating “use Git” as permission to run destructive Git commands.
  \item Treating “the repo is messy” as justification to copy, stash, reset, or clean without a manifest.
  \item Treating generated files as trash without checking whether they are source artifacts.
  \item Treating local-only state as less valuable than remote state.
  \item Treating an agent’s own convenience as more important than the user’s recoverability.
  \item Optimizing for “green status” instead of “preserve every byte I did not create.”
  \item Failing to understand that **untracked files may be the only copy of important work**.
\end{itemize}

\subsection{1. Destructive reset/restore/clean behavior}
\begin{itemize}
  \item Running \texttt{git restore .}, \texttt{git checkout -- .}, \texttt{git checkout -- <file>}, or similar commands to “clean up” without explicit approval.
  \item Running \texttt{git reset --hard} to get to a “known good state” while the user has uncommitted work.
  \item Running \texttt{git clean -fd}, \texttt{git clean -fdx}, or \texttt{git clean -xdf} to remove “junk,” thereby deleting untracked work, generated outputs, experiments, local notes, test fixtures, or user-created files. Git’s own docs describe \texttt{git clean} as removing untracked files and directories, and \texttt{-x} can remove ignored build products too.
  \item Treating untracked files as disposable. They may be the user’s newest work.
  \item Treating ignored files as disposable. Ignored files often include local databases, \texttt{.env} files, generated test data, downloaded assets, or benchmark outputs.
  \item Running \texttt{git stash -u} or \texttt{git stash --all} without understanding that those options also clean untracked or ignored files from the worktree after stashing.
  \item Doing \texttt{stash pop} into the wrong branch or wrong worktree, causing conflicts or silently dropping changes.
  \item Losing track of stash entries, then telling the user “nothing is lost” when recovery is uncertain.
  \item Running \texttt{rm -rf} cleanup commands with unsafe paths, bad shell expansion, or overly broad globs.
  \item Deleting \texttt{.git/}, \texttt{.git/worktrees/}, \texttt{.gitmodules}, \texttt{.git/info}, refs, hooks, or packed refs.
  \item Reinitializing a repo with \texttt{git init} because Git looks “broken,” thereby masking or damaging the original repository state.
  \item Removing a linked worktree with uncommitted work, or using force cleanup to bypass Git’s safety checks. Git’s worktree docs say only clean worktrees can normally be removed; force removes unclean worktrees or those with submodules.
  \item Running \texttt{git worktree prune} or deleting worktree folders without first checking whether the worktree contains uncommitted changes, unpushed commits, or active agent sessions.
  \item Running destructive commands after seeing unexpected changes instead of stopping and asking. OpenAI’s Codex prompting guide explicitly says not to revert user changes, not to amend commits unless requested, and never to use destructive commands like \texttt{git reset --hard} or \texttt{git checkout --} unless specifically requested or approved.
\end{itemize}

\subsection{1. Dirty-worktree negligence}
\begin{itemize}
  \item **Does not run \texttt{git status --short --branch} before changing files.**
  \item **Does not inspect staged vs. unstaged vs. untracked files before acting.**
  \item **Treats a dirty tree as “probably mine” instead of “unknown ownership.”**
  \item **Edits on top of a user’s uncommitted work without first identifying what belongs to whom.**
  \item **Stages or commits the user’s existing changes along with the agent’s changes.**
  \item **Overwrites user edits because it assumes the last-read file is still current.**
  \item **Ignores untracked files because \texttt{git diff} is empty.**
  \item **Assumes \texttt{git reset --hard} restores “everything,” while untracked files remain unless separately cleaned.**
  \item **Runs cleanup based on \texttt{git status -uno}, thereby hiding untracked agent output.** A Claude Code report describes stale cleanup using \texttt{git status --porcelain -uno}, missing untracked files and allowing deletion of worktrees containing unadded output.
  \item **Fails to detect local commits not pushed.**
  \item **Fails to detect remote commits not pulled.**
  \item **Fails to detect branch divergence.**
  \item **Fails to detect an in-progress merge, rebase, cherry-pick, bisect, or revert.**
  \item **Continues after Git reports conflicts or lock failures.**
  \item **Misreports “clean” when ignored files, untracked generated files, or staged changes exist.**
  \item **Does not snapshot the starting commit, branch, and working-tree state.**
  \item **Does not provide a “before” diff before making broad changes.**
  \item **Does not ask the user when the worktree is dirty and ownership is unclear.**
  \item **Interprets “fix it” as permission to reorganize the whole repo.**
  \item **Proceeds in a dirty tree after explicitly being told to avoid dirty trees.**
\end{itemize}

\subsection{1. Preflight and state-discovery failures}
\begin{itemize}
  \item Editing before running \texttt{git status --porcelain=v1 -z}.
  \item Editing before recording \texttt{HEAD}, current branch, remote, and worktree path.
  \item Editing before checking whether the repo has uncommitted user changes.
  \item Editing before checking for untracked files.
  \item Editing before checking for ignored files that may matter.
  \item Editing before checking for staged files.
  \item Editing before checking for stashes.
  \item Editing before checking whether the branch is ahead or behind upstream.
  \item Editing before checking whether the repo is in a merge, rebase, cherry-pick, bisect, or detached-HEAD state.
  \item Editing before checking submodule status.
  \item Editing before checking Git LFS status.
  \item Editing before checking sparse-checkout or partial-clone status.
  \item Editing before checking \texttt{.gitignore}, \texttt{.gitattributes}, hooks, and repo-specific agent instructions.
  \item Parsing human \texttt{git status} output instead of porcelain output.
  \item Using \texttt{git status -uno} and thereby hiding untracked files.
  \item Running Git commands from the wrong directory.
  \item Running Git commands in a parent repo when intending a submodule.
  \item Running Git commands in a submodule when intending the parent repo.
  \item Confusing local machine, cloud sandbox, container, VM, and remote dev host.
  \item Confusing the user’s repo with a copied repo.
  \item Confusing a real branch with a temporary worktree branch.
  \item Failing to distinguish user changes from agent changes.
  \item Failing to create a touched-file manifest.
  \item Failing to save a pre-change patch for files the agent will edit.
  \item Failing to check whether another agent or human is concurrently editing the repo.
  \item Failing to notice that “clean” in an IDE UI may not mean \texttt{git status} clean.
  \item Trusting a stale review panel instead of actual Git state; Cursor users have reported review lists persisting after files were reverted or committed while \texttt{git status} was clean.
\end{itemize}

\subsection{1. Preflight blindness}
\begin{itemize}
  \item Run and read \texttt{git status --short --branch}.
  \item Identify the repo root before editing.
  \item Identify whether they are in the main checkout, a linked worktree, a submodule, a nested repo, or a detached \texttt{HEAD}.
  \item Check whether the worktree is dirty before destructive commands.
  \item Distinguish staged, unstaged, untracked, ignored, and unpushed changes.
  \item Check for in-progress operations: merge, rebase, cherry-pick, bisect, revert, apply/patch, or conflicted index.
  \item Check whether the current branch is protected, shared, published, or user-owned.
  \item Check whether local commits are ahead of the remote.
  \item Check whether the remote branch has diverged.
  \item Record a baseline commit SHA before starting.
  \item Record a baseline diff before editing.
  \item Read project Git instructions such as \texttt{AGENTS.md}, \texttt{CLAUDE.md}, \texttt{.cursor/rules}, contribution docs, or release docs.
  \item Notice changes made by the human while the agent is running.
  \item Stop when unexpected changes appear.
  \item Ask before touching files it did not create.
  \item Notice that the current branch is not the intended target branch.
  \item Notice it is operating in a clone of a clone, a temp checkout, or a stale worktree.
  \item Notice untracked local-only project assets such as docs, datasets, notebooks, database files, screenshots, \texttt{.env}, or customer files.
  \item Notice ignored files that are still valuable.
  \item Notice submodule dirtiness.
  \item Notice Git LFS pointer/object state.
  \item Notice line-ending, filemode, symlink, or case-only rename risk.
\end{itemize}

\subsection{1. Treating “dirty worktree” as a problem to eliminate}
\begin{itemize}
  \item Seeing uncommitted work and deciding the repo must be “cleaned.”
  \item Treating local changes as disposable clutter.
  \item Treating unpushed local commits as a sync error.
  \item Treating “ahead of origin” as wrong.
  \item Treating untracked files as garbage.
  \item Treating staged-but-uncommitted work as safe to overwrite.
  \item Treating “git status is noisy” as permission to reset, stash, clean, or copy the repo.
  \item Optimizing for “command succeeds” rather than “human work is preserved.”
  \item Trying to “restore everything first” before starting a task.
  \item Running cleanup to make tests easier without knowing what is user work.
  \item Assuming a dirty worktree belongs to the agent rather than the human.
  \item Assuming the agent can distinguish its edits from existing human edits without checking.
  \item Assuming the user wants repo state normalized just because they asked for a coding task.
  \item Assuming a clean tree is required before reading, planning, or editing.
\end{itemize}

\subsection{10. Large-file, generated-code, and repository-bloat failures}
\begin{itemize}
  \item **Commits large binaries directly instead of using LFS or external object storage.**
  \item **Commits model weights, datasets, videos, archives, or build bundles.**
  \item **Commits generated lock/cache blobs that change every run.**
  \item **Commits minified bundles when source maps or generated assets are not expected.**
  \item **Commits package tarballs.**
  \item **Commits vendored dependencies without policy.**
  \item **Runs code generation and commits every generated file without identifying the generator.**
  \item **Edits generated code directly instead of the source template/schema.**
  \item **Regenerates files using a different tool version, causing huge unrelated diffs.**
  \item **Changes line endings in generated files.**
  \item **Adds binary files that exceed platform limits.** GitHub recommends single objects stay around 1 MB and enforces 100 MB; it recommends Git LFS for binary files and storing generated files outside Git.
  \item **Creates full repo copies that double or triple disk use and slow all search/build operations.**
  \item **Creates branch/worktree explosions that slow fetches and CI.**
  \item **Does full clones repeatedly in automation rather than using existing object stores, shallow clones, or proper worktrees.**
\end{itemize}

\subsection{11. Multi-agent coordination failures}

  \item Agents deleting each other’s tests.
  \item Agents rebasing on half-finished work from another agent.

  \item Agents touching shared router, schema, migration, lockfile, or config files without coordination.
  \item Two agents adding duplicate helper functions in different modules.
  \item Two agents implementing incompatible versions of the same interface.
  \item One agent updating tests while another changes behavior underneath.
  \item Agents running formatters over each other’s files.
  \item Agents switching branches in shared terminals.
  \item Agents using shared stashes.
  \item Agents using shared temp directories.
  \item Agents using shared branch names.
  \item Agents using shared databases or dev servers and invalidating each other’s results.
  \item Parent agent running destructive Git commands while subagents are active.
  \item Cleaning up subagent work before parent has reviewed it.
  \item Not recording agent ownership of files.
  \item Not locking or claiming files in parallel runs.
  \item Merging agent outputs without integration tests.
  \item Letting parallel speed create more review work than it saves.
\end{itemize}

\subsection{12. Git LFS and binary-file badness}
\begin{itemize}
  \item Committing large binaries directly instead of using LFS where the repo expects LFS.
  \item Replacing LFS pointer files with actual binary contents.
  \item Editing LFS pointer files manually.
  \item Failing to run \texttt{git lfs pull} before editing binary assets.
  \item Failing to install LFS hooks when required.
  \item Adding LFS tracking rules without committing \texttt{.gitattributes}.
  \item Changing \texttt{.gitattributes} broadly and affecting line endings or diff behavior.
  \item Running LFS migration without consent.
  \item Deleting LFS files because only pointers are visible.

  \item Leaving temp files **ignored**, so humans do not see them but tools do.
  \item Leaving temp files that tests import or execute accidentally.
  \item Leaving temp scripts containing destructive commands.
  \item Leaving temp files with secrets.
  \item Leaving temp worktrees with unmerged commits.
  \item Leaving temp branches that block deletion.
  \item Leaving temp \texttt{.git} metadata that breaks IDEs.
  \item Leaving temp copies that double search/build scope.
  \item It is not a Git worktree; it is just a duplicate source tree.
  \item It can be accidentally staged by \texttt{git add .}.
  \item It can be traversed by tests, linters, formatters, type checkers, and search tools.
  \item It can create duplicate symbols, duplicate packages, duplicate migrations, or duplicate test suites.
  \item It can contain stale code that future agents treat as authoritative.
  \item It bloats disk and CI.
  \item It may copy secrets or local config.
  \item It may copy build artifacts and caches.
  \item It may contain a nested \texttt{.git}, confusing Git operations.
  \item It may hide the fact that the agent refused to reason about dirty state.
  \item It is often a symptom of the agent optimizing for its own convenience over repository hygiene.
  \item The correct behavior is usually: stop, inspect dirty state, ask for ownership confirmation, create a proper \texttt{git worktree} outside the tracked tree or under a clearly ignored/managed directory, record ownership/base, and clean it up with \texttt{git worktree remove}.
\end{itemize}

  \item Create raw full-repo copies as a substitute for Git hygiene.
  \item Modify \texttt{.git/} internals directly.
  \item Clean up paths it has not proven are agent-owned.
  \item Delete branches it did not create.
  \item Use shared \texttt{refs/stash} as if it were per-worktree.
  \item Work in a volatile temp directory without warning.
  \item Operate without confirming repo root, branch, dirty state, and ownership of existing changes.
  \item Commit or push unrelated changes.
  \item Commit secrets, temp files, raw copies, caches, or local config.
  \item Hide commands, paths, stashes, worktrees, branches, commits, or skipped tests.
  \item Continue after unexpected changes, wrong CWD, failed checkout, failed LFS/filter/git-crypt operation, or \texttt{.git} corruption.
  \item Assume worktree isolation solves runtime isolation.
\end{itemize}
